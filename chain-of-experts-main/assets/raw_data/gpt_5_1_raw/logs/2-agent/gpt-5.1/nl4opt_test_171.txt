[MODELER]:
Model this as an integer linear programming (ILP) problem.

**Decision variables**

Let  
- \(x\) = number of glass jars to fill (250 ml each)  
- \(y\) = number of plastic jars to fill (300 ml each)  

Both must be nonnegative integers:
\[
x \ge 0,\quad y \ge 0,\quad x, y \in \mathbb{Z}
\]

**Objective**

Maximize the total number of jars:
\[
\max \; Z = x + y
\]

**Constraints**

1. **Honey availability (cannot exceed 20,000 ml):**
   \[
   250x + 300y \le 20000
   \]

2. **At least twice as many plastic jars as glass jars:**
   \[
   y \ge 2x
   \]

3. **At least 20 glass jars:**
   \[
   x \ge 20
   \]

**Complete mathematical model**

\[
\begin{aligned}
\max\quad & x + y \\
\text{s.t.}\quad 
& 250x + 300y \le 20000, \\
& y \ge 2x, \\
& x \ge 20, \\
& x, y \in \mathbb{Z}_{\ge 0}.
\end{aligned}
\]

---

### Solving analytically (to get the numeric answer)

Since each jar uses honey, to maximize the number of jars we want to get as many jars as possible from the given 20,000 ml.

From the constraint \(y \ge 2x\), the minimum \(y\) for a given \(x\) is \(y = 2x\). To maximize the total number of jars, we will try to use **all** the honey:

\[
250x + 300y = 20000
\]
Substitute \(y = 2x\):
\[
250x + 300(2x) = 20000 \\
250x + 600x = 20000 \\
850x = 20000 \\
x = \frac{20000}{850} \approx 23.53
\]
So the best integer candidates near this value are \(x = 23, 24\) (and we must have \(x \ge 20\)).

For each, use \(y = 2x\) (since that minimizes honey per jar and thus tends to maximize the number of jars).

- For \(x = 23\), \(y = 46\):
  \[
  250(23) + 300(46) = 5750 + 13800 = 19550 \le 20000
  \]
  Total jars \(= 23 + 46 = 69\).

- For \(x = 24\), \(y = 48\):
  \[
  250(24) + 300(48) = 6000 + 14400 = 20400 > 20000
  \]
  Not feasible.

Try increasing \(y\) for \(x=23\) to use more honey (more jars):

Let \(x = 23\), increase \(y\) subject to:
\[
250(23) + 300y \le 20000 \Rightarrow 5750 + 300y \le 20000 \Rightarrow 300y \le 14250 \Rightarrow y \le 47.5
\]
So feasible integer \(y \le 47\), and must also satisfy \(y \ge 2x = 46\). So check \(y = 46,47\):

- \(x=23, y=46\): total 69 jars, honey 19,550 ml (feasible).  
- \(x=23, y=47\): total 70 jars, honey \(5750 + 300*47 = 5750 + 14100 = 19850\) ml (feasible, and uses more honey).

Try \(x=22\) or lower to see if more jars are possible:

For general check, we can directly optimize:

We want to maximize \(x + y\), subject to:
\[
250x + 300y \le 20000,\quad y \ge 2x,\quad x \ge 20
\]

Use \(y = 2x + k\) with \(k \ge 0\):

Then
\[
x + y = x + (2x + k) = 3x + k
\]
Honey constraint:
\[
250x + 300(2x + k) \le 20000 \\
250x + 600x + 300k \le 20000 \\
850x + 300k \le 20000
\]

To maximize \(3x + k\), we want to use as much honey as possible, i.e., make \(850x + 300k\) as close to 20000 as feasible.

Check a few integer \(x \ge 20\):

- \(x=20\):  
  \(850*20 + 300k \le 20000 \Rightarrow 17000 + 300k \le 20000 \Rightarrow 300k \le 3000 \Rightarrow k \le 10\).  
  So \(y = 2*20 + k \le 40 + 10 = 50\).  
  Best \(k=10\): \(y=50\), total jars \(=20+50=70\), honey used \(=250*20 + 300*50 = 5000 + 15000 = 20000\) (perfectly uses all honey).

- \(x=21\):  
  \(850*21 + 300k \le 20000 \Rightarrow 17850 + 300k \le 20000 \Rightarrow 300k \le 2150 \Rightarrow k \le 7\).  
  Best \(k=7\): \(y=2*21+7=49\), total jars \(=21+49=70\), honey \(=250*21+300*49=5250+14700=19950\).

- \(x=22\):  
  \(850*22 + 300k \le 20000 \Rightarrow 18700 + 300k \le 20000 \Rightarrow 300k \le 1300 \Rightarrow k \le 4\).  
  Best \(k=4\): \(y=2*22+4=48\), total jars \(=22+48=70\), honey \(=250*22+300*48=5500+14400=19900\).

- \(x=23\): we already saw best is \(y=47\), total 70, honey 19850.

No combination allows more than 70 jars, and we have found feasible solutions achieving 70.

Among those, a particularly nice one is the one that uses exactly all honey:  
\[
x = 20,\quad y = 50.
\]

**Answer (optimal jar counts)**  
- Glass jars: **20**  
- Plastic jars: **50**  
Total jars filled: **70**, using all 20,000 ml of honey.

**Sketch with PuLP:**

```python
import pulp as pl

# Problem
prob = pl.LpProblem("Honey_Jars", pl.LpMaximize)

# Variables
x = pl.LpVariable('glass_jars', lowBound=20, cat='Integer')
y = pl.LpVariable('plastic_jars', lowBound=0, cat='Integer')

# Objective
prob += x + y

# Constraints
prob += 250*x + 300*y <= 20000
prob += y >= 2*x

# Solve
prob.solve()

print("Status:", pl.LpStatus[prob.status])
print("Glass jars:", pl.value(x))
print("Plastic jars:", pl.value(y))
print("Total jars:", pl.value(x + y))
```

[PROGRAMMER-VALIDATOR]:
import pulp as pl

def solve_honey_jars():
    # Define the problem
    prob = pl.LpProblem("Honey_Jars", pl.LpMaximize)

    # Decision variables
    x = pl.LpVariable('glass_jars', lowBound=0, cat='Integer')
    y = pl.LpVariable('plastic_jars', lowBound=0, cat='Integer')

    # Objective: maximize total jars
    prob += x + y

    # Constraints
    prob += 250 * x + 300 * y <= 20000           # Honey availability
    prob += y >= 2 * x                           # At least twice as many plastic jars
    prob += x >= 20                              # At least 20 glass jars

    # Solve
    prob.solve(pl.PULP_CBC_CMD(msg=False))

    x_val = pl.value(x)
    y_val = pl.value(y)
    total_jars = pl.value(x + y)

    # Optionally print internal results (not requested but useful)
    # print("Glass jars:", x_val)
    # print("Plastic jars:", y_val)
    # print("Total jars:", total_jars)

    return total_jars

if __name__ == "__main__":
