[MODELER]:
Here is a clean mathematical formulation of the problem, followed by recommendations for solving it in Python.

---

## 1. Decision Variables

Let

- \( x \): number of mangos to sell (and therefore buy)
- \( y \): number of guavas to sell (and therefore buy)

Both are nonnegative integers.

\[
x \ge 0,\quad y \ge 0,\quad x,y \in \mathbb{Z}
\]

---

## 2. Objective Function

Profit per unit:

- Each mango: profit = \$3  
- Each guava: profit = \$4  

Total profit:

\[
\text{Maximize } Z = 3x + 4y
\]

---

## 3. Constraints

### 3.1 Budget Constraint

The owner can spend at most \$20,000 on purchasing fruit:

- Mango costs \$5 each
- Guava costs \$3 each

\[
5x + 3y \le 20000
\]

### 3.2 Mango Sales Bounds

The owner estimates:

- At least 100 mangos sold
- At most 150 mangos sold

\[
100 \le x \le 150
\]

### 3.3 Guava Sales Relative to Mangos

The number of guavas sold is at most one-third of the number of mangos sold:

\[
y \le \frac{1}{3}x
\]

To keep this as a linear inequality (and if integrality is enforced, it's naturally integer), we can write:

\[
3y \le x
\]

### 3.4 Non-negativity

\[
x \ge 0,\quad y \ge 0
\]

(Already implied by other constraints, but typically stated explicitly.)

---

## 4. Complete Linear Programming Model

**Maximize**
\[
Z = 3x + 4y
\]

**subject to**
\[
5x + 3y \le 20000
\]
\[
100 \le x \le 150
\]
\[
3y \le x
\]
\[
x,y \ge 0,\; x,y \in \mathbb{Z}
\]

If you treat \(x,y\) as continuous (not integer), it is a standard Linear Program (LP). Since the numbers are small, solving it as an integer program (ILP) or LP will give essentially the same solution here.

---

## 5. Solving in Python

This is a small linear/integer optimization problem. Suitable libraries:

- **PuLP** (`pulp`): easy to use for teaching and small LP/ILP models.
- **OR-Tools** (`ortools.linear_solver`): very efficient and industrial-grade.
- **SciPy** (`scipy.optimize.linprog`): suitable for continuous LP (no integer requirements).

### Example with PuLP (integer version)

```python
import pulp as pl

# Define problem
prob = pl.LpProblem("Fruit_Truck_Profit_Maximization", pl.LpMaximize)

# Decision variables (integer)
x = pl.LpVariable('mangos', lowBound=100, upBound=150, cat='Integer')
y = pl.LpVariable('guavas', lowBound=0, cat='Integer')

# Objective function
prob += 3 * x + 4 * y, "Total_Profit"

# Constraints
prob += 5 * x + 3 * y <= 20000, "Budget"
prob += 3 * y <= x, "Guava_to_Mango_Ratio"

# Solve
prob.solve()

print("Status:", pl.LpStatus[prob.status])
print("Optimal mangos:", x.varValue)
print("Optimal guavas:", y.varValue)
print("Maximum profit:", pl.value(prob.objective))
```

### Example with OR-Tools (integer version)

```python
from ortools.linear_solver import pywraplp

solver = pywraplp.Solver.CreateSolver('CBC')

x = solver.IntVar(100, 150, 'mangos')
y = solver.IntVar(0, solver.infinity(), 'guavas')

# Budget
solver.Add(5 * x + 3 * y <= 20000)
# Guava to mango ratio
solver.Add(3 * y <= x)

# Objective
solver.Maximize(3 * x + 4 * y)

status = solver.Solve()
if status == pywraplp.Solver.OPTIMAL:
    print("Optimal mangos:", x.solution_value())
    print("Optimal guavas:", y.solution_value())
    print("Maximum profit:", solver.Objective().Value())
```

[PROGRAMMER-VALIDATOR]:
import pulp as pl

def solve_fruit_truck_problem():
    # Define the problem
    prob = pl.LpProblem("Fruit_Truck_Profit_Maximization", pl.LpMaximize)

    # Decision variables (integer)
    x = pl.LpVariable('mangos', lowBound=100, upBound=150, cat='Integer')
    y = pl.LpVariable('guavas', lowBound=0, cat='Integer')

    # Objective function: Maximize profit = 3x + 4y
    prob += 3 * x + 4 * y, "Total_Profit"

    # Constraints
    prob += 5 * x + 3 * y <= 20000, "Budget"
    prob += 3 * y <= x, "Guava_to_Mango_Ratio"

    # Solve the problem
    prob.solve(pl.PULP_CBC_CMD(msg=False))

    # Get the objective value
    objective_value = pl.value(prob.objective)
    return objective_value

if __name__ == "__main__":
    final_obj = solve_fruit_truck_problem()
    print(f"FINAL_OBJ: {final_obj}")
