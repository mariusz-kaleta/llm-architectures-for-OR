[MODELER]:
Let’s turn this into a clear optimization model and then solve it.

### Decision variables
Let  
- \(x\) = number of wired headphones produced per day  
- \(y\) = number of wireless headphones produced per day  

Both must be nonnegative:
- \(x \ge 0\)  
- \(y \ge 0\)

### Objective (maximize profit)
Profit per unit:
- Wired: \$50 per unit  
- Wireless: \$20 per unit  

Total daily profit:
\[
\text{Maximize } Z = 50x + 20y
\]

### Constraints

1. **Wired team capacity**  
   The wired team can make at most 100 wired headphones per day:
   \[
   x \le 100
   \]

2. **Wireless team capacity**  
   The wireless team can make at most 170 wireless headphones per day:
   \[
   y \le 170
   \]

3. **Shared testing machine capacity**  
   The test machine can test at most 150 headphones per day, regardless of type:
   \[
   x + y \le 150
   \]

4. **Nonnegativity**  
   \[
   x \ge 0,\quad y \ge 0
   \]

This is a linear programming (LP) problem.

### Solving analytically

Because the profit per wired headphone (\$50) is higher than for wireless (\$20), and both use the testing machine equally (1 test per headphone), it is best to prioritize wired headphones while respecting all constraints.

From the constraints:

- Wired capacity: \(x \le 100\)
- Test capacity: \(x + y \le 150\)

To maximize profit, set \(x\) as large as possible:
- So take \(x = 100\) (wires fully utilize their own team).

Then, from the test machine constraint:
\[
x + y \le 150 \Rightarrow 100 + y \le 150 \Rightarrow y \le 50
\]

We can set \(y = 50\) (within wireless capacity \(y \le 170\)).

Check profit:
\[
Z = 50(100) + 20(50) = 5000 + 1000 = 6000
\]

Other obvious corners give lower profit, e.g.:
- \(x = 0, y = 150 \Rightarrow Z = 20(150) = 3000\)
- \(x = 100, y = 0 \Rightarrow Z = 5000\)

So the optimal plan is:

- **Produce 100 wired headphones per day**  
- **Produce 50 wireless headphones per day**  
- **Maximum profit = \$6000 per day**

### Suitable Python libraries to model and solve it

You can formulate and solve this LP using any of these:

- **PuLP** (`pulp`) – very simple for educational LPs.
- **SciPy** (`scipy.optimize.linprog`) – general-purpose linear programming.
- **OR-Tools** (`ortools.linear_solver`) – efficient and industrial-strength.

#### Example in PuLP

```python
import pulp as pl

# Define problem
prob = pl.LpProblem("Headphone_Production", pl.LpMaximize)

# Decision variables
x = pl.LpVariable("wired", lowBound=0)
y = pl.LpVariable("wireless", lowBound=0)

# Objective
prob += 50 * x + 20 * y

# Constraints
prob += x <= 100          # Wired capacity
prob += y <= 170          # Wireless capacity
prob += x + y <= 150      # Testing machine capacity

# Solve
prob.solve(pl.PULP_CBC_CMD(msg=False))

print("Optimal wired:", x.value())
print("Optimal wireless:", y.value())
print("Max profit:", pl.value(prob.objective))
```

This will output approximately:
- Optimal wired: 100
- Optimal wireless: 50
- Max profit: 6000

import pulp as pl

prob = pl.LpProblem("Headphone_Production", pl.LpMaximize)

y = pl.LpVariable("wireless", lowBound=0)
