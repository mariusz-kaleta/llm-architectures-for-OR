## 1. Decision variables

Let

- \(x\) = number of large bags  
- \(y\) = number of tiny bags

## 2. Objective: maximize total grain transported

Each large bag holds 25 kg, each tiny bag holds 6 kg.  

Total weight of grain:

\[
\text{Total weight} = 25x + 6y.
\]

**Objective function**:
\[
\max \; Z = 25x + 6y

## 3. Constraints

### (a) Energy constraint

Each large bag uses 4 units of energy; each tiny bag uses 1.5 units. Total available energy is 110 units:

\[
4x + 1.5y \le 110.
\]

### (b) Popularity (twice as many large bags as tiny bags)

> “there must be twice as many large bags as tiny bags”  

This is a strict proportionality relation:

\[
x = 2y.
\]

### (c) Minimum number of tiny bags

> “there must be at least 20 tiny bags”  

\[
y \ge 20.
\]

### (d) Integrality and nonnegativity

\[
x, y \in \mathbb{Z}, \quad x \ge 0,\; y \ge 0.
\]

## 4. Simplified model

Using the relation \(x = 2y\) in the other constraints:

- Plug into energy:
  \[
  4(2y) + 1.5y \le 110
  \;\Rightarrow\;
  8y + 1.5y \le 110
  \;\Rightarrow\;
  9.5y \le 110
  \;\Rightarrow\;
  y \le \frac{110}{9.5} \approx 11.58.
  \]

- But we also must have \(y \ge 20\).

Thus we need:
\[
y \ge 20 \quad \text{and} \quad y \le 11.58,
\]
which is impossible.

So the constraint set is **infeasible**; there is **no** combination of \(x, y\) that satisfies all of:

- energy ≤ 110,
- \(x = 2y\),
- \(y \ge 20\).

Hence, strictly speaking, the optimization problem has **no feasible solution**, and therefore no optimal solution.

import itertools

def is_feasible(x, y):
    # Constraints:
    # 1) Energy: 4x + 1.5y <= 110
    # 2) Proportionality: x = 2y
    # 3) Minimum tiny bags: y >= 20
    # 4) Nonnegativity and integrality ensured by iteration
    if x != 2 * y:
        return False
    if y < 20:
        return False
    if 4 * x + 1.5 * y > 110:
        return False
    return True

best_value = None
best_solution = None

# We can bound search: from constraints:
# x = 2y, y >= 20, but also must satisfy energy, so let's just search a safe range.
# Since energy is small, y won't be huge; we'll search a modest range.
for y in range(0, 200):
    x = 2 * y
    if is_feasible(x, y):
        value = 25 * x + 6 * y
        if best_value is None or value > best_value:
            best_value = value
            best_solution = (x, y)

# If there is no feasible solution, define objective as None or a sentinel.
# Here we encode infeasibility by using None and printing a descriptive value.
if best_value is None:
    # No feasible solution
    FINAL_OBJ = float('nan')
else:
    FINAL_OBJ = best_value

print(f"FINAL_OBJ: {FINAL_OBJ}")
