[MODELER]:
Here is a clean optimization model for this problem.

---

## 1. Decision variables

Let:

- \( x \in \mathbb{Z}_{\ge 0} \): number of camel caravan trips  
- \( y \in \mathbb{Z}_{\ge 0} \): number of desert truck trips  

---

## 2. Objective

We want to **minimize the total number of hours** spent transporting the goods:

- Each camel caravan trip takes 12 hours  
- Each truck trip takes 5 hours  

So the objective function is:

\[
\min\; Z = 12x + 5y
\]

---

## 3. Constraints

### 3.1 Demand satisfaction (at least 1500 units)

- Each camel caravan trip delivers 50 units  
- Each truck trip delivers 150 units  
- Total delivery must be at least 1500 units:

\[
50x + 150y \;\ge\; 1500
\]

You can simplify by dividing by 50:

\[
x + 3y \;\ge\; 30
\]

### 3.2 Preference for more camels than trucks

The company prefers **more camel caravans than desert trucks**:

\[
x \;\ge\; y
\]

(You could also model “strictly more” as \(x \ge y + 1\) if that interpretation is needed.)

### 3.3 Integrality and nonnegativity

\[
x, y \in \mathbb{Z}_{\ge 0}
\]

---

## 4. Full mathematical model

\[
\begin{aligned}
\min\quad & 12x + 5y \\
\text{s.t.}\quad 
& x + 3y \;\ge\; 30 \\
& x - y \;\ge\; 0 \\
& x, y \in \mathbb{Z}_{\ge 0}
\end{aligned}
\]

---

## 5. Solving the model (sketch)

Because this is a small integer linear program (ILP), it can be solved exactly by enumeration or by an ILP solver.

Manually:

- From \(x + 3y \ge 30\) and \(x \ge y\), try small values of \(y\):

For \(y = 7\):  
- Demand: \(x + 3 \cdot 7 = x + 21 \ge 30 \Rightarrow x \ge 9\)  
- Preference: \(x \ge y = 7\) → satisfied by \(x = 9\).  
- Objective: \(Z = 12(9) + 5(7) = 108 + 35 = 143\).

Check neighboring feasible solutions to ensure minimality:

- \(y = 6\): then \(x + 18 \ge 30 \Rightarrow x \ge 12\); \(Z = 12(12)+5(6)=174\) (worse).
- \(y = 8\): then \(x + 24 \ge 30 \Rightarrow x \ge 6\) but need \(x \ge y = 8\); pick \(x=8\); \(Z = 12(8)+5(8)=136\).  
  However demand: \(50(8)+150(8)=400+1200=1600 \ge 1500\) → feasible, and \(Z=136\) is better.
- \(y = 9\): \(x+27 \ge 30 \Rightarrow x\ge 3\), but \(x \ge 9\); \(Z=12(9)+5(9)=153\) (worse).

Try \(y = 8, x = 8\) more carefully:  
- Demand: \(x + 3y = 8 + 24 = 32 \ge 30\) ✔  
- Preference: \(x \ge y\) → \(8 \ge 8\) ✔  
- Objective: \(Z = 12·8 + 5·8 = 96 + 40 = 136\).

No combination with smaller \(Z\) (e.g., fewer trucks with more camels) will beat 136 given the constraint that camels are slower and must be at least as many as trucks.

So an optimal solution is:
- \(x^* = 8\) camel caravan trips  
- \(y^* = 8\) truck trips  
- Total hours: \(12·8 + 5·8 = 136\) hours

## 6. Suitable Python libraries

You can implement and solve this ILP with:

- **PuLP** (`pulp`): Simple modeling syntax for linear and integer programs.
- **OR-Tools** (`ortools.linear_solver`): Google’s optimization suite with MILP solvers.
- **Pyomo**: More general algebraic modeling; good for larger projects.

For this small linear integer model, **PuLP** or **OR-Tools** are ideal.

Example outline in PuLP:

```python
import pulp as pl

# Problem
prob = pl.LpProblem("DesertTransport", pl.LpMinimize)

# Variables
x = pl.LpVariable("camels", lowBound=0, cat="Integer")
y = pl.LpVariable("trucks", lowBound=0, cat="Integer")

# Objective
prob += 12*x + 5*y

# Constraints
prob += x + 3*y >= 30      # demand
prob += x >= y             # more camels than trucks

# Solve
prob.solve()

print("Status:", pl.LpStatus[prob.status])
print("Camels:", x.value())
print("Trucks:", y.value())
print("Total hours:", 12*x.value() + 5*y.value())
```

This directly encodes the mathematical model above.

[PROGRAMMER-VALIDATOR]:
import pulp as pl

def main():
    # Create the problem
    prob = pl.LpProblem("DesertTransport", pl.LpMinimize)

    # Decision variables
    x = pl.LpVariable("camels", lowBound=0, cat="Integer")
    y = pl.LpVariable("trucks", lowBound=0, cat="Integer")

    # Objective function: minimize total hours
    prob += 12 * x + 5 * y

    # Constraints
    prob += x + 3 * y >= 30    # Demand constraint
    prob += x >= y             # Preference constraint: at least as many camels as trucks

    # Solve the problem
    prob.solve()

    # Obtain the optimal objective value
    final_obj = pl.value(prob.objective)

    # Print final objective in required format
    print(f"FINAL_OBJ: {final_obj}")

if __name__ == "__main__":
    main()
