   - The ratio constraint T ≥ 4N strongly pushes the optimal solution to have many more throat swabs than nasal swabs; check whether this binds at optimum (likely T = 4N at optimum).  
   - Combine the time constraint with T = 4N to reduce to a single-variable problem (in N), making it easy to find the maximum feasible N (and therefore corresponding T).

## 1. Decision variables

Let

- \(T \in \mathbb{Z}_{\ge 0}\): number of throat swabs  
- \(N \in \mathbb{Z}_{\ge 0}\): number of nasal swabs  

(Each patient receives exactly one swab, so total patients = \(T + N\).)

---

## 2. Objective

Maximize the total number of patients seen:

\[
\max \; Z = T + N
\]

---

## 3. Constraints

1. **Time capacity (20,000 minutes total)**  
   - Throat swab: 5 minutes each  
   - Nasal swab: 3 minutes each  

   \[
   5T + 3N \le 20000
   \]

2. **Minimum nasal swabs**

   \[
   N \ge 30
   \]

3. **Discomfort-based ratio (at least 4 times as many throat as nasal)**

   \[
   T \ge 4N
   \]

4. **Non-negativity and integrality**

   \[
   T \ge 0,\; N \ge 0,\; T,N \in \mathbb{Z}
   \]

---

## 4. Structure and reduction

Because the objective is to *increase* \(T\) and \(N\) and time is limited, at optimum the time constraint will be tight (fully used) and the ratio will typically be tight too (otherwise we could increase \(N\) and \(T\) and see more patients).

Assume the ratio constraint binds: \(T = 4N\). Substitute into the time constraint:

\[
5(4N) + 3N \le 20000
\]
\[
20N + 3N \le 20000
\]
\[
23N \le 20000
\]
\[
N \le \frac{20000}{23} \approx 869.565
\]

Since \(N\) must be integer, \(N \le 869\).

We also require \(N \ge 30\), and 869 satisfies this, so the maximum feasible \(N\) is:

\[
N^\* = 869
\]

Then

\[
T^\* = 4N^\* = 4 \times 869 = 3476
\]

Check time usage:

\[
5T^\* + 3N^\* = 5(3476) + 3(869) = 17380 + 2607 = 19987 \le 20000
\]

This is feasible. The time is not exactly full, but if we try to increase \(N\) to 870:

- From the ratio constraint: \(T \ge 4 \times 870 = 3480\)
- Time used: \(5(3480) + 3(870) = 17400 + 2610 = 20010 > 20000\), which is infeasible.

So \((T^\*,N^\*) = (3476, 869)\) is optimal.

Total patients:

\[
T^\* + N^\* = 3476 + 869 = 4345
\]

---

## 5. Final answer

- Throat swabs: **3476**  
- Nasal swabs: **869**  
- Maximum patients seen: **4345**

## 6. Suitable Python libraries

To implement and solve this ILP in Python, you can use:

- **PuLP** (`pulp`) – very simple modeling interface, uses CBC by default.
- **OR-Tools Linear Solver** (`ortools.linear_solver.pywraplp`) – efficient and well-maintained.
- **Pyomo** – more general algebraic modeling, good for larger/complex models.

Example using PuLP:

```python
import pulp as pl

# Model
model = pl.LpProblem("Swab_Optimization", pl.LpMaximize)

# Variables
T = pl.LpVariable("T", lowBound=0, cat="Integer")
N = pl.LpVariable("N", lowBound=0, cat="Integer")

# Objective
model += T + N

# Constraints
model += 5*T + 3*N <= 20000          # time
model += N >= 30                     # min nasal
model += T >= 4*N                    # ratio

# Solve
model.solve(pl.PULP_CBC_CMD(msg=False))

print("Status:", pl.LpStatus[model.status])
print("T (throat) =", pl.value(T))
print("N (nasal)  =", pl.value(N))
print("Total patients =", pl.value(T + N))
```

This will return \(T = 3476\), \(N = 869\).

import pulp as pl

def main():
    # Create the model
    model = pl.LpProblem("Swab_Optimization", pl.LpMaximize)

    # Decision variables
    T = pl.LpVariable("T", lowBound=0, cat="Integer")  # Throat swabs
    N = pl.LpVariable("N", lowBound=0, cat="Integer")  # Nasal swabs

    # Objective: maximize total patients
    model += T + N

    # Constraints
    model += 5 * T + 3 * N <= 20000    # Time capacity
    model += N >= 30                  # Minimum nasal swabs
    model += T >= 4 * N               # Ratio constraint

    # Solve
    solver = pl.PULP_CBC_CMD(msg=False)
    model.solve(solver)

    # Extract solution
    T_val = pl.value(T)
    N_val = pl.value(N)

if __name__ == "__main__":
