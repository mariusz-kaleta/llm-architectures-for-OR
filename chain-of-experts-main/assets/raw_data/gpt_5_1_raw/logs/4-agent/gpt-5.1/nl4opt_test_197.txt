## 1. Decision variables

Let

- \(x_1\) = number of first-dose vaccines to produce  
- \(x_2\) = number of second-dose vaccines to produce  

Because vaccines are counted in units, it is natural to require:

- \(x_1, x_2 \in \mathbb{Z}_{\ge 0}\) (non-negative integers)

You can also relax them to continuous variables if an approximate fractional solution is acceptable.

---

## 2. Objective function

We want to minimise total gelatine used.

- First-dose uses 20 mg per vaccine  
- Second-dose uses 60 mg per vaccine  

Total gelatine (mg) used:

\[
\text{Gelatine} = 20 x_1 + 60 x_2
\]

**Objective:**

\[
\min \; Z = 20 x_1 + 60 x_2
\]

---

## 3. Constraints

### 3.1 Antibiotics availability

Total antibiotics used cannot exceed 35,000 units.

- First-dose uses 30 units  
- Second-dose uses 65 units  

\[
30 x_1 + 65 x_2 \le 35000
\]

---

### 3.2 Dose sequencing / logical requirement

There must be **more first-dose than second-dose vaccines** manufactured:

\[
x_1 > x_2
\]

In an optimisation model, strict inequalities are not allowed directly. Two standard formulations:

- If \(x_1, x_2\) are integers:

  \[
  x_1 \ge x_2 + 1
  \]

- If \(x_1, x_2\) are continuous:

  \[
  x_1 \ge x_2 + \delta
  \]
  with a small positive \(\delta\) (e.g. 0.001); but integer modelling is more precise here.

Given vaccine counts, using integers and:

\[
x_1 \ge x_2 + 1
\]

is recommended.

---

### 3.3 Minimum second-dose requirement

At least 40 second-dose vaccines must be produced:

\[
x_2 \ge 40
\]

---

### 3.4 Non-negativity and integrality

\[
x_1 \ge 0, \quad x_2 \ge 0
\]

and (if enforced):

\[
x_1, x_2 \in \mathbb{Z}
\]

---

## 4. Complete mathematical model

\[
\begin{aligned}
\min \quad & Z = 20 x_1 + 60 x_2 \\[4pt]
\text{s.t.} \quad 
& 30 x_1 + 65 x_2 \le 35000 && \text{(antibiotics limit)} \\[4pt]
& x_1 \ge x_2 + 1          && \text{(more first-dose than second-dose)} \\[4pt]
& x_2 \ge 40               && \text{(minimum second-dose)} \\[4pt]
& x_1, x_2 \ge 0           && \text{(non-negativity)} \\[4pt]
& x_1, x_2 \in \mathbb{Z}  && \text{(integrality, if required)}
\end{aligned}
\]

## 5. Suitable Python libraries

This is a small linear (integer) program with 2 variables and a few constraints. Any standard LP/MIP solver is suitable:

1. **PuLP**
   - High-level LP/MIP modelling in pure Python.
   - Can use CBC (bundled), GLPK, CPLEX, Gurobi, etc.
   - Very easy for educational and small problems.
   - Recommended: `pulp.LpProblem`, `pulp.LpVariable`, `LpMinimize`.

2. **OR-Tools (Google)**
   - Excellent for linear and integer optimisation.

   - Good if you might extend to more complex combinatorial problems.

3. **Pyomo**
   - More general algebraic modelling language in Python.
   - Good if you plan to scale to larger/more complex models.

For this specific problem, **PuLP** or **OR-Tools** is more than sufficient and very convenient.

import pulp

def main():
    # Create the problem: minimize total gelatine
    prob = pulp.LpProblem("Vaccine_Production", pulp.LpMinimize)

    # Decision variables (integers, non-negative)
    x1 = pulp.LpVariable('x1', lowBound=0, cat='Integer')  # first-dose vaccines
    x2 = pulp.LpVariable('x2', lowBound=0, cat='Integer')  # second-dose vaccines

    # Objective function: minimize gelatine usage
    prob += 20 * x1 + 60 * x2

    # Constraints
    # Antibiotics limit
    prob += 30 * x1 + 65 * x2 <= 35000

    # More first-dose than second-dose: x1 >= x2 + 1
    prob += x1 >= x2 + 1

    # Minimum second-dose requirement
    prob += x2 >= 40

    # Solve the problem using the default solver (CBC bundled with PuLP)
    prob.solve(pulp.PULP_CBC_CMD(msg=False))

    # Get the final objective value
    final_obj = pulp.value(prob.objective)

    print(f"FINAL_OBJ: {final_obj}")

if __name__ == "__main__":
    main()
