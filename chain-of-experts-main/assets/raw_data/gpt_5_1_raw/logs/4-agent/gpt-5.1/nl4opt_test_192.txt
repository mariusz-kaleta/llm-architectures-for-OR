   - Interpretation of “average chance of losing”: Treat as a weighted average of loss probabilities over the money allocated to each bet.

2. Suggested model type  
   - Model: Linear Programming (LP).  
   - Rationale:  
     - Objective is a linear function of the allocation amounts and fixed per-dollar payouts.  
     - The “average chance of losing” can be expressed as a linear constraint using weighted probabilities.  
     - Budget and non-negativity are linear.

3. Main decision variables  
   Define a decision variable for the amount of money allocated to each bet:

   - \(x_B\): dollars bet on basketball tournament  
   - \(x_H\): dollars bet on horse race  
   - \(x_S\): dollars bet on soccer game  

   The total initial capital is 100,000, and these are the only bets.

   - Per-dollar payout multipliers (as given):  
     - Basketball: 1.2 per dollar  
     - Horse race: 0.5 per dollar  
     - Soccer: 0.1 per dollar  
   - Treat these as the “payout contribution” per dollar and aim to maximize the total payout:  
     - Objective: Maximize  
       - \(1.2 \cdot x_B + 0.5 \cdot x_H + 0.1 \cdot x_S\)

   If desired, you could also model expected return explicitly by including win/lose probabilities, but the prompt’s wording aligns with simply maximizing the weighted sum of these given payout rates under the risk constraint.

5. Key constraints to consider  

   a) Budget (capital) constraint  
   - The total amount invested cannot exceed her available bankroll:  
     - \(x_B + x_H + x_S \le 100000\)  
   - Optionally, you can enforce that all money must be used:  
     - \(x_B + x_H + x_S = 100000\)  
     This depends on whether “keeping some money safe” is allowed or not. If you interpret “has $100000 to gamble” as “can allocate up to 100000,” use ≤; if it means “must allocate,” use =.

   b) Average chance of losing constraint  
   - Given loss probabilities:  
     - Basketball loss probability: 0.50  
     - Horse race loss probability: 0.25  
     - Soccer loss probability: 0.10  
   - She wants the (allocation-weighted) average loss probability to be at most 0.30.  
   - For the case where all money is invested (recommended interpretation):  
     - Weighted average loss probability =  
       \((0.50 \cdot x_B + 0.25 \cdot x_H + 0.10 \cdot x_S) / (x_B + x_H + x_S)\)  
       ≤ 0.30  
     - With \(x_B + x_H + x_S = 100000\), this simplifies to a linear constraint:  
       - \(0.50 \cdot x_B + 0.25 \cdot x_H + 0.10 \cdot x_S \le 0.30 \cdot 100000\)

   - If you allow uninvested money (budget ≤ 100000), you have two main options:
     1. Define an additional variable for uninvested funds and exclude it from the average-loss denominator (average is over gambled funds only). Then use the same structure as above, but denominator becomes \(x_B + x_H + x_S\) and needs a linearization (e.g., by enforcing full investment or by fixing total bet amount).
     2. Enforce full investment to keep a simple LP (preferred, as the prompt doesn’t emphasize holding cash).

   c) Non-negativity constraints  
   - Each allocation must be non-negative:  
     - \(x_B \ge 0\)  
     - \(x_H \ge 0\)  
     - \(x_S \ge 0\)

6. Optional / refinement considerations

   - Sensitivity analysis: Once solved, check shadow prices for the risk constraint to see how much additional expected payout could be gained by relaxing the “at most 30%” loss requirement.

## 1. Decision variables

Let

- \(x_B \ge 0\): dollars bet on the basketball tournament  
- \(x_H \ge 0\): dollars bet on the horse race  
- \(x_S \ge 0\): dollars bet on the soccer game  

We will assume she **must invest all** \$100,000 (this makes the “average chance” constraint linear and is the natural interpretation in most textbook problems).

---

## 2. Objective: maximize total payout

Payout per \$1:

- Basketball: 1.2  
- Horse race: 0.5  
- Soccer: 0.1  

Total payout:
\[
\text{Payout} = 1.2 x_B + 0.5 x_H + 0.1 x_S
\]

**Objective:**

\[
\max\; Z = 1.2 x_B + 0.5 x_H + 0.1 x_S
\]

---

## 3. Constraints

### 3.1 Budget (capital) constraint

She has \$100,000 to allocate, and we assume all must be bet:

\[
x_B + x_H + x_S = 100000
\]

(If you want to allow uninvested money, replace “=” with “≤”, but then the average-loss constraint becomes non-linear unless you add an “unbet” variable and change the definition of the average.)

### 3.2 Average chance of losing ≤ 30%

Loss probabilities:

- Basketball: 0.50  
- Horse race: 0.25  
- Soccer: 0.10  

Define the **weighted average loss probability** over the invested money:

\[
\text{AvgLoss} = 
\frac{0.50 x_B + 0.25 x_H + 0.10 x_S}{x_B + x_H + x_S}
\]

Requirement:

\[
\text{AvgLoss} \le 0.30
\]

Because we enforce \(x_B + x_H + x_S = 100000\), this simplifies to a linear inequality:

\[
\frac{0.50 x_B + 0.25 x_H + 0.10 x_S}{100000} \le 0.30
\]

Multiply both sides by 100000:

\[
0.50 x_B + 0.25 x_H + 0.10 x_S \le 0.30 \cdot 100000
\]

\[
0.50 x_B + 0.25 x_H + 0.10 x_S \le 30000
\]

### 3.3 Non-negativity

\[
x_B \ge 0,\quad x_H \ge 0,\quad x_S \ge 0
\]

---

## 4. Final LP model (compact form)

\[
\begin{aligned}
\max\quad & 1.2 x_B + 0.5 x_H + 0.1 x_S \\
\text{s.t.}\quad
& x_B + x_H + x_S = 100000, \\
& 0.50 x_B + 0.25 x_H + 0.10 x_S \le 30000, \\
& x_B, x_H, x_S \ge 0.
\end{aligned}
\]

This is a standard linear program.

## 6. Example implementation with PuLP

```python
import pulp as pl

# Problem
prob = pl.LpProblem("Gambling_Allocation", pl.LpMaximize)

# Decision variables (continuous, >= 0)
x_B = pl.LpVariable('x_B', lowBound=0)
x_H = pl.LpVariable('x_H', lowBound=0)
x_S = pl.LpVariable('x_S', lowBound=0)

# Objective: maximize total payout
prob += 1.2 * x_B + 0.5 * x_H + 0.1 * x_S

# Budget constraint
prob += x_B + x_H + x_S == 100000, "Budget"

# Average loss <= 30% → linearized
prob += 0.50 * x_B + 0.25 * x_H + 0.10 * x_S <= 30000, "AvgLoss"

# Solve
prob.solve(pl.PULP_CBC_CMD(msg=False))

print("Status:", pl.LpStatus[prob.status])
print("Basketball bet (x_B):", x_B.value())
print("Horse race bet (x_H):", x_H.value())
print("Soccer bet (x_S):", x_S.value())
print("Total payout:", pl.value(prob.objective))
```

Any other LP modeling library (cvxpy, OR-Tools) would be set up with the same objective and constraints.

import pulp as pl

def main():
    # Create the linear programming problem (maximize)
    prob = pl.LpProblem("Gambling_Allocation", pl.LpMaximize)

    # Decision variables (non-negative continuous)
    x_B = pl.LpVariable('x_B', lowBound=0)  # Basketball
    x_H = pl.LpVariable('x_H', lowBound=0)  # Horse race
    x_S = pl.LpVariable('x_S', lowBound=0)  # Soccer

    # Objective: maximize total payout
    prob += 1.2 * x_B + 0.5 * x_H + 0.1 * x_S

    # Budget constraint: must invest all 100000
    prob += x_B + x_H + x_S == 100000, "Budget"

    # Average loss <= 30% → linearized constraint
    prob += 0.50 * x_B + 0.25 * x_H + 0.10 * x_S <= 30000, "AvgLoss"

    # Solve the problem
    prob.solve(pl.PULP_CBC_CMD(msg=False))

    # Get the final objective value
    final_obj = pl.value(prob.objective)

    # Print the final objective in the required format
    print(f"FINAL_OBJ: {final_obj}")

if __name__ == "__main__":
    main()
