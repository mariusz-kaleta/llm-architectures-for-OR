[PLANNER]:
1. Key optimisation goal  
Goal: Minimise total pollution generated by transporting employees to work.

2. Suggested model type  
Model: Integer Linear Programming (ILP), since the numbers of cars and buses must be integers.

3. Main decision variables  
Define integer decision variables such as:
- Number of buses used.
- Number of cars used.

(Optionally, a variable for number of employees transported could be included for clarity, but it can also be expressed directly in terms of cars and buses.)

4. Key parameters  
- Capacity per car: 4 employees.
- Capacity per bus: 20 employees.
- Pollution per car: 10 units.
- Pollution per bus: 30 units.
- Minimum employees to transport: 300.
- Maximum number of buses: 4.

5. Objective function (conceptual)  
- Minimise: (pollution per car × number of cars) + (pollution per bus × number of buses).

6. Key constraints to consider  

a) Demand (coverage) constraint  
- Total capacity of selected vehicles must be at least the number of employees needing transport:
  - Capacity per car × number of cars  
  + Capacity per bus × number of buses  
  ≥ 300 employees.

b) Fleet/bus availability constraint  
- Number of buses cannot exceed 4.

c) Non-negativity and integrality  
- Number of cars ≥ 0, integer.  
- Number of buses ≥ 0, integer.

7. Modelling strategy / structure for the Modeller  

- Use a two-variable ILP (cars, buses). The problem size is tiny, so the solution can also be checked by enumeration, but the formulation must be general and clean.
- Check whether the capacity constraint is binding (likely around 300 employees or slightly more) and whether the bus limit constraint is binding (up to 4 buses).
- Consider the pollution “per seat” ratio to understand structure:
  - Car: 10 units / 4 seats = 2.5 units per employee.
  - Bus: 30 units / 20 seats = 1.5 units per employee.
  This indicates buses are more pollution-efficient, guiding expectations: use as many buses as allowed, then fill the remaining requirement with cars.
- After solving, verify:
  - The capacity is at least 300 (no shortfall).
  - The bus count does not exceed 4.
  - The solution is integral and pollution is minimised relative to nearby integer combinations.

[MODELER]:
Here is a clean mathematical model of the problem as an Integer Linear Program (ILP), plus suggestions for solving it in Python.

---

## 1. Decision variables

Let

- \(x \in \mathbb{Z}_{\ge 0}\): number of cars used  
- \(y \in \mathbb{Z}_{\ge 0}\): number of buses used  

(Both must be non-negative integers.)

---

## 2. Parameters

- Capacity per car: \(c_{\text{car}} = 4\) employees/car  
- Capacity per bus: \(c_{\text{bus}} = 20\) employees/bus  
- Pollution per car: \(p_{\text{car}} = 10\) units/car  
- Pollution per bus: \(p_{\text{bus}} = 30\) units/bus  
- Minimum employees to transport: \(D = 300\)  
- Maximum number of buses: \(B_{\max} = 4\)

---

## 3. Objective function

Minimize total pollution:

\[
\min \; Z = p_{\text{car}} \cdot x + p_{\text{bus}} \cdot y
         = 10x + 30y
\]

---

## 4. Constraints

1. **Demand (coverage) constraint**  
   Total capacity must transport at least 300 employees:

   \[
   c_{\text{car}} \cdot x + c_{\text{bus}} \cdot y \;\ge\; D
   \]
   \[
   4x + 20y \;\ge\; 300
   \]

2. **Bus availability constraint**

   \[
   y \;\le\; B_{\max} = 4
   \]

3. **Non-negativity and integrality**

   \[
   x \;\ge\; 0,\quad y \;\ge\; 0,\quad x, y \in \mathbb{Z}
   \]

---

## 5. Complete ILP model (compact form)

\[
\begin{aligned}
\min \quad & 10x + 30y \\
\text{s.t.} \quad
& 4x + 20y \ge 300 \\
& y \le 4 \\
& x \ge 0,\; y \ge 0 \\
& x, y \in \mathbb{Z}.
\end{aligned}
\]

---

## 6. Suitable Python libraries

Any Mixed-Integer Linear Programming (MILP) capable library can solve this:

- **PuLP** (`pulp`): Very convenient, pure Python, works with CBC by default.
- **OR-Tools** (`ortools.linear_solver`): Google’s optimization suite, efficient for ILPs.
- **Pyomo**: More general modeling framework, supports many solvers.
- **SciPy** is *not* ideal here because it does not natively support integer variables.

For a small ILP like this, **PuLP** or **OR-Tools** is ideal.

---

## 7. Example PuLP formulation (minimal)

```python
import pulp as pl

# Problem
prob = pl.LpProblem("Transport_Pollution_Min", pl.LpMinimize)

# Decision variables
x = pl.LpVariable("cars", lowBound=0, cat="Integer")
y = pl.LpVariable("buses", lowBound=0, cat="Integer")

# Objective
prob += 10 * x + 30 * y

# Constraints
prob += 4 * x + 20 * y >= 300      # demand
prob += y <= 4                     # bus limit

# Solve
prob.solve()

print("Status:", pl.LpStatus[prob.status])
print("Cars:", x.value())
print("Buses:", y.value())
print("Total pollution:", pl.value(prob.objective))
```

This code directly implements the formal model above and will produce the optimal combination of cars and buses that minimizes total pollution.

[CODER]:
import pulp as pl

def main():
    # Parameters
    c_car = 4
    c_bus = 20
    p_car = 10
    p_bus = 30
    D = 300
    B_max = 4

    # Problem definition
    prob = pl.LpProblem("Transport_Pollution_Min", pl.LpMinimize)

    # Decision variables
    x = pl.LpVariable("cars", lowBound=0, cat="Integer")
    y = pl.LpVariable("buses", lowBound=0, cat="Integer")

    # Objective function
    prob += p_car * x + p_bus * y

    # Constraints
    prob += c_car * x + c_bus * y >= D        # Demand constraint
    prob += y <= B_max                        # Bus availability

    # Solve the ILP
    prob.solve(pl.PULP_CBC_CMD(msg=False))

    # Get the objective value
    final_obj = pl.value(prob.objective)

    # Print final objective in required format
    print(f"FINAL_OBJ: {final_obj}")

if __name__ == "__main__":
    main()
