1. **Key optimisation goal**

- **Goal:** Maximize the total number of meals that can be delivered given the limited charging capacity and policy constraints on vehicle mix and minimum scooters.

---

2. **Suggested model type**

- **Model:** Mixed-Integer Linear Programming (MILP)  
  - A pure LP is sufficient for the continuous relaxation, but since we are dealing with integer counts of bikes and scooters, and with a ratio constraint involving a minimum proportion, it is safest to model vehicle counts as integers → MILP.

---

3. **Main decision variables**

Define decision variables representing the number of each type of vehicle used:

- Let:
  - **B** = number of bikes used (integer, ≥ 0)
  - **S** = number of scooters used (integer, ≥ 0)

Optional derived/auxiliary expressions (not separate decision variables, but relevant for thinking):

- Total vehicles: **V = B + S**
- Total meals: **M = 8B + 5S**
- Total charge used: **C = 3B + 2S**

---

4. **Key constraints to consider**

Model the problem using the following constraint groups:

1. **Charging capacity constraint**
   - The total charge consumed by all bikes and scooters cannot exceed available charge:
     - 3 units per bike, 2 units per scooter
     - Constraint:  
       - Total charge used ≤ 200  
       - In variables: 3B + 2S ≤ 200

2. **Vehicle mix (policy) constraint**
   - At most 30% of all electric vehicles can be bikes.
   - Formally: B ≤ 0.3(B + S)
   - This is linear; rearrange to an equivalent form for the model:
     - 0.7B ≤ 0.3S → 7B ≤ 3S
   - This ensures the proportion of bikes among all vehicles does not exceed 30%.

3. **Minimum number of scooters**
   - There must be at least 20 scooters:
     - S ≥ 20

4. **Non-negativity and integrality**
   - The number of each vehicle must be non-negative integers:
     - B ≥ 0, S ≥ 0
     - B, S ∈ Integers

---

5. **Objective function structure**

- **Objective:** Maximize the total number of meals delivered:
  - Each bike carries 8 meals; each scooter carries 5 meals:
  - Maximize M = 8B + 5S

---

6. **Modelling notes / checks**

- The ratio constraint (at most 30% bikes) is the only potentially tricky piece; ensure it is properly linearized as **7B ≤ 3S**.
- Confirm that the chosen units match the problem statement:
  - Charge “units” are consistent across bikes and scooters.
  - Vehicle counts are small enough that integer modelling is appropriate (no fractional vehicles).
- Once this MILP is set up, the modeller can:
  1. Solve via a standard MILP solver.
  2. Optionally examine the LP relaxation to see if the optimal solution is naturally integral; if so, an LP suffices.

This is all that is needed: two integer decision variables, a linear objective, and four main linear constraints.

## 1. Decision variables

Let

- \(B \in \mathbb{Z}_{\ge 0}\): number of electric bikes used  
- \(S \in \mathbb{Z}_{\ge 0}\): number of electric scooters used  

(Only these two integer variables are needed.)

---

## 2. Objective

Maximize the total number of meals delivered.

- Each bike carries 8 meals
- Each scooter carries 5 meals

Objective function:

\[
\max\; M = 8B + 5S
\]

---

## 3. Constraints

1. **Charging capacity**

Each bike uses 3 units of charge, each scooter 2 units. Total charge ≤ 200:

\[
3B + 2S \le 200
\]

2. **Vehicle mix (at most 30% bikes)**

At most 30% of all electric vehicles can be bikes:

\[
B \le 0.3(B + S)
\]

Rearranging to a linear form:

\[
B \le 0.3B + 0.3S \quad\Rightarrow\quad 0.7B \le 0.3S \quad\Rightarrow\quad 7B \le 3S
\]

So we use:

\[
7B \le 3S
\]

3. **Minimum number of scooters**

At least 20 scooters must be used:

\[
S \ge 20
\]

4. **Non-negativity and integrality**

\[
B \ge 0,\; S \ge 0,\quad B, S \in \mathbb{Z}
\]

---

## 4. Full MILP formulation

\[
\begin{array}{ll}
\text{Maximize} & 8B + 5S \\[4pt]
\text{subject to} & 3B + 2S \le 200 \\[4pt]
                  & 7B \le 3S \\[4pt]
                  & S \ge 20 \\[4pt]
                  & B \ge 0,\; S \ge 0 \\[4pt]
                  & B, S \in \mathbb{Z}
\end{array}
\]

---

## 5. Suitable Python tools

This is a small Mixed-Integer Linear Program. Any of the following Python libraries is suitable:

- **PuLP**: Simple, pure-Python modeling language; uses CBC by default.
- **OR-Tools** (`ortools.linear_solver` or `ortools.sat.python.cp_model`): Very efficient open-source solvers.
- **Pyomo**: More general algebraic modeling environment; supports many MILP solvers (CBC, GLPK, Gurobi, CPLEX, etc.).
- **SciPy** is *not* ideal here because `scipy.optimize.linprog` does linear programming without integer constraints.

---

## 6. Minimal example with PuLP

```python
import pulp as pl

# Problem
prob = pl.LpProblem("Meal_Delivery_Vehicles", pl.LpMaximize)

# Decision variables (non-negative integers)
B = pl.LpVariable("Bikes", lowBound=0, cat="Integer")
S = pl.LpVariable("Scooters", lowBound=0, cat="Integer")

# Objective: maximize 8B + 5S
prob += 8 * B + 5 * S, "Total_meals"

# Constraints
prob += 3 * B + 2 * S <= 200          # Charge capacity
prob += 7 * B <= 3 * S                # At most 30% bikes
prob += S >= 20                       # At least 20 scooters

# Solve
prob.solve(pl.PULP_CBC_CMD(msg=False))

print("Status:", pl.LpStatus[prob.status])
print("Bikes (B):", pl.value(B))
print("Scooters (S):", pl.value(S))
print("Max meals:", pl.value(prob.objective))
```

This model directly implements the formulation above and will yield the optimal numbers of bikes and scooters under the given constraints.

import pulp as pl

def main():
    # Problem definition
    prob = pl.LpProblem("Meal_Delivery_Vehicles", pl.LpMaximize)

    # Decision variables (non-negative integers)
    B = pl.LpVariable("Bikes", lowBound=0, cat="Integer")
    S = pl.LpVariable("Scooters", lowBound=0, cat="Integer")

    # Objective: maximize 8B + 5S
    prob += 8 * B + 5 * S, "Total_meals"

    # Constraints
    prob += 3 * B + 2 * S <= 200          # Charge capacity
    prob += 7 * B <= 3 * S                # At most 30% bikes
    prob += S >= 20                       # At least 20 scooters

    # Solve using CBC (default solver in PuLP)
    prob.solve(pl.PULP_CBC_CMD(msg=False))

    # Extract objective value
    obj_value = pl.value(prob.objective)

    # Final required print
    print(f"FINAL_OBJ: {obj_value}")

if __name__ == "__main__":
    main()
