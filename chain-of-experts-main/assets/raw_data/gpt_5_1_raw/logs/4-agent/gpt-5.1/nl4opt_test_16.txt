1. Key optimisation goal  
Goal: Minimise the total “wait time” until the pool is ready, given that all chlorine and all water softener must be added in sequence (all of one type first, then all of the other type), and each unit has a different time to become effective.

Clarify interpretation for the Modeller:  
- Because the note says “chlorine must be added before water softener or vice versa” and each unit takes a fixed time to become effective, the relevant objective is the time at which *all* added chemicals have become effective, assuming:
  - We add units one after another (cannot add both types simultaneously).
  - The pool is “ready” only when every unit of both chemicals has had its full reaction time.
- Therefore:
  - If chlorine is added first: total completion time = time to add all chlorine + time for last chlorine unit to become effective (1 minute after its addition) + time to add all softener + time for last softener unit to become effective (2 minutes after its addition).
  - If softener is added first: symmetric structure with softener’s 2‑minute effect time first, then chlorine’s 1‑minute time.
- Since these shift by constants, it is enough to minimise a linear combination of the total quantities, and then check both sequences if needed.

2. Suggested model type  
Model: Linear Programming (LP) with a small discrete case check (two possible sequences: chlorine-first, softener-first).  
- Because the sequencing alternatives are only two, we can:
  - Either handle them as two separate LPs and choose the better result, or
  - Introduce a binary variable to encode sequence and still keep it very small (MIP).  
- For a “clean” optimisation strategy, prefer: solve two LPs (one for each order), compare objective values, and pick the minimum.

3. Main decision variables  

Let:
- \(C\): number of units of chlorine to add.
- \(S\): number of units of water softener to add.

Possible additional (auxiliary) variable:
- \(T\): total time until the pool is ready, if you want an explicit “minimise T” formulation, though you can also write T directly from C and S.

No need for discrete variables for C and S unless the Modeller is asked to enforce integrality explicitly; they can be treated as continuous non-negative reals if units are divisible.

4. Key constraints to consider  

Problem constraints:
1. Total chemical volume:
   - \(C + S = 500\) units of chemicals must be added in total.

2. Minimum chlorine requirement:
   - \(C \ge 200\).

3. Safety ratio on chlorine:
   - “At most half as much chlorine as water softener” → chlorine cannot exceed half the amount of softener:
   - \(C \le 0.5\,S\).

4. Non-negativity:
   - \(C \ge 0, S \ge 0\).

5. Order-of-addition cases (handle as two separate LPs):

   Case A: Chlorine added first, then softener.  
   - Time structure:
     - Total addition time: proportional to (C + S), but we’re not given explicit per-unit addition times; we only know how long each unit takes to become effective *after* being added.
     - A standard simplifying interpretation for this type of textbook problem is:
       - All units of a given chemical are dumped at once (or fast enough that addition time is negligible compared to effectiveness time), but their *effectiveness* lag is: 1 minute for chlorine, 2 minutes for softener.
       - Because we must finish adding all of one chemical before starting the other, the bottleneck is simply:  
         - If chlorine first: effective completion time = 1 minute (chlorine lag) + 2 minutes (softener lag) = 3 minutes, regardless of how much is added.  
         - Under this interpretation, the objective does not depend on C and S at all, which contradicts the presence of an optimisation.  
   - To make the optimisation meaningful, we instead model “time” as proportional to “total active-chemical-minutes” or total waiting time for each unit. Then:
     - If we add all chlorine units first, then softener:
       - Each chlorine unit must “wait” 1 minute to be effective, each softener unit 2 minutes.
       - If addition itself is effectively instantaneous, again total completion time doesn’t depend on quantities.
   - Therefore, for a modelling plan, we should **adopt a specific linear objective** that accords with typical LP exercises:
     - Objective: Minimise total “chemical-minute waiting cost” = 1 × C + 2 × S (or a variant clearly specified by the Modeller).
     - This is justified as: each unit of chlorine contributes 1 minute of wait, each unit of softener contributes 2 minutes of wait; we seek the combination that meets constraints while minimising total waiting impact.

   Case B: Softener added first, then chlorine.  
   - Under the same “chemical-minute” objective, order does not change the linear combination 1×C + 2×S; both sequences have the same objective form.
   - Hence, the sequence constraint becomes irrelevant for the optimisation itself, and both cases produce the same optimal (C, S), so the Modeller only needs one LP.

- Then note that the “order” constraint has no effect on that objective (only on process description), so it can be ignored in the optimisation.

\[

\[

\[

C \ge 0,\quad S \ge 0

### 3. Objective function (what is being minimised)

The wording “One unit of chlorine takes 1 minute to be effective while one unit of water softener takes 2 minutes to be effective” is ambiguous about *process time*, but for this to be a meaningful LP (as the planner argued), we adopt the “chemical‑minute” interpretation:

- Each unit of chlorine contributes 1 minute of “waiting effect”.
- Each unit of softener contributes 2 minutes of “waiting effect”.

We then minimise the total waiting effect across all units:

\[
\min \; Z = 1 \cdot C + 2 \cdot S
\]

Subject to the constraints above.

Note: Under this interpretation, the order “chlorine first or softener first” does **not** change the objective value, because the objective is purely a function of the amounts \(C\) and \(S\). So we do not need explicit sequence variables.

### 4. Full LP model

\[
\begin{array}{ll}
\text{Minimise}   & Z = C + 2S \\[4pt]
\text{subject to} & C + S = 500 \\[2pt]
                  & C \ge 200 \\[2pt]
                  & C \le \tfrac{1}{2} S \\[2pt]
                  & C \ge 0,\; S \ge 0
\end{array}
\]

If integrality of units is required:

\[
C, S \in \mathbb{Z}_{\ge 0}
\]

### 5. How to solve in Python

Because this is a very small linear program, you can use any standard LP/MIP solver. Suitable libraries:

- **PuLP** (pure Python, easy for teaching/demo)
- **OR‑Tools linear solver** (`pywraplp`) by Google
- **SciPy**’s `optimize.linprog` (for continuous LP only; no integer constraints)

Below is an example in PuLP (continuous LP):

```python
import pulp as pl

# Problem
prob = pl.LpProblem("Pool_Chemicals", pl.LpMinimize)

# Decision variables
C = pl.LpVariable('C', lowBound=0)
S = pl.LpVariable('S', lowBound=0)

# Objective
prob += C + 2*S

# Constraints
prob += C + S == 500
prob += C >= 200
prob += C <= 0.5 * S

# Solve
prob.solve(pl.PULP_CBC_CMD(msg=False))

print("Status:", pl.LpStatus[prob.status])
print("C (chlorine):", C.value())
print("S (softener):", S.value())
print("Objective (total wait):", pl.value(prob.objective))
```

To enforce integer units, just define:

```python
C = pl.LpVariable('C', lowBound=0, cat='Integer')
S = pl.LpVariable('S', lowBound=0, cat='Integer')
```

The same model can be directly translated to OR‑Tools or `scipy.optimize.linprog` with the same objective and constraints.

import pulp as pl

def solve_pool_chemicals():
    # Create the problem: minimize total waiting effect
    prob = pl.LpProblem("Pool_Chemicals", pl.LpMinimize)

    # Decision variables: continuous, non-negative
    C = pl.LpVariable('C', lowBound=0)
    S = pl.LpVariable('S', lowBound=0)

    # Objective function: minimize C + 2S
    prob += C + 2 * S

    # Constraints
    prob += C + S == 500        # Total amount of chemicals
    prob += C >= 200            # Minimum chlorine requirement
    prob += C <= 0.5 * S        # Safety ratio: at most half as much chlorine as softener

    # Solve the problem
    prob.solve(pl.PULP_CBC_CMD(msg=False))

    # Extract objective value
    obj_value = pl.value(prob.objective)

    return obj_value

if __name__ == "__main__":
    value = solve_pool_chemicals()
    print(f"FINAL_OBJ: {value}")
