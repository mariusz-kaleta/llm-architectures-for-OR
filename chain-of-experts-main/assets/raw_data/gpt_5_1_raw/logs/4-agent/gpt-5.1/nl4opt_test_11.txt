[PLANNER]:
1. Key optimisation goal  
   - Goal: Minimise the total number of bikes used, while still transporting at least 500 people and respecting the limit on the fraction of cars.

2. Suggested model type  
   - Model: Integer Linear Programming (ILP), or a small discrete optimisation model (variables are counts of vehicles, so integrality is important).

3. Main decision variables  
   Define integer non‑negative variables:
   - Number of bikes used.
   - Number of cars used.

4. Key constraints to consider  

   a) Capacity (people transported)  
   - Each bike carries 3 people.  
   - Each car carries 5 people.  
   - Total carried must be at least 500 people.  
   ⇒ Construct a capacity constraint of the form:  
   (3 × number of bikes) + (5 × number of cars) ≥ 500.

   b) Car proportion limit  
   - At most 40% of the vehicles can be cars.  
   - “Vehicles” = bikes + cars.  
   ⇒ Fraction of vehicles that are cars ≤ 0.4.  
   - Translate this into a linear inequality without division (cross‑multiply using non‑negativity):  
     number of cars ≤ 0.4 × (number of bikes + number of cars).

   c) Non‑negativity and integrality  
   - Number of bikes ≥ 0, integer.  
   - Number of cars ≥ 0, integer.

5. Objective function structure  
   - Objective: Minimise the number of bikes.  
   - Do not penalise or minimise cars directly; they will be indirectly controlled by the 40% constraint and the capacity requirement.

6. Modelling strategy / notes for the Modeller  
   - First, set up the inequalities symbolically with the two decision variables.  
   - Simplify the “at most 40% cars” constraint into a linear relation between the two integer variables.  
   - Use the capacity constraint plus that relation to characterise the feasible region.  
   - Since there are only two variables and the problem is small, one can:
     - Either solve graphically/analytically (treat variables as continuous, find boundary/extreme points, then round and check nearby integer points), or  
     - Directly search over feasible car counts consistent with the 40% rule and choose the smallest bike count satisfying the capacity constraint.
   - Verify integrality: ensure the final chosen pair (bikes, cars) are integers and meet all constraints.

Here is a clean mathematical/algorithmic model of the problem, followed by suitable Python tools to solve it.

---

## 1. Decision variables

Let

- \(b \in \mathbb{Z}_{\ge 0}\): number of bikes  
- \(c \in \mathbb{Z}_{\ge 0}\): number of cars  

---

## 2. Objective

Minimise the total number of bikes:

\[
\min \; b
\]

(We are not directly minimising cars; their use is constrained by the 40% rule and capacity.)

---

## 3. Constraints

### a) Capacity constraint (at least 500 people)

Each bike carries 3 people, each car carries 5 people:

\[
3b + 5c \;\ge\; 500
\]

---

### b) Car proportion limit (at most 40% of vehicles are cars)

“Vehicles” = bikes + cars = \(b + c\).  
At most 40% of all vehicles can be cars:

\[
\frac{c}{b + c} \;\le\; 0.4
\]

Since \(b, c \ge 0\) and \(b + c > 0\) for any nontrivial solution, we can rewrite without division:

\[
c \;\le\; 0.4(b + c)
\]

\[
c \;\le\; 0.4b + 0.4c
\]

\[
c - 0.4c \;\le\; 0.4b
\]

\[
0.6c \;\le\; 0.4b
\]

\[
3c \;\le\; 2b
\]

So the linear inequality is:

\[
3c \;\le\; 2b
\]

---

### c) Non‑negativity and integrality

\[
b \in \mathbb{Z}_{\ge 0}, \quad c \in \mathbb{Z}_{\ge 0}
\]

---

## 4. Complete ILP model

\[
\begin{aligned}
\min \quad & b \\
\text{s.t.} \quad
& 3b + 5c \;\ge\; 500, \\
& 3c \;\le\; 2b, \\
& b, c \in \mathbb{Z}_{\ge 0}.
\end{aligned}
\]

---

## 5. Algorithmic solution approach (if coded directly)

Because there are only two integer variables and the model is small:

1. Loop over feasible car counts \(c = 0,1,2,\dots\) up to some safe upper bound.  
2. For each \(c\), compute the minimum \(b\) that satisfies:
   - Capacity: \(b \ge \lceil (500 - 5c)/3 \rceil\)
   - Car ratio: \(b \ge \left\lceil \tfrac{3}{2}c \right\rceil\)
   - And \(b \ge 0\).
3. Among all feasible pairs \((b,c)\), choose the one with minimum \(b\).

This brute-force search is trivial in size for a computer.

---

## 6. Suitable Python libraries

Any standard mixed-integer linear programming (MILP) or ILP solver can handle this:

- **PuLP**  
  - Very straightforward for educational/small ILP models.  
  - Example structure:
    ```python
    import pulp as pl

    model = pl.LpProblem("BanquetTransport", pl.LpMinimize)

    b = pl.LpVariable("bikes", lowBound=0, cat="Integer")
    c = pl.LpVariable("cars", lowBound=0, cat="Integer")

    model += b  # objective: minimize bikes
    model += 3*b + 5*c >= 500
    model += 3*c <= 2*b

    model.solve(pl.PULP_CBC_CMD(msg=False))
    print(b.value(), c.value())
    ```

- **OR-Tools (Python)**  
  - Use `ortools.linear_solver.pywraplp` or `ortools.sat.python.cp_model`.  
  - Good for integer/CP optimization, very fast.

- **Pyomo** + a MILP solver (e.g. CBC, GLPK, Gurobi, CPLEX)  
  - More general modeling framework; good if the model will grow in complexity.

- **scipy.optimize** is *not* ideal here because it primarily targets continuous problems; you’d need an extra layer to enforce integrality, so MILP-specific libraries like PuLP or OR-Tools are preferable.

import math

def solve_bikes_cars():
    best_b = None
    best_c = None

    # Safe upper bound for cars:
    # If all people were in cars: 5c >= 500 -> c >= 100
    max_c = 200

    for c in range(max_c + 1):
        # Capacity constraint: 3b + 5c >= 500 -> b >= (500 - 5c)/3
        if 5 * c >= 500:
            b_cap = 0
        else:
            b_cap = math.ceil((500 - 5 * c) / 3)

        # Car proportion constraint: 3c <= 2b -> b >= (3/2)c
        b_ratio = math.ceil(1.5 * c)

        # Non-negativity
        b_min = max(0, b_cap, b_ratio)

        # Check feasibility: if b_min satisfies constraints, it's feasible for this c
        if 3 * b_min + 5 * c >= 500 and 3 * c <= 2 * b_min:
            if best_b is None or b_min < best_b:
                best_b = b_min
                best_c = c

    return best_b, best_c

if __name__ == "__main__":
    b_opt, c_opt = solve_bikes_cars()
    # Objective is to minimize b
    FINAL_OBJ = b_opt
    print(f"FINAL_OBJ: {FINAL_OBJ}")
