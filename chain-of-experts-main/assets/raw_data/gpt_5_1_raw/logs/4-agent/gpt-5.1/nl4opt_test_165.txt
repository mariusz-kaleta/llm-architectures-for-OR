[MODELER]:
Below is a clean mathematical model of the problem, followed by notes on how to solve it with Python.

---

### 1. Decision variables

- \(x_1 \ge 0\): number of regular tacos to make (and sell)  
- \(x_2 \ge 0\): number of deluxe tacos to make (and sell)

(If you want integer counts of tacos, you can additionally require \(x_1, x_2 \in \mathbb{Z}\), but the core model is linear and works as an LP.)

---

### 2. Objective function

Each regular taco earns \$2.50 profit; each deluxe taco earns \$3.55 profit.

Maximize total profit:
\[
\max\; Z = 2.50\,x_1 + 3.55\,x_2
\]

---

### 3. Constraints

1. **Demand constraints (upper bounds by type)**  
   At most 50 regular tacos can be sold:
   \[
   x_1 \le 50
   \]
   At most 40 deluxe tacos can be sold:
   \[
   x_2 \le 40
   \]

2. **Total supply / production capacity**  
   Total tacos of all types cannot exceed 70:
   \[
   x_1 + x_2 \le 70
   \]

3. **Non-negativity**
   \[
   x_1 \ge 0,\quad x_2 \ge 0
   \]

4. **(Optional) Integrality**
   \[
   x_1, x_2 \in \mathbb{Z}
   \]
   if you require whole tacos. For most textbook LP treatments, this is omitted and \(x_1, x_2\) are allowed to be continuous.

---

### 4. Full model (compact form)

\[
\begin{aligned}
\max_{x_1,x_2} \quad & 2.50\,x_1 + 3.55\,x_2 \\
\text{s.t.}\quad
& x_1 \le 50 \\
& x_2 \le 40 \\
& x_1 + x_2 \le 70 \\
& x_1 \ge 0,\; x_2 \ge 0 \\
& (\text{optionally } x_1, x_2 \in \mathbb{Z})
\end{aligned}
\]

---

### 5. Suitable Python libraries

To implement and solve this model in Python, any standard LP/MIP library will work:

- **PuLP**  
  - Easy, pure-Python modeling interface.  
  - Good for teaching and small/medium-sized problems.
  - Example usage: `pulp.LpProblem`, `LpVariable`, `LpMaximize`.

- **OR-Tools (Google OR-Tools)**  
  - Very powerful, supports LP and MIP via `pywraplp`.  
  - Good if you might extend to larger or more complex models.

- **SciPy (`scipy.optimize.linprog`)**  
  - Good for continuous LPs (no integrality).  
  - Less convenient for explicit integer constraints.

For a straightforward educational problem like this, **PuLP** or **OR-Tools** is typically the best choice.

[CODER]:
import pulp

def main():
    # Create the LP problem: maximize profit
    prob = pulp.LpProblem("Taco_Production", pulp.LpMaximize)

    # Decision variables: x1 = regular tacos, x2 = deluxe tacos
    x1 = pulp.LpVariable('x1', lowBound=0, cat='Continuous')
    x2 = pulp.LpVariable('x2', lowBound=0, cat='Continuous')

    # Objective function: maximize 2.50 x1 + 3.55 x2
    prob += 2.50 * x1 + 3.55 * x2

    # Constraints
    prob += x1 <= 50          # At most 50 regular tacos
    prob += x2 <= 40          # At most 40 deluxe tacos
    prob += x1 + x2 <= 70     # Total tacos at most 70

    # Solve the problem
    prob.solve(pulp.PULP_CBC_CMD(msg=False))

    # Get the objective value
    final_obj = pulp.value(prob.objective)

    # Print the result in the required format
    print(f"FINAL_OBJ: {final_obj}")

if __name__ == "__main__":
    main()
