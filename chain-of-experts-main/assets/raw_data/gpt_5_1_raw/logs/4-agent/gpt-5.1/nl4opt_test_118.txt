[PLANNER]:
1. Key optimisation goal  
Goal: Minimize the total transportation time required to move at least 300 fish from the pond, given capacity and trip-mix restrictions.

2. Suggested model type  
Model: Integer Linear Programming (ILP)  
– Reason:  
  - Number of trips by helicopter and car are naturally integer.  
  - Objective (total time) is linear in the number of trips.  
  - Capacity and policy restrictions are all linear relationships.

3. Main decision variables  
Define integer, non-negative variables such as:  
- H = number of helicopter trips  
- C = number of car trips  

(If desired, “total fish transported” can be a derived variable, but is not essential.)

4. Key constraints to consider  

a) Capacity / demand constraint  
- Each helicopter trip transports 30 fish.  
- Each car trip transports 20 fish.  
- At least 300 fish must be transported.  
→ Total fish transported by all trips ≥ 300.

b) Limit on helicopter usage  
- At most 5 helicopter trips are allowed.  
→ H ≤ 5.

c) Proportion of car trips  
- At least 60% of all trips must be by car.  
- “Trips” here means helicopter + car trips.  
→ C must be at least 60% of (H + C).  
  - This can be written as a linear inequality relating C and H.

d) Non-negativity and integrality  
- H ≥ 0, C ≥ 0.  
- H and C must be integers.

5. Objective function structure  

Objective: Minimize total time spent on all trips.  
- Each helicopter trip takes 40 minutes.  
- Each car trip takes 30 minutes.  
→ Total time is a linear function of H and C:  
  - 40 minutes × H  
  - 30 minutes × C  

The modeller should:  
- Formulate the linear objective with the above coefficients.  
- Implement the linear constraints for fish capacity, helicopter trip limit, and car-trip proportion.  
- Enforce integrality on H and C.  
- Solve for the integer solution (H, C) that yields minimum total time while transporting at least 300 fish and respecting all constraints.

[MODELER]:
Here is a clean mathematical optimisation model for the problem, followed by Python library suggestions.

---

## 1. Decision variables

Let  
- \(H \in \mathbb{Z}_{\ge 0}\): number of helicopter trips  
- \(C \in \mathbb{Z}_{\ge 0}\): number of car trips  

Both must be integers and non‑negative.

---

## 2. Objective function

Minimize total transportation time (in minutes):

\[
\min\; T = 40H + 30C
\]

- Each helicopter trip: 40 minutes  
- Each car trip: 30 minutes

---

## 3. Constraints

1. **Fish demand (capacity) constraint**

Each helicopter carries 30 fish, each car 20 fish; at least 300 fish must be moved:

\[
30H + 20C \;\ge\; 300
\]

2. **Helicopter trip limit**

At most 5 helicopter trips are allowed:

\[
H \;\le\; 5
\]

3. **Car-trip proportion constraint**

At least 60% of all trips must be by car, i.e.:

\[
C \;\ge\; 0.6(H + C)
\]

Rewriting to a linear form:

\[
C \;\ge\; 0.6H + 0.6C
\]

\[
0.4C \;\ge\; 0.6H
\]

\[
2C \;\ge\; 3H
\]

So the linear constraint is:

\[
2C - 3H \;\ge\; 0
\]

4. **Integrality and non-negativity**

\[
H \in \mathbb{Z}_{\ge 0}, \quad C \in \mathbb{Z}_{\ge 0}
\]

---

## 4. Complete ILP model

\[
\begin{aligned}
\min\quad & 40H + 30C \\
\text{s.t.}\quad 
& 30H + 20C \ge 300 \quad &\text{(fish demand)} \\
& H \le 5 \quad &\text{(helicopter limit)} \\
& 2C - 3H \ge 0 \quad &\text{(car-trip proportion)} \\
& H, C \in \mathbb{Z}_{\ge 0} \quad &\text{(integrality, non-negativity)}
\end{aligned}
\]

---

## 5. Suitable Python libraries

This is a small Integer Linear Program. Any standard MILP or LP+integer solver will work efficiently. Good Python options:

- **PuLP**  
  - Simple to model ILPs with clear syntax.  
  - Can use CBC (bundled), GLPK, or commercial solvers.  
  - Good for teaching, small to medium problems.

- **OR-Tools (Google OR-Tools, `ortools.linear_solver`)**  
  - Very efficient MILP solvers (SCIP or CBC backends).  
  - Excellent for combinatorial/operations research problems.  

- **Pyomo**  
  - More general algebraic modelling language in Python.  
  - Suitable if you plan to scale to more complex models.

- **SciPy (`scipy.optimize`)**  
  - Primarily for continuous optimisation; integer programming is not directly supported.  
  - Not recommended here since integrality is essential.

For this specific problem, **PuLP** or **OR-Tools** are the most straightforward choices.

[CODER]:
import pulp

def main():
    # Create the problem
    prob = pulp.LpProblem("Fish_Transport_Optimization", pulp.LpMinimize)

    # Decision variables
    H = pulp.LpVariable('H', lowBound=0, cat='Integer')  # helicopter trips
    C = pulp.LpVariable('C', lowBound=0, cat='Integer')  # car trips

    # Objective function: Minimize total time
    prob += 40 * H + 30 * C, "Total_Transport_Time"

    # Constraints
    # Fish demand: 30H + 20C >= 300
    prob += 30 * H + 20 * C >= 300, "Fish_Demand"

    # Helicopter trip limit: H <= 5
    prob += H <= 5, "Helicopter_Limit"

    # Car-trip proportion: 2C - 3H >= 0
    prob += 2 * C - 3 * H >= 0, "Car_Proportion"

    # Solve the problem
    prob.solve(pulp.PULP_CBC_CMD(msg=0))

    # Get the optimal objective value
    final_obj = pulp.value(prob.objective)

    # Print the final objective value in required format
    print(f"FINAL_OBJ: {final_obj}")

if __name__ == "__main__":
    main()
