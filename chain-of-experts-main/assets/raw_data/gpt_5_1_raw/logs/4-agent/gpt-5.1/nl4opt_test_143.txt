1. **Key optimisation goal**

- **Goal:** Minimize the **total time spent transporting** the required 1500 units, while respecting the company’s preference for **more camel caravans than desert trucks**.

   Here "total time" should be interpreted as the **sum of hours across all trips** (i.e., number of caravans × 12 hours + number of trucks × 5 hours), unless a different interpretation is specified.

2. **Suggested model type**

- **Model:** Integer Linear Programming (ILP)
  - Decision variables are counts of trips, so they must be **non‑negative integers**.
  - Objective and constraints are linear.

3. **Main decision variables**

Define at least:

- **C** = number of camel caravan trips used.
- **T** = number of desert truck trips used.

Both must be integers ≥ 0.

(If needed later, one could introduce binary variables to handle more complex preference structures, but for now two integer variables are sufficient.)

4. **Key constraints to consider**

a. **Demand satisfaction (capacity constraint)**  
   - Each camel caravan trip delivers 50 units.  
   - Each truck trip delivers 150 units.  
   - Total delivered must be at least the required demand:
     - 50·C + 150·T ≥ 1500

   Optionally, model it as equality if the problem assumes no over‑delivery:
   - 50·C + 150·T = 1500  
   but using ≥ allows over‑delivery if that could yield fewer total hours.

b. **Preference for more camel caravans than trucks**  
   - The statement “prefers to have more camel caravans than desert trucks” should be encoded as:
     - C ≥ T  
   or, if the interpretation is “strictly more”:
     - C ≥ T + 1

   The modeller should choose one based on how strictly the preference must be enforced.

c. **Non‑negativity and integrality**
   - C ≥ 0, T ≥ 0
   - C, T are integers.

d. **Optional practical constraints (if needed)**  
   These are not stated but could be considered depending on context:
   - **Upper bounds** on available caravans or trucks:
     - C ≤ C_max, T ≤ T_max
   - **Trip simultaneity / fleet limits** if the time notion is reinterpreted as makespan (see below).

5. **Objective function design**

Clarify the interpretation of “minimize the total number of hours required”:

- **Most straightforward interpretation (sum of hours)**  
  - Total time cost:
    - TotalHours = 12·C + 5·T  
  - **Objective:** Minimize 12·C + 5·T

  This treats each trip’s hours additively, modeling “total work-hours” (or machine hours).

- **Alternative interpretation (makespan / completion time)** – only if problem is re‑interpreted:
  - If the company can send all C caravans and T trucks **in parallel** and only cares about when the **last** trip finishes:
    - Time to finish is max(12 if C>0, 5 if T>0), which is always 12 hours if any caravans are used.
  - That interpretation does not align with the idea of minimizing “total hours” with a camel preference, so the additive formulation is more natural.  
  The modeller should stick to the **additive objective** unless instructed otherwise.

6. **Trade‑offs / modelling notes**

- Because trucks are more time-efficient (150 units in 5 hours vs. 50 units in 12 hours), the unconstrained time-minimizing solution (ignoring preference) would likely use mostly or only trucks.
- The **preference constraint (C ≥ T or C ≥ T+1)** is therefore crucial; it will push the solution to include more caravans, possibly sacrificing some time efficiency.
- It may be useful to:
  - First solve the model **without** the preference constraint to see the theoretical minimum hours.
  - Then solve **with** C ≥ T (or C ≥ T+1) to get the preferred solution and compare.

7. **Summary for the modeller**

- Use an **ILP model** with:
  - Decision variables: C, T (non-negative integers).
  - Objective: Minimize 12·C + 5·T.
  - Core constraints:
    - 50·C + 150·T ≥ 1500 (or = 1500 if no over‑delivery).
    - C ≥ T (or C ≥ T + 1, per strictness of preference).
- Solve and report:
  - Optimal C and T.
  - Total units delivered.
  - Total hours = 12·C + 5·T.

## 1. Decision variables

Let

- \(C \in \mathbb{Z}_{\ge 0}\): number of camel caravan trips.
- \(T \in \mathbb{Z}_{\ge 0}\): number of desert truck trips.

(Each variable counts trips of that type.)

---

## 2. Parameters

- Demand: \(D = 1500\) units.
- Camel capacity per trip: \(q_C = 50\) units.
- Truck capacity per trip: \(q_T = 150\) units.
- Camel trip time: \(h_C = 12\) hours.
- Truck trip time: \(h_T = 5\) hours.

---

## 3. Constraints

### 3.1 Demand satisfaction (capacity constraint)

Total transported must cover the demand of 1500 units.

You can model either:

- Allowing over‑delivery:
  \[
  50C + 150T \;\ge\; 1500
  \]

or, if “exactly 1500” is intended with no over‑delivery:

- Exact demand:
  \[
  50C + 150T \;=\; 1500
  \]

From a pure “minimize total hours” perspective, allowing \(\ge\) is slightly more realistic, because sometimes a small over‑delivery can reduce the number of trips; choose based on the interpretation required.

### 3.2 Preference: more camel caravans than trucks

The text: “the company prefers to have more camel caravans than desert trucks.”

Two possible encodings:

- Weak preference (at least as many caravans as trucks):
  \[
  C \;\ge\; T
  \]

- Strict preference (strictly more caravans than trucks):
  \[
  C \;\ge\; T + 1
  \]

Specify one of these. Mathematically both are linear integer constraints.

### 3.3 Non‑negativity and integrality

\[
C \;\ge\; 0,\quad T \;\ge\; 0,\quad C,T \in \mathbb{Z}
\]

---

## 4. Objective function

We interpret “minimize the total number of hours required” as minimizing the **sum of trip-hours**:

\[
\text{TotalHours} = 12C + 5T
\]

Objective:

\[
\min \; 12C + 5T
\]

This is linear in the integer variables, so the full model is a (small) Integer Linear Program (ILP).

---

## 5. Full ILP model

Using the ≥‑demand and weak preference example:

\[
\begin{aligned}
\min\quad & 12C + 5T \\
\text{s.t.}\quad 
& 50C + 150T \;\ge\; 1500 \\
& C \;\ge\; T \\
& C, T \in \mathbb{Z}_{\ge 0}
\end{aligned}
\]

Or substitute the equality and/or \(C \ge T+1\) depending on the exact interpretation you adopt.

---

## 6. Suitable Python libraries

Because this is a linear objective with linear constraints and integer decision variables, any standard MILP solver or modeling library is appropriate:

- **PuLP**  
  - Simple, pure‑Python modeling library for LP/MILP.
  - Interfaces to CBC by default (open source), and can connect to other solvers.

- **OR-Tools (Google OR-Tools)**  
  - Has a CP-SAT solver and a linear solver interface.
  - For this problem size, CP-SAT or the MIP interface will solve instantly.

- **Pyomo**  
  - General algebraic modeling language in Python.
  - More flexible if the model grows more complex.

- **scipy.optimize** is **not ideal** here, because it targets continuous (non-integer) problems and would require extra work to enforce integrality.

For a quick solution, **PuLP** or **OR-Tools** are the most straightforward choices.

import pulp

def solve_camel_truck_problem():
    # Parameters
    D = 1500        # Demand
    qC = 50         # Camel capacity
    qT = 150        # Truck capacity
    hC = 12         # Camel trip time
    hT = 5          # Truck trip time

    # Define problem: minimize total hours
    prob = pulp.LpProblem("Camel_Truck_Trips", pulp.LpMinimize)

    # Decision variables: non-negative integers
    C = pulp.LpVariable("C", lowBound=0, cat=pulp.LpInteger)
    T = pulp.LpVariable("T", lowBound=0, cat=pulp.LpInteger)

    # Objective function: minimize total hours
    prob += hC * C + hT * T

    # Constraints
    # Demand satisfaction (allow over-delivery)
    prob += qC * C + qT * T >= D

    # Preference: at least as many caravans as trucks (weak preference)
    prob += C >= T

    # Solve
    prob.solve(pulp.PULP_CBC_CMD(msg=False))

    # Extract objective value
    total_hours = pulp.value(prob.objective)
    return total_hours

if __name__ == "__main__":
    value = solve_camel_truck_problem()
    print(f"FINAL_OBJ: {value}")
