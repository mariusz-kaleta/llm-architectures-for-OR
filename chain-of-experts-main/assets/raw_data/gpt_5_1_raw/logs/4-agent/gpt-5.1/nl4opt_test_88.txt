1. Key optimisation goal  
Goal: Minimise the total number of probes used across both tests.

---

2. Suggested model type  
Model: Linear Programming (LP) with two decision variables and simple linear constraints (small enough to solve graphically or by simplex).

---

3. Main decision variables  
Define non‑negative variables:

- \(x\): Number of salinity tests to perform.  
- \(y\): Number of pH tests to perform.

(Assume \(x, y\) can be real numbers unless the problem explicitly demands integers. If integer counts are required, treat it as an Integer Linear Program (ILP).)

---

4. Objective function (conceptual)  
Total probes used:

- Each salinity test uses 3 probes → contribution: \(3x\).
- Each pH test uses 2 probes → contribution: \(2y\).

Objective:  
Minimise total probes = \(3x + 2y\).

---

5. Key constraints to consider  

a) Minimum number of pH tests  
- The company must perform at least 250 pH tests.  
  - Constraint: \(y \ge 250\).

b) Minimum total number of tests  
- Total tests (salinity + pH) must be at least 400.  
  - Total tests: \(x + y\).  
  - Constraint: \(x + y \ge 400\).

c) Balance between pH and salinity tests  
- There must be at most 1.5 times more pH tests than salinity tests.  
  - “At most 1.5 times more pH than salinity” → \(y \le 1.5x\).

d) Non‑negativity  
- Cannot perform a negative number of tests.  
  - \(x \ge 0\), \(y \ge 0\).

e) (Optional) Integrality  
- If required by the context: \(x, y\) must be integers.

---

6. Solution strategy for the modeller  

1. Formulate the LP with:
   - Objective: Minimise \(3x + 2y\).
   - Constraints:  
     - \(y \ge 250\)  
     - \(x + y \ge 400\)  
     - \(y \le 1.5x\)  
     - \(x, y \ge 0\)  
     - (Optionally: \(x, y \in \mathbb{Z}\) if integer solutions are required)

2. Identify the feasible region:
   - Plot or conceptually examine the three main constraints in the \(x\)-\(y\) plane.
   - The feasible region will be the intersection where all inequalities hold.

3. Locate candidate optimal points:
   - For an LP, the optimum lies at a vertex (corner point) of the feasible region.
   - Compute the intersection points of relevant constraint pairs:
     - \(y = 250\) with \(x + y = 400\).  
     - \(y = 250\) with \(y = 1.5x\) (if feasible).  
     - \(x + y = 400\) with \(y = 1.5x\).  
   - Check which intersections satisfy all constraints.

4. Evaluate the objective:
   - For each feasible corner point, compute total probes \(3x + 2y\).
   - Choose the point with minimum total probes.

5. If integrality is enforced:
   - Check nearest integer points around the continuous optimum if necessary.
   - Confirm which integer pair minimises probes while satisfying constraints.

This provides the full structure needed to build and solve the optimisation model.

## 4. Summary of the optimisation model

\[
\begin{aligned}
\min_{x, y} \quad & 3x + 2y \\
\text{s.t.} \quad 
& y \ge 250, \\
& x + y \ge 400, \\
& y \le 1.5x, \\
& x \ge 0, \ y \ge 0, \\
& (\text{optionally } x, y \in \mathbb{Z}).
\end{aligned}
\]

This is a Linear Programming (LP) problem; with integer requirements, it becomes an Integer Linear Programming (ILP) problem.

---

## 5. Suitable Python libraries

For an LP/ILP of this size, any standard optimisation library is appropriate:

- **PuLP**  
  - Easy to use, pure Python, good for educational or small optimisation tasks.
  - Can model LP/ILP and call CBC solver by default.
- **OR-Tools (Google OR-Tools)**  
  - Very robust, includes linear solver and MIP solver.
- **SciPy (`scipy.optimize.linprog`)**  
  - Good for continuous LP (no integers), but not for integer constraints.

Given the simplicity of this model, **PuLP** or **OR-Tools** are most convenient.

## 6. Example PuLP formulation (continuous version)

```python
import pulp as pl

# Create problem: minimize total probes
prob = pl.LpProblem("Probe_Minimization", pl.LpMinimize)

# Decision variables (continuous, >= 0)
x = pl.LpVariable('salinity_tests', lowBound=0)      # x
y = pl.LpVariable('ph_tests', lowBound=0)            # y

# Objective function: minimize 3x + 2y
prob += 3 * x + 2 * y

# Constraints
prob += y >= 250, "Min_pH_tests"
prob += x + y >= 400, "Min_total_tests"
prob += y <= 1.5 * x, "Balance_constraint"

# Solve
prob.solve(pl.PULP_CBC_CMD(msg=False))

print("Status:", pl.LpStatus[prob.status])
print("Optimal salinity tests (x):", x.value())
print("Optimal pH tests (y):", y.value())
print("Minimum total probes:", pl.value(prob.objective))
```

To enforce integer test counts, declare:

```python
x = pl.LpVariable('salinity_tests', lowBound=0, cat='Integer')
y = pl.LpVariable('ph_tests', lowBound=0, cat='Integer')
```

The same structure applies if you use OR-Tools’ linear or MIP solver; only the API syntax changes.

[CODER]:
import pulp as pl

def main():
    # Create problem: minimize total probes
    prob = pl.LpProblem("Probe_Minimization", pl.LpMinimize)

    # Decision variables (integer, >= 0)
    x = pl.LpVariable('salinity_tests', lowBound=0, cat='Integer')  # x
    y = pl.LpVariable('ph_tests', lowBound=0, cat='Integer')        # y

    # Objective function: minimize 3x + 2y
    prob += 3 * x + 2 * y

    # Constraints
    prob += y >= 250, "Min_pH_tests"
    prob += x + y >= 400, "Min_total_tests"
    prob += y <= 1.5 * x, "Balance_constraint"

    # Solve
    prob.solve(pl.PULP_CBC_CMD(msg=False))

    final_obj = pl.value(prob.objective)
    print(f"FINAL_OBJ: {final_obj}")

if __name__ == "__main__":
    main()
