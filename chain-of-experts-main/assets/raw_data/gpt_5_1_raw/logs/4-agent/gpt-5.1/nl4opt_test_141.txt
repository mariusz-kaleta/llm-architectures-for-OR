1. **Key optimisation goal**  
   - Goal: Maximize the truck owner’s total monthly profit from selling mangos and guavas, given a budget limit and sales constraints.

2. **Suggested model type**  
   - Model: Linear Programming (LP)  
   - Reason: Objective and all constraints are linear in the numbers of mangos and guavas.

3. **Main decision variables**  
   Define decision variables as:
   - \(x\): number of mangos purchased/sold per month  
   - \(y\): number of guavas purchased/sold per month  

   (Implicit assumption: everything purchased is sold; no inventory or wastage.)

4. **Objective function (conceptual)**  
   - Profit per mango = selling profit = \$3  
   - Profit per guava = selling profit = \$4  
   - Objective: Maximize total profit = 3·(number of mangos) + 4·(number of guavas), i.e.  
     - Maximize: \(3x + 4y\)

5. **Key constraints to consider**

   a. **Budget (spending) constraint**  
   - Total purchase cost ≤ \$20,000  
   - Cost per mango = \$5  
   - Cost per guava = \$3  
   - So: 5·(number of mangos) + 3·(number of guavas) ≤ 20,000  
     - Constraint: \(5x + 3y \le 20000\)

   b. **Mango demand bounds**  
   - At least 100 mangos are sold each month  
   - At most 150 mangos are sold each month  
     - Constraints: \(x \ge 100\) and \(x \le 150\)

   c. **Guava relative demand constraint**  
   - Number of guavas sold is at most one-third of the mangos sold  
     - Constraint: \(y \le \frac{1}{3}x\)

   d. **Non-negativity and integrality**  
   - Cannot sell a negative number of fruits  
     - Constraints: \(x \ge 0, \; y \ge 0\)  
   - Realistically, the numbers of fruits should be integers; depending on modelling requirement, the modeller can:  
     - Start with continuous LP (real-valued x,y) and round as needed, or  
     - Formulate as an Integer Linear Program (ILP) if exact integrality is required.

   - Formulate the LP with above objective and constraints.
   - Work in two dimensions (x,y), so graphical solution is straightforward:
     - Plot feasible region defined by:
       - Budget line: 5x + 3y ≤ 20000  
       - Mango bounds: 100 ≤ x ≤ 150

   - Identify corner (extreme) points of the feasible region:
     - Intersections of:
       - x = 100, x = 150

       - With y = 0 if relevant
   - Evaluate the profit function 3x + 4y at each feasible corner point to find the maximum.

## 1. Decision variables

Let  
- \(x\) = number of mangos purchased/sold per month

We will treat \(x, y\) as continuous in the LP. If you want exact integer counts, we can easily change to integer variables.

## 2. Objective function

- Guava: profit = \$4 per guava

\[

## 4. Full LP / ILP model

**Continuous LP version:**

\[
\begin{aligned}
\max_{x,y}\quad & 3x + 4y \\
\text{s.t.}\quad 
& 5x + 3y \le 20000 \\
& 100 \le x \le 150 \\
& y \le \frac{1}{3}x \\
& x \ge 0,\; y \ge 0
\end{aligned}

**Integer version (if exact counts are required):**

Same as above, plus \(x,y\) integers.

## 6. Example implementation in Python (PuLP)

```python
import pulp as pl

# Problem
prob = pl.LpProblem("Fruit_Profit_Maximization", pl.LpMaximize)

# Decision variables (continuous; use cat="Integer" if integrality is required)
x = pl.LpVariable('mangos', lowBound=0)
y = pl.LpVariable('guavas', lowBound=0)

# Objective function: Maximize 3x + 4y
prob += 3 * x + 4 * y

# Constraints
prob += 5 * x + 3 * y <= 20000          # Budget
prob += x >= 100                        # At least 100 mangos
prob += x <= 150                        # At most 150 mangos
prob += y <= (1.0 / 3.0) * x            # Guavas at most one-third of mangos

# Solve
prob.solve(pl.PULP_CBC_CMD(msg=False))

print("Status:", pl.LpStatus[prob.status])
print("Optimal mangos (x):", x.varValue)
print("Optimal guavas (y):", y.varValue)
print("Maximum profit:", pl.value(prob.objective))
```

To enforce integrality:

```python
x = pl.LpVariable('mangos', lowBound=0, cat="Integer")
y = pl.LpVariable('guavas', lowBound=0, cat="Integer")
```
