[PLANNER]:
1. Key optimisation goal  
   - Goal: Maximise the store owner’s monthly profit from selling carrots and cucumbers, given budget and sales pattern estimates.

2. Suggested model type  
   - Model: Linear Programming (LP) with two decision variables.

3. Main decision variables  
   Define two non‑negative decision variables (continuous or integer; integer is more realistic but not essential for the basic LP structure):
   - Variable 1: Number of carrots sold per month.  
   - Variable 2: Number of cucumbers sold per month.

4. Objective function structure  
   - Profit per carrot = selling profit = 0.75 dollars.  
   - Profit per cucumber = selling profit = 0.80 dollars.  
   - Objective: Maximise total profit = (profit per carrot × number of carrots) + (profit per cucumber × number of cucumbers).

5. Key constraints to consider  

   a) Monthly spending (budget) constraint on purchasing vegetables  
   - Total cost of buying carrots and cucumbers cannot exceed 500 dollars.  
   - Cost per carrot = 0.30 dollars.  
   - Cost per cucumber = 0.50 dollars.  
   - Constraint: “0.30 × carrots + 0.50 × cucumbers ≤ 500”.

   b) Relationship between cucumber and carrot sales  
   - The number of cucumbers sold is at most one third of the number of carrots sold.  
   - Constraint: cucumbers ≤ (1/3) × carrots.

   c) Minimum and maximum carrot sales  
   - At least 300 carrots sold each month: carrots ≥ 300.  
   - At most 500 carrots sold each month: carrots ≤ 500.

   d) Non‑negativity  
   - Carrots ≥ 0, cucumbers ≥ 0.

6. Modelling strategy / solution approach  

   - Since there are only two decision variables and all constraints are linear, this is a simple 2‑variable LP.  
   - The feasible region can be sketched graphically:
     - Use the carrot range [300, 500].  
     - For each feasible carrot value, use the cucumber upper bounds from:
       - Budget: 0.30·carrots + 0.50·cucumbers ≤ 500.  
       - Ratio: cucumbers ≤ (1/3)·carrots.  
     - Cucumbers are bounded by the minimum of these two upper bounds.
   - Compute the coordinates of all corner (extreme) points determined by:
     - Intersections of the lines:  
       - Budget line with carrot = 300 and carrot = 500.  
       - Ratio line with carrot = 300 and carrot = 500.  
       - Intersection of budget line and ratio line (if within carrot range).
   - Evaluate the objective (profit) at each feasible corner point.  
   - Select the carrot and cucumber quantities corresponding to the maximum profit.

## 1. Decision variables

Let

- \(x \ge 0\): number of carrots sold per month  
- \(y \ge 0\): number of cucumbers sold per month  

(They can be treated as continuous for a basic LP, or as integers for a more realistic model. The linear structure is the same.)

---

## 2. Objective function

Profit per unit:

- Profit per carrot: \(0.75\) dollars  
- Profit per cucumber: \(0.80\) dollars  

Maximise total monthly profit:

\[
\max \; Z = 0.75x + 0.80y
\]

---

## 3. Constraints

1. **Budget constraint (purchase cost ≤ \$500)**  
   - Purchase cost per unit:
     - Carrot: \$0.30
     - Cucumber: \$0.50  
   - Total purchase cost per month:

\[
0.30x + 0.50y \le 500
\]

2. **Sales ratio constraint (cucumbers at most one third of carrots)**

\[
y \le \frac{1}{3}x
\]

3. **Carrot sales bounds**

\[
300 \le x \le 500
\]

4. **Non‑negativity**

\[
x \ge 0,\quad y \ge 0
\]

(These are already implied by the other constraints for \(x\), but we state them explicitly for the LP.)

---

## 4. Full LP model

\[
\begin{aligned}
\text{Maximise} \quad & Z = 0.75x + 0.80y \\[4pt]
\text{subject to} \quad 
& 0.30x + 0.50y \le 500 \\[2pt]
& y \le \tfrac{1}{3}x \\[2pt]
& 300 \le x \le 500 \\[2pt]
& x \ge 0,\; y \ge 0
\end{aligned}
\]

This is a standard 2‑variable Linear Program.

## 5. Suitable Python libraries

Any LP solver can handle this easily. Common choices:

1. **PuLP**  
   - High-level Python library for LP/MIP.  
   - Simple syntax, integrates with CBC (default) and other solvers.
   - Good for educational or small/medium problems.

2. **SciPy (`scipy.optimize.linprog`)**  
   - General scientific computing library; `linprog` solves LPs.
   - Good if you’re already using SciPy and want minimal dependencies.

3. **OR‑Tools (Google)**  
   - `ortools.linear_solver.pywraplp` for LP/MIP.
   - Industrial‑strength; supports multiple solvers (CBC, GLOP, SCIP, etc.).
   - Good if you anticipate scaling up or adding integer/binary decisions later.

## 6. Example PuLP formulation (sketch)

```python
import pulp as pl

# Problem
prob = pl.LpProblem("Veg_Profit_Max", pl.LpMaximize)

# Decision variables (continuous; use cat='Integer' for integer version)
x = pl.LpVariable('carrots', lowBound=0)
y = pl.LpVariable('cucumbers', lowBound=0)

# Objective
prob += 0.75 * x + 0.80 * y

# Constraints
prob += 0.30 * x + 0.50 * y <= 500       # budget
prob += y <= (1/3) * x                   # ratio
prob += x >= 300                         # min carrots
prob += x <= 500                         # max carrots

# Solve
prob.solve()
print("Status:", pl.LpStatus[prob.status])
print("Optimal carrots:", x.value())
print("Optimal cucumbers:", y.value())
print("Max profit:", pl.value(prob.objective))
```

This directly encodes the model above and will return the optimal number of carrots and cucumbers to sell per month.

[CODER]:
import pulp as pl

def main():
    # Define the problem
    prob = pl.LpProblem("Veg_Profit_Max", pl.LpMaximize)

    # Decision variables (continuous, non-negative)
    x = pl.LpVariable('carrots', lowBound=0)
    y = pl.LpVariable('cucumbers', lowBound=0)

    # Objective function: Maximize profit
    prob += 0.75 * x + 0.80 * y

    # Constraints
    prob += 0.30 * x + 0.50 * y <= 500        # Budget constraint
    prob += y <= (1.0 / 3.0) * x              # Sales ratio constraint
    prob += x >= 300                          # Minimum carrot sales
    prob += x <= 500                          # Maximum carrot sales

    # Solve the problem
    prob.solve(pl.PULP_CBC_CMD(msg=False))

    # Get the optimal objective value
    optimal_profit = pl.value(prob.objective)

    # Print the final objective value in the required format
    print(f"FINAL_OBJ: {optimal_profit}")

if __name__ == "__main__":
    main()
