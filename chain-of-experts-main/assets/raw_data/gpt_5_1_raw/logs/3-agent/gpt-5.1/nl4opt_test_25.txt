[MODELER]:
Define decision variables:

- Let \(x\) = number of Zodiac pills taken per day  
- Let \(y\) = number of Sunny pills taken per day  

(If pills must be whole, then \(x, y\) are integers; otherwise they can be continuous.)

Objective:  
Minimize total daily cost:
\[
\min \; Z = 1x + 3y
\]

Subject to medicine requirements:

1. Z1 requirement (at least 5 g):

\[
1.3x + 1.2y \;\geq\; 5
\]

2. D3 requirement (at least 10 g):

\[
1.5x + 5y \;\geq\; 10
\]

3. Non-negativity:

\[
x \geq 0,\quad y \geq 0
\]

If you want integer pills:

\[
x, y \in \mathbb{Z}_{\ge 0}
\]

This is a linear programming (LP) problem, or a mixed-integer linear programming (MILP) problem if you enforce integer pills.

Suitable Python libraries:

- `PuLP` – simple interface for LP/MILP, good for teaching and small models.
- `ortools.linear_solver` (Google OR-Tools) – efficient for LP/MILP.
- `scipy.optimize.linprog` – good if you treat \(x, y\) as continuous (no integer requirement).

[CODER]:
import pulp

# Define the problem
prob = pulp.LpProblem("Pill_Optimization", pulp.LpMinimize)

# Decision variables: integer non-negative
x = pulp.LpVariable('x', lowBound=0, cat='Integer')  # Zodiac pills per day
y = pulp.LpVariable('y', lowBound=0, cat='Integer')  # Sunny pills per day

# Objective function: minimize total daily cost
prob += 1 * x + 3 * y

# Constraints
prob += 1.3 * x + 1.2 * y >= 5    # Z1 requirement
prob += 1.5 * x + 5.0 * y >= 10   # D3 requirement

# Solve the problem
prob.solve(pulp.PULP_CBC_CMD(msg=False))

# Extract objective value
final_obj = pulp.value(prob.objective)

# Print final objective value in required format
print(f"FINAL_OBJ: {final_obj}")
