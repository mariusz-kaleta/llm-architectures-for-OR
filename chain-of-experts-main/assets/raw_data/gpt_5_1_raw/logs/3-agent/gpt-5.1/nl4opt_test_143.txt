Here is a clear mathematical optimization model for the problem and how you’d solve it.

---

## 1. Define decision variables

Let

- \(x\) = number of camel caravan trips  
- \(y\) = number of desert truck trips  

Both should be nonnegative integers:

\[
x \ge 0,\quad y \ge 0,\quad x, y \in \mathbb{Z}
\]

---

## 2. Objective: minimize total hours

Each camel trip takes 12 hours, each truck trip takes 5 hours.  
Total hours:

\[
\text{Total Hours} = 12x + 5y
\]

Objective:

\[
\min\; 12x + 5y
\]

---

## 3. Constraints

### 3.1 Demand constraint

Each camel trip delivers 50 units, each truck trip delivers 150 units.  
We must deliver at least 1500 units:

\[
50x + 150y \;\ge\; 1500
\]

(Since both methods deliver in discrete trip sizes, “at least” is appropriate; perfect equality may not be attainable in general problems, but here it will be.)

You can, if you want exactly 1500 units and are sure it’s feasible, set:

\[
50x + 150y = 1500
\]

In this specific problem, that equality is consistent and simplifies the algebra.

### 3.2 Preference constraint (more camels than trucks)

The company prefers more camel caravans than trucks:

\[
x > y
\]

For an integer optimization model, you usually encode this as:

\[
x \ge y + 1
\]

---

## 4. Algebraic simplification (optional reasoning)

Using equality for deliveries:

\[
50x + 150y = 1500
\quad\Rightarrow\quad
x + 3y = 30
\quad\Rightarrow\quad
x = 30 - 3y
\]

Substitute this into the preference constraint:

\[
30 - 3y \ge y + 1 
\;\Rightarrow\; 30 - 1 \ge 4y 
\;\Rightarrow\; 29 \ge 4y 
\;\Rightarrow\; y \le 7.25
\]

So integer \(y \in \{0,1,2,3,4,5,6,7\}\).  
Also \(x = 30 - 3y\) must be nonnegative, which it is for these values.

Total hours as a function of \(y\):

\[
\begin{aligned}
\text{Hours}(y) 
&= 12x + 5y 
= 12(30 - 3y) + 5y \\
&= 360 - 36y + 5y 
= 360 - 31y
\end{aligned}
\]

To minimize hours, maximize \(y\) in the allowed set.  
Largest feasible \(y\) is 7:

\[
x = 30 - 3\cdot7 = 9,\quad y = 7
\]

Total hours:

\[
12(9) + 5(7) = 108 + 35 = 143 \text{ hours}
\]

So the optimal plan:

- Camel caravan trips: \(x = 9\)  
- Desert truck trips: \(y = 7\)

---

## 5. Final optimization model

**Decision variables**

- \(x \in \mathbb{Z}_{\ge 0}\): number of camel caravans  
- \(y \in \mathbb{Z}_{\ge 0}\): number of desert trucks  

**Objective**

\[
\min\; 12x + 5y
\]

**Constraints**

\[
\begin{aligned}
50x + 150y &\ge 1500 &\text{(demand)} \\
x &\ge y + 1 &\text{(more camels than trucks)} \\
x, y &\in \mathbb{Z}_{\ge 0} &\text{(integrality, nonnegativity)}
\end{aligned}
\]

---

## 6. Suitable Python libraries

This is a small **integer linear programming** (ILP) problem.

Recommended libraries:

- **PuLP** (easy to use, pure Python interface to CBC or other solvers)
- **OR-Tools** (`ortools.linear_solver` for MIP; very fast and well-maintained)
- **Pyomo** (more general modeling framework, a bit more verbose)
- **SciPy** is less suitable here because:
  - `scipy.optimize.linprog` does not directly handle integer variables
  - You would have to add integer handling manually or rely on rounding (not recommended)

For teaching or small problems, **PuLP** or **OR-Tools** are ideal.

import pulp

def solve_camel_truck_problem():
    # Create the problem: minimize total hours
    prob = pulp.LpProblem("Camel_Truck_Trips", pulp.LpMinimize)

    # Decision variables: nonnegative integers
    x = pulp.LpVariable('x', lowBound=0, cat='Integer')  # camel trips
    y = pulp.LpVariable('y', lowBound=0, cat='Integer')  # truck trips

    # Objective: minimize 12x + 5y
    prob += 12 * x + 5 * y, "Total_Hours"

    # Constraints
    # Demand: at least 1500 units delivered
    prob += 50 * x + 150 * y >= 1500, "Demand"

    # Preference: more camels than trucks (x >= y + 1)
    prob += x >= y + 1, "More_Camels_Than_Trucks"

    # Solve
    prob.solve(pulp.PULP_CBC_CMD(msg=False))

    # Extract optimal objective value
    final_obj = pulp.value(prob.objective)

    return final_obj

if __name__ == "__main__":
    value = solve_camel_truck_problem()
    print(f"FINAL_OBJ: {value}")
