Let’s turn the description into a precise optimization model and then solve it.

---

## 1. Decision variables

Let  
- \(x\) = number of liquid hand sanitizers produced  
- \(y\) = number of foam hand sanitizers produced  

We assume \(x, y \ge 0\) and (since they are “number of sanitizers”) typically integer, but this is a linear program either way.

---

## 2. Objective function

Each liquid sanitizer cleans 30 hands; each foam sanitizer cleans 20 hands.

Maximize the total number of hands cleaned:
\[
\max Z = 30x + 20y
\]

---

## 3. Constraints

### (a) Water constraint

- Liquid uses 40 units of water per unit.
- Foam uses 60 units of water per unit.
- Total available water is 2000 units.

\[
40x + 60y \le 2000
\]

### (b) Alcohol constraint

- Liquid uses 50 units of alcohol per unit.
- Foam uses 40 units of alcohol per unit.
- Total available alcohol is 2100 units.

\[
50x + 40y \le 2100
\]

### (c) Foam exceeds liquid

The number of foam sanitizers must exceed the number of liquid sanitizers:

\[
y > x
\]

In linear programming solvers we typically encode this as:
\[
y \ge x + \epsilon
\]
for a very small \(\epsilon > 0\), or (if integer variables) as:
\[
y \ge x + 1
\]

### (d) Limit on liquid sanitizers

At most 30 liquid sanitizers:

\[
x \le 30
\]

### (e) Non-negativity

\[
x \ge 0, \quad y \ge 0
\]

And optionally integrality (if we require whole units):

\[
x, y \in \mathbb{Z}_{\ge 0}
\]

---

## 4. Solve analytically (continuous version)

Consider the key resource constraints:

1. Water: \(40x + 60y \le 2000\)
2. Alcohol: \(50x + 40y \le 2100\)
3. \(x \le 30\)
4. \(y > x\)

Check the alcohol constraint at \(x = 30\) (max allowed):

\[
50(30) + 40y \le 2100 \Rightarrow 1500 + 40y \le 2100 \Rightarrow 40y \le 600 \Rightarrow y \le 15
\]

Check the water constraint at \(x = 30\):

\[
40(30) + 60y \le 2000 \Rightarrow 1200 + 60y \le 2000 \Rightarrow 60y \le 800 \Rightarrow y \le \frac{800}{60} \approx 13.33
\]

So with \(x = 30\), we must have:
\[
y \le 13.33
\]
But we also need \(y > x = 30\). This is impossible. Hence, the optimum cannot be at \(x = 30\).

We need \(y > x\), and the alcohol constraint tends to be tighter. Try setting the alcohol constraint to equality and using \(y = x + k\) with \(k > 0\):

\[
50x + 40y = 2100
\]
Substitute \(y = x + k\):
\[
50x + 40(x + k) = 2100 \Rightarrow 90x + 40k = 2100 \Rightarrow x = \frac{2100 - 40k}{90}
\]

To maximize the objective \(Z = 30x + 20y = 30x + 20(x + k) = 50x + 20k\).

Because the coefficients of \(x\) and \(k\) are both positive in the objective, we want \(x\) as large as possible, while respecting:
- Water: \(40x + 60y \le 2000\)
- \(y > x\) ⇒ \(k > 0\)
- \(x \le 30\)

Instead, we can directly solve with the equality \(50x + 40y = 2100\) and the water constraint for equality at optimum (typical for LP corner point):

Solve:
\[
\begin{cases}
40x + 60y = 2000 \\
50x + 40y = 2100
\end{cases}
\]

Multiply first equation by 2:
\[
80x + 120y = 4000
\]
Multiply second equation by 3:
\[
150x + 120y = 6300
\]
Subtract:
\[
(150x + 120y) - (80x + 120y) = 6300 - 4000 \Rightarrow 70x = 2300 \Rightarrow x = \frac{2300}{70} = \frac{230}{7} \approx 32.857
\]

This violates \(x \le 30\), so that intersection is infeasible.

Thus the optimum must lie:
- On \(x = 30\), or
- On \(y = x\), or
- On one resource constraint boundary,

but we already saw \(x = 30\) fails \(y > x\). So consider the boundary \(y = x\):

1. Substitute \(y = x\) into water:
   \[
   40x + 60x \le 2000 \Rightarrow 100x \le 2000 \Rightarrow x \le 20
   \]
2. Substitute \(y = x\) into alcohol:
   \[
   50x + 40x \le 2100 \Rightarrow 90x \le 2100 \Rightarrow x \le \frac{2100}{90} = \frac{70}{3} \approx 23.33
   \]

The tighter bound is \(x \le 20\). On this line, the objective is:
\[
Z = 30x + 20x = 50x
\]
So to maximize, choose the largest feasible \(x\) on \(y = x\), which is \(x = 20\), \(y = 20\). But the constraint is \(y > x\), not \(y \ge x\); so we move slightly above this line.

Take \(x = 20\), \(y = 20 + \epsilon\) for a tiny \(\epsilon > 0\). Check constraints:

- Water:
  \[
  40(20) + 60(20 + \epsilon) = 800 + 1200 + 60\epsilon = 2000 + 60\epsilon
  \]
  This exceeds 2000 for any \(\epsilon > 0\).

Thus, we cannot strictly exceed \(y > x\) at \(x = 20\). So the actual feasible region with \(y > x\) will have its best point just below \(x = 20\). But if we require integer \(x,y\), the next candidate is:

Try \(x = 19, y = 20\):

- Water:
  \[
  40(19) + 60(20) = 760 + 1200 = 1960 \le 2000
  \]
- Alcohol:
  \[
  50(19) + 40(20) = 950 + 800 = 1750 \le 2100
  \]
- \(y > x\): \(20 > 19\) satisfied
- \(x \le 30\) satisfied

Objective:
\[
Z = 30(19) + 20(20) = 570 + 400 = 970
\]

Check if increasing \(x\) while keeping \(y > x\) yields more:

Try \(x = 20, y = 21\):

- Water:
  \[
  40(20) + 60(21) = 800 + 1260 = 2060 > 2000
  \]
Infeasible.

Try more foam-dominant:

- \(x = 18, y = 21\):
  - Water: \(40(18) + 60(21) = 720 + 1260 = 1980 \le 2000\)
  - Alcohol: \(50(18) + 40(21) = 900 + 840 = 1740 \le 2100\)
  - Objective: \(30(18) + 20(21) = 540 + 420 = 960\) < 970

- \(x = 17, y = 22\):
  - Water: \(40(17) + 60(22) = 680 + 1320 = 2000\)
  - Alcohol: \(50(17) + 40(22) = 850 + 880 = 1730 \le 2100\)
  - Objective: \(30(17) + 20(22) = 510 + 440 = 950\) < 970

So among reasonable integer points, \((x,y) = (19,20)\) gives the highest objective value.

Thus, under integer decision variables:

- Optimal solution:  
  - 19 liquid sanitizers  
  - 20 foam sanitizers  
- Maximum hands cleaned:  
  \[
  30 \cdot 19 + 20 \cdot 20 = 570 + 400 = 970
  \]

import pulp

# Create the problem: maximize total hands cleaned
prob = pulp.LpProblem("Hand_Sanitizer_Production", pulp.LpMaximize)

# Decision variables (integers, non-negative)
x = pulp.LpVariable('x', lowBound=0, cat='Integer')  # liquid
y = pulp.LpVariable('y', lowBound=0, cat='Integer')  # foam

# Objective function: maximize 30x + 20y
prob += 30 * x + 20 * y

# Constraints
prob += 40 * x + 60 * y <= 2000      # Water constraint
prob += 50 * x + 40 * y <= 2100      # Alcohol constraint
prob += y >= x + 1                   # Foam exceeds liquid (strictly, via integer model)
prob += x <= 30                      # Limit on liquid

# Solve the problem
prob.solve(pulp.PULP_CBC_CMD(msg=False))

# Get the objective value
final_obj = pulp.value(prob.objective)

print(f"FINAL_OBJ: {final_obj}")
