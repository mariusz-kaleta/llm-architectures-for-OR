[MODELER]:
Formal model

Decision variables
- T = dollars invested in telecom (continuous, T ≥ 0)
- H = dollars invested in healthcare (continuous, H ≥ 0)

Objective
- Maximise annual return: maximize R = 0.03 T + 0.01 H

Constraints
1. Budget: T + H ≤ 100000
2. Relative allocation: T ≥ 3 H
3. Telecom upper bound: T ≤ 70000
4. Non-negativity: T ≥ 0, H ≥ 0

Solution (analysis)
- Telecom gives higher return per dollar (3% > 1%), so we want T as large as allowed (T = 70000) and then set H as large as allowed by the T ≥ 3H constraint and the budget.
- With T = 70000, the ratio constraint implies H ≤ T/3 = 70000/3 ≈ 23333.333. The budget allows H ≤ 30000, so the tighter is H ≤ 70000/3.
- Set H = 70000/3 ≈ 23333.333. This gives T + H = 70000 + 70000/3 = 70000*(4/3) = 93333.333 ≤ 100000, so the budget constraint is not binding.

Optimal solution
- T* = $70,000
- H* = 70,000 / 3 ≈ $23,333.33
- Annual return R* = 0.03·70000 + 0.01·(70000/3) = 2100 + 700/3 = 7000/3 ≈ $2,333.33

Which constraints bind
- Telecom upper bound: T = 70000 (binding)
- Relative allocation: T = 3H (binding)
- Budget: not binding (T + H = 93,333.33 < 100,000)

Why this is optimal (brief)
- Telecom has strictly higher marginal return, so increase T up to its upper bound. Any remaining feasible increase in H raises the objective (positive coefficient 0.01), so set H to the maximum allowed by T ≥ 3H. Other candidate corners (e.g., T=70000,H=0 or trying to use the full budget under the ratio constraint would require T = 75,000 which violates T ≤ 70,000) give strictly lower return.

Solving programmatically
Recommended Python libraries:
- pulp (simple LP modeling; uses CBC by default)
- scipy.optimize.linprog
- ortools.linear_solver (Google OR-Tools)
- cvxpy (convex optimization modeling)

Minimal example using pulp
from pulp import LpProblem, LpMaximize, LpVariable, lpSum, value

prob = LpProblem("Invest", LpMaximize)
T = LpVariable("T", lowBound=0)
H = LpVariable("H", lowBound=0)
prob += 0.03*T + 0.01*H
prob += T + H <= 100000
prob += T >= 3*H
prob += T <= 70000
prob.solve()
print(value(T), value(H), value(prob.objective))

This will return T = 70000, H ≈ 23333.333, objective ≈ 2333.333.

If you want integer-dollar investments, you can add integer constraints (but the problem as stated is continuous).
