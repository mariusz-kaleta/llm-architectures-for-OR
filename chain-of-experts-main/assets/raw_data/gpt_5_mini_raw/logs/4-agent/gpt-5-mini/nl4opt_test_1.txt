Formal ILP formulation

Decision variables
- L ∈ Z≥0 : number of large pills produced
- S ∈ Z≥0 : number of small pills produced

Objective
- Minimise total filler: minimize Z = 2·L + 1·S

Constraints
1. Medicinal ingredient availability:
   3·L + 2·S ≤ 1000
2. Minimum large-pill requirement:
   L ≥ 100
3. Popularity proportion (at least 60% small):
   S ≥ 0.6·(L + S)  — which is linearised to S ≥ 1.5·L  (since 0.4S ≥ 0.6L ⇒ S ≥ 1.5L)
4. Integrality and non‑negativity:
   L, S ∈ Z≥0

Thus the ILP is
minimize Z = 2L + S
subject to
  3L + 2S ≤ 1000
  L ≥ 100
  S ≥ 1.5L
  L, S ∈ Z≥0

Solution (analytic and verification)
- From S ≥ 1.5L and L ≥ 100, the smallest feasible L is L = 100, which forces S ≥ 1.5·100 = 150. Take S = 150 (smallest S allowed).
- Check medicinal constraint: 3·100 + 2·150 = 300 + 300 = 600 ≤ 1000 (feasible).
- Objective value (total filler) at (L,S) = (100,150): Z = 2·100 + 1·150 = 350.
- Any increase in L requires S to increase (because S ≥ 1.5L), so total filler increases: increasing L by 1 increases filler by at least 2 + 1.5·1 = 3.5 (in LP terms) and in integer terms will increase filler by an integer ≥ 3. So no feasible (L,S) with L > 100 can give smaller filler. Reducing S is not allowed by the proportion constraint. Therefore (L,S) = (100,150) with total filler 350 is globally optimal.

Additional insight
- Filler per medicinal unit: large = 2/3 ≈ 0.667, small = 1/2 = 0.5. Small pills use less filler per medicinal unit, so if medicinal availability were tight you’d favour small pills — but the constraints here (L ≥ 100 and the 60% small requirement) determine the minimum production mix.
- Medicinal capacity is not binding at the optimum (600 used of 1000).

# Self-contained script to solve the ILP by simple enumeration
# Decision variables:
#   L >= 100 integer
#   S >= ceil(1.5 * L) integer
# Constraints:
#   3L + 2S <= 1000
# Objective:
#   minimize Z = 2L + S

best_obj = None
best_L = None
best_S = None

# feasible L range: start at 100. Upper bound can be set to 1000//3 (safe), enumeration is small.
for L in range(100, 1000 // 3 + 1):
    # minimum S due to popularity proportion: ceil(1.5 * L) = ceil(3L/2)
    s_min = (3 * L + 1) // 2  # integer ceiling
    # maximum S due to medicinal ingredient: 3L + 2S <= 1000 -> S <= floor((1000 - 3L)/2)
    s_max = (1000 - 3 * L) // 2
    if s_min <= s_max and s_min >= 0:
        S = s_min  # objective coefficient of S is positive so minimal S is best for fixed L
        obj = 2 * L + S
        if best_obj is None or obj < best_obj:
            best_obj = obj
            best_L = L
            best_S = S

# Final output: required exact format
print(f"FINAL_OBJ: {best_obj}")
