- Cookies: profit per oven hour = 3 / 1 = 3

Because cookies give higher profit per oven hour, oven capacity will be used by cookies. Check feasible extreme points (axes and intersection). The feasible region is bounded by the oven constraint and axes (the mixer constraint is not restrictive in the positive quadrant here). The extreme points are:
- (x,y) = (0,0) → Z = 0
- (x,y) = (1000,0) (oven binds: 3x = 3000) → Z = 5·1000 = 5000
- (x,y) = (0,3000) (oven binds: y = 3000) → Z = 3·3000 = 9000

Optimal solution: produce only cookies:
- x* = 0 loaves, y* = 3000 batches of cookies

#!/usr/bin/env python3
import math

# Problem data
# Maximise Z = 5x + 3y
# s.t.
#   x + 0.5 y <= 3000  (mixer)
#   3x + y <= 3000     (oven)
#   x >= 0, y >= 0

tol = 1e-9

# Define constraint lines (as functions to compute intersections)
# Lines: x + 0.5 y = 3000  and  3x + y = 3000
# We'll consider intersections of these with axes and with each other,
# plus the origin.

candidates = []

# origin
candidates.append((0.0, 0.0))

# Intersection of oven with x-axis (y=0 -> 3x=3000)
candidates.append((3000.0/3.0, 0.0))  # (1000,0)

# Intersection of oven with y-axis (x=0 -> y=3000)
candidates.append((0.0, 3000.0))

# Intersection of mixer with x-axis (y=0 -> x=3000)
candidates.append((3000.0, 0.0))

# Intersection of mixer with y-axis (x=0 -> 0.5 y = 3000 -> y=6000)
candidates.append((0.0, 6000.0))

# Intersection of the two constraint lines:
# Solve:
#   x + 0.5 y = 3000
#   3x + y = 3000
# Use linear algebra
# From second: y = 3000 - 3x
# Substitute: x + 0.5*(3000 - 3x) = 3000
# => x + 1500 - 1.5x = 3000 => -0.5x = 1500 => x = -3000, y = 3000 - 3*(-3000) = 12000
candidates.append((-3000.0, 12000.0))

# Evaluate feasibility and objective
best_obj = -math.inf
best_point = None

for x, y in candidates:
    # Check non-negativity
    if x < -tol or y < -tol:
        continue
    # Check constraints
    if x + 0.5*y <= 3000 + tol and 3*x + y <= 3000 + tol:
        obj = 5*x + 3*y
        if obj > best_obj + tol:
            best_obj = obj
            best_point = (x, y)

# As a fallback (shouldn't be needed), also check integer corner produced by oven constraint when setting x=0
# (already included). If best_obj still -inf, set to 0 at origin.
if best_obj == -math.inf:
    best_obj = 0.0
    best_point = (0.0, 0.0)

# If the objective is very close to an integer, cast to int for nicer output
if abs(best_obj - round(best_obj)) < 1e-6:
    best_obj_to_print = int(round(best_obj))
else:
    best_obj_to_print = best_obj

# Final required print
print(f"FINAL_OBJ: {best_obj_to_print}")
