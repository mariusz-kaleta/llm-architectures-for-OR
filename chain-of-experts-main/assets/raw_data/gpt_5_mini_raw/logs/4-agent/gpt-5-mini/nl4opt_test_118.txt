Minimise total transport time:

4) Constraints (algebraic)
- Capacity: 30 h + 20 c ≥ 300
- Helicopter upper bound: 0 ≤ h ≤ 5
- Car-share (at least 60% of trips are by car):
  c ≥ 0.6 (h + c)  which rearranges to 0.4 c ≥ 0.6 h  →  c ≥ 1.5 h

- Integrality and nonnegativity: h, c ∈ Z≥0

5) Small-domain solution method (recommended)
Because h is bounded 0..5, it is simplest and exact to enumerate h = 0..5 and for each take the smallest integer c that satisfies both:
  c ≥ ceil(1.5 h) and c ≥ ceil((300 − 30 h)/20) (if RHS ≤ 0 then use 0).
Compute T = 40 h + 30 c and pick the minimum feasible pair.

Minimum total time is 430 minutes at h = 4, c = 9. This pair transports 30·4 + 20·9 = 300 fish exactly and satisfies c ≥ 1.5·4 = 6 (indeed c = 9).

There are no other (h,c) pairs achieving a lower T in the feasible set, so the optimum is unique: h = 4 helicopter trips, c = 9 car trips, total time 430 minutes, total fish = 300.

[CODER]:
import math

# Parameters
capacity_helicopter = 30
capacity_car = 20
time_helicopter = 40
time_car = 30
required_fish = 300
h_upper = 5

best_T = None
best_h = None
best_c = None

for h in range(0, h_upper + 1):
    # car share constraint: c >= ceil(1.5 * h)
    c_min_share = math.ceil(1.5 * h)
    # capacity constraint: 30*h + 20*c >= 300 -> c >= ceil((300 - 30*h)/20)
    rem = required_fish - capacity_helicopter * h
    c_min_capacity = 0
    if rem > 0:
        c_min_capacity = math.ceil(rem / capacity_car)
    c = max(c_min_share, c_min_capacity)
    # c must be non-negative integer
    if c < 0:
        c = 0
    T = time_helicopter * h + time_car * c
    # store best (minimize T). If tie, keep first found.
    if (best_T is None) or (T < best_T):
        best_T = T
        best_h = h
        best_c = c

# final objective value
print(f"FINAL_OBJ: {best_T}")
