Decision variables
- A = acres planted with apples (continuous, ≥ 0; optionally integer).
- P = acres planted with pears (continuous, ≥ 0; optionally integer).

Objective
- Maximise total profit: maximize Z = 2·A + 4·P.

Constraints
- Total land: A + P ≤ 50.
- Minimum plantings: A ≥ 5, P ≥ 10.
- Workforce ratio (hard): P ≤ 2·A.
- (Soft preference) Ben prefers more pears than apples: P ≥ A — treat as optional (not enforced unless requested).
- Integrality: A, P integer only if acres must be integer.

2) Solve analytically (continuous LP)
At optimum the model will try to allocate as many acres to pears as allowed because pears pay more ($4) than apples ($2). The likely optimal lies at the intersection of the two active constraints:
- A + P = 50
- P = 2·A

Solve simultaneously:
A + 2A = 50 → 3A = 50 → A = 50/3 ≈ 16.6667 acres
P = 2·A = 100/3 ≈ 33.3333 acres

Objective value:
Z* = 2·(50/3) + 4·(100/3) = 500/3 ≈ $166.6667

Check minima: A = 50/3 > 5 and P = 100/3 > 10, so the minimum-acre constraints are not active. The soft preference P ≥ A is satisfied (33.33 ≥ 16.67), so adding it would not change the continuous solution.

3) Integer-acres variant
Solve the integer program (A, P integers). Search or reason with constraints:
- With A + P ≤ 50 and P ≤ 2A, the sum is maximised for integer A that allows the largest feasible P.
- The integer solution that maximises profit is A = 17, P = 33 (A+P = 50; P ≤ 2A holds because 33 ≤ 34).

Profit (integer solution):
Z_int = 2·17 + 4·33 = 34 + 132 = $166.00

This is the integer-optimal allocation; it is very close to the continuous optimum (loss of $0.6667).

4) Which constraints are binding
- Continuous optimum: A + P = 50 (land) and P = 2A (workforce) are both binding. Minimum bounds are inactive.
- Integer optimum: A + P = 50 (land) binds; P ≤ 2A is not tight (33 < 34), minimum bounds inactive.

5) Sensitivity (short)

```python
# Self-contained script to solve the apple/pear planting LP (continuous) and its integer variant.
# It computes:
#  - Continuous LP optimum (analytic)
#  - Integer (A,P integer) optimum by enumeration
# At the very end it prints the final objective value in the required format.

# Problem data (from the model)
total_land = 50
min_A = 5
min_P = 10

# Objective coefficients
coef_A = 2.0
coef_P = 4.0

# Workforce constraint: P <= 2*A

# 1) Continuous solution (analytical): solve A + P = 50 and P = 2*A
A_cont = total_land / 3.0              # 50 / 3
P_cont = 2.0 * A_cont                  # 100 / 3
obj_cont = coef_A * A_cont + coef_P * P_cont

# 2) Integer solution (enumeration)
best_int_obj = None
best_int_solution = None

# A must be at least min_A and P at least min_P and A+P <= total_land
# search integer A in [min_A, total_land - min_P] and P accordingly
for A in range(int(min_A), int(total_land) + 1):
    # feasible P range given land and min_P
    P_min_feasible = int(min_P)
    P_max_feasible = int(total_land - A)
    if P_max_feasible < P_min_feasible:
        continue
    for P in range(P_min_feasible, P_max_feasible + 1):
        # workforce constraint
        if P <= 2 * A:
            obj = coef_A * A + coef_P * P
            if (best_int_obj is None) or (obj > best_int_obj):
                best_int_obj = obj
                best_int_solution = (A, P)

# Print summary results (additional prints allowed before the final required line)
print("Continuous (analytic) optimum:")
print(f"  A = {A_cont:.6f} acres (apples)")
print(f"  P = {P_cont:.6f} acres (pears)")
print(f"  Objective (continuous) = {obj_cont:.6f}")

if best_int_solution is not None:
    A_int, P_int = best_int_solution
    print("Integer optimum (enumeration):")
    print(f"  A = {A_int} acres (apples)")
    print(f"  P = {P_int} acres (pears)")
    print(f"  Objective (integer) = {best_int_obj:.6f}")
else:
    print("No integer feasible solution found under the given constraints.")

# Final required print line (must be the very last line)
# We choose to report the continuous optimum objective value as the final objective.
print(f"FINAL_OBJ: {obj_cont}")
```
