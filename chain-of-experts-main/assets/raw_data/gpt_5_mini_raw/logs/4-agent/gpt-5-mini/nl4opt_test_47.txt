- Model: Mixed-Integer Linear Program (MILP). The problem is linear with small integer decision variables (counts of fixtures), so a MILP solver or simple integer enumeration is appropriate.

- L = number of LED fixtures to install (integer, ≥ 0).
- F = number of fluorescent fixtures to install (integer, ≥ 0).

4. Objective (description for modeller)
- Minimise total changes over a decade = (changes per LED) * L + (changes per fluorescent) * F.
  - LED changes = 3 per decade.
  - Fluorescent changes = 4 per decade.
  So objective is to minimise 3*L + 4*F (express as a linear objective).

5. Key constraints to include
- Minimum total fixtures: L + F ≥ 300 (customer requirement).
- Electricity use limit: 5*L + 8*F ≤ 2000 (electricity units per hour cap).

- Integrality and non-negativity: L, F are integer and ≥ 0.
- Practical variable bounds (useful for solver efficiency):
  - L ≤ floor(2000/5) = 400 (electricity cap alone).
  - F ≤ floor(2000/8) = 250.

- Since objective coefficients are positive, the solver will prefer fewer total fixtures; therefore the minimum total fixtures constraint will bind (choose N = 300) unless electricity/proportion constraints force a higher N. You can therefore first fix N = 300 and solve for L and F to simplify (still keep integrality).
- Because LEDs have lower change rate and lower electricity use, the objective will push to maximize L subject to the fluorescence proportion and electricity cap. Expect an optimal solution at the minimum allowed F (from the 30% rule) provided the electricity cap is not violated by that choice.

#!/usr/bin/env python3
# Self-contained solver for the MILP described:
# Minimize Z = 3L + 4F
# subject to:
#   L + F >= 300
#   5L + 8F <= 2000
#   F >= 0.30*(L+F)  <=>  7F >= 3L
#   L, F integers >= 0
#
# Brute-force search over practical bounds L <= 400, F <= 250.

def main():
    L_bound = 400
    F_bound = 250

    best_obj = None
    best_solutions = []

    for F in range(F_bound + 1):
        # For given F, L must be at least max(0, 300 - F) to meet N>=300
        L_min = max(0, 300 - F)
        # and at most L_bound
        for L in range(L_min, L_bound + 1):
            # Electricity constraint
            if 5 * L + 8 * F > 2000:
                continue
            # Fluorescent share: 7F >= 3L
            if 7 * F < 3 * L:
                continue
            # All constraints satisfied; compute objective
            Z = 3 * L + 4 * F
            if (best_obj is None) or (Z < best_obj):
                best_obj = Z
                best_solutions = [(L, F)]
            elif Z == best_obj:
                best_solutions.append((L, F))

    if best_obj is None:
        print("No feasible solution found.")
        value = None
    else:
        # Choose the first best solution for reporting
        L_opt, F_opt = best_solutions[0]
        N_opt = L_opt + F_opt
        electricity = 5 * L_opt + 8 * F_opt
        fluorescent_share = F_opt / N_opt if N_opt > 0 else 0.0

        print(f"Optimal solutions (may be multiple). Best objective Z = {best_obj}")
        print("Example optimal solution:")
        print(f"  L (LED fixtures)        = {L_opt}")
        print(f"  F (Fluorescent fixtures)= {F_opt}")
        print(f"  Total fixtures N        = {N_opt}")
        print(f"  Electricity consumption = {electricity} (<= 2000)")
        print(f"  Fluorescent share       = {fluorescent_share:.4f} (>= 0.30)")
        value = best_obj

    # Final required line (must be the last line printed)
    print(f"FINAL_OBJ: {value}")

if __name__ == "__main__":
    main()
