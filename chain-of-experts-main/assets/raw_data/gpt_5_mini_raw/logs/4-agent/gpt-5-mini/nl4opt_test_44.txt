- Goal: Maximise the total number of bottles produced (vintage + regular).

2. Suggested model type
- Model: Integer Linear Programming (ILP) — two integer decision variables with linear constraints and a linear objective. (Because bottles are discrete, integrality is required. For quick checks one can also do simple discrete enumeration.)

3. Main decision variables
- v = number of vintage bottles produced (integer, >= 0)
- r = number of regular bottles produced (integer, >= 0)

4. Key constraints to include
- Volume capacity: Total volume used by v and r cannot exceed available 100000 ml (each vintage uses 500 ml, each regular 750 ml).
- Ratio constraint: The number of regular bottles must be at least four times the number of vintage bottles (r >= 4·v).
- Minimum vintage: At least 10 vintage bottles must be produced (v >= 10).
- Integrality and nonnegativity: v and r must be integer and nonnegative.
- Practical bounds (optional but useful for enumeration/solver): v ≤ floor(100000/500) = 200, r ≤ floor(100000/750) = 133 (these tighten search/solver domains).

- Simple enumeration: Iterate v from 10 up to its upper bound (200). For each v, iterate r from ceil(4v) up to floor((100000 − 500·v)/750) and track the feasible (v,r) that gives the largest v + r. This is fast because variable ranges are small.
- Relaxation check: Solve the LP relaxation (drop integrality) to get an upper bound and guide search; however final answer must be integer.
- Modeling tip: Because regulars must be ≥ 4·v, feasible solutions are constrained; ensure when adding extra bottles beyond minimum r you still satisfy r ≥ 4·v.

#!/usr/bin/env python3
# Self-contained script to solve the integer optimization by enumeration

def main():
    capacity = 100000  # ml
    # Coefficients
    vol_v = 500
    vol_r = 750

    best_obj = None
    best_solutions = []

    # v in {10,...,28} as derived
    for v in range(10, 29):
        # r = 4*v + k, with k >= 0 and 3500*v + 750*k <= capacity
        remaining_after_base = capacity - 3500 * v
        if remaining_after_base < 0:
            continue
        k_max = remaining_after_base // 750
        r = 4 * v + k_max
        # Verify feasibility
        if r < 0:
            continue
        if vol_v * v + vol_r * r > capacity:
            continue
        obj = v + r
        if (best_obj is None) or (obj > best_obj):
            best_obj = obj
            best_solutions = [(v, r)]
        elif obj == best_obj:
            best_solutions.append((v, r))

    # At least one feasible solution should exist
    if best_obj is None:
        best_obj = 0

    # Final required output line (no other text on that line)
    print(f"FINAL_OBJ: {best_obj}")

if __name__ == "__main__":
    main()
