2) Suggested model type
- Model: Mixed-Integer Linear Program (MILP).  
  Reason: trip counts are integer variables (you cannot make a fractional trip) and a strict ordering constraint ("dog trips must be less than truck trips") is easiest to enforce with integer variables. If you must use a pure continuous LP, you can relax integrality but you will need to round results (may become infeasible or suboptimal).

3) Main decision variables
- Nd = number of sled-dog trips (integer, >= 0).  
- Nt = number of truck trips (integer, >= 0).  
- (Optional convenience variable) F = total fish transported (continuous, >= 0), where F = Nd * DogCapability + Nt * TruckCapability. You can also compute the objective directly from Nd and Nt.

(Use the concrete data: DogCapability = 100 fish/trip, TruckCapability = 300 fish/trip, DogCost = 50 per trip, TruckCost = 100 per trip, MaxBudget = 1000.)

4) Key constraints to include (conceptual description)
- Budget constraint: total transport cost cannot exceed MaxBudget. Form the cost as DogCost * Nd + TruckCost * Nt <= MaxBudget.
- Capacity -> objective link: total fish moved is capacity per trip times trips (F = DogCapability * Nd + TruckCapability * Nt, or compute directly in objective).
- Ordering constraint (problem-specific): the number of dog trips must be less than the number of truck trips. Represent this as Nd <= Nt - 1. Note: this enforces a strict “less than” when Nd, Nt are integers. If you use continuous variables you cannot enforce strict inequality; you'd have to use Nd <= Nt - epsilon or change the rule.

- Implicit upper bounds from budget (useful for solver efficiency): Nd <= floor(MaxBudget / DogCost) and Nt <= floor(MaxBudget / TruckCost). These are not additional logical constraints (they follow from budget) but help define variable bounds.
- Feasibility check: because Nd <= Nt - 1, if budget does not allow at least one truck trip (Nt >= 1), any positive Nd will be infeasible. Ensure the model allows Nt = 0 and Nd = 0 as a feasible solution, or decide to require at least one truck trip explicitly if that matches business rules.

- Maximise F = DogCapability * Nd + TruckCapability * Nt (or directly maximise DogCapability * Nd + TruckCapability * Nt).

- If you must produce a pure LP solution, relax integrality, solve, then round down Nd and Nt to integers and re-check budget and ordering constraints; if violated, adjust using small combinatorial search (this is suboptimal compared to an MILP).

# Self-contained script to solve the MILP by direct enumeration (no external solvers required)

# Parameters
DogCapability = 100
TruckCapability = 300
DogCost = 50
TruckCost = 100
MaxBudget = 1000

# Implicit bounds
Nd_max = MaxBudget // DogCost  # 20
Nt_max = MaxBudget // TruckCost  # 10

best_obj = None
best_Nd = None
best_Nt = None

for Nt in range(0, Nt_max + 1):
    for Nd in range(0, Nd_max + 1):
        # Constraints
        if 50 * Nd + 100 * Nt > MaxBudget:
            continue
        if Nd > Nt - 1:
            continue
        # Objective
        obj = DogCapability * Nd + TruckCapability * Nt
        if (best_obj is None) or (obj > best_obj):
            best_obj = obj
            best_Nd = Nd
            best_Nt = Nt

# If no feasible solution found, set objective to 0
if best_obj is None:
    best_obj = 0

print(f"FINAL_OBJ: {int(best_obj)}")
