C ≤ 0.4(B + C) ⇒ 0.6C ≤ 0.4B ⇒ C ≤ (2/3) B.
Thus C ≤ floor((2/3) B) when enforcing integrality.

Substitute the upper bound on C into capacity to get a continuous lower bound on B:
3B + 5·(2/3)B ≥ 500 ⇒ (3 + 10/3)B = (19/3)B ≥ 500 ⇒ B ≥ 1500/19 ≈ 78.947.
So integer B must be at least 79.

5) Small discrete search (optimality logic)
For a fixed B, capacity is maximized by taking C as large as allowed, C = floor((2/3)B). Therefore to test feasibility for minimizing B, iterate B upward from ceil(1500/19) and for each B set C = floor((2/3)B) and check 3B + 5C ≥ 500. The first B that satisfies the capacity constraint with C = floor((2/3)B) is the optimal (minimal) B.

import math

# Problem data
demand = 500           # people to transport
bike_capacity = 3      # people per bike
car_capacity = 5       # people per car
max_car_fraction = 0.4 # at most 40% of vehicles may be cars

# Derived exact upper bound on C: C <= (2/3) B (from C <= 0.4*(B+C))
# So for a given B, best to take C = floor((2/3)*B)
# Compute a starting B from continuous relaxation:
start_B = math.ceil(demand * 3 / 19)  # ceil(1500/19) = 79

solution = None
# Search upward from start_B until a feasible solution is found
for B in range(start_B, 10000):
    C = (2 * B) // 3  # floor((2/3)*B)
    capacity = bike_capacity * B + car_capacity * C
    # Check capacity constraint
    if capacity >= demand:
        solution = (B, C, capacity)
        break

if solution is None:
    raise RuntimeError("No feasible solution found in search range.")

B_opt, C_opt, capacity_opt = solution

# Final objective is to minimize B, so objective value is B_opt
final_obj = B_opt

# Output as required
print(f"FINAL_OBJ: {final_obj}")
