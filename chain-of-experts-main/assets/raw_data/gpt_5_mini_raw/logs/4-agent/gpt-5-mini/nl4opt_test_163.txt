3) Constraints (linear form)
- Customer-interaction requirement:
70 C + 85 H ≥ 4000

- Cart-share requirement C/(C+H) ≥ 0.70, linearized:
C ≥ 0.70 (C + H)  →  0.30 C ≥ 0.70 H  →  multiply by 10 → 3 C − 7 H ≥ 0

- Minimum hand servers:
H ≥ 3

C, H ∈ Z_{\ge 0}

4) Practical tightening (bounds)
- Lower bound on total servers: ceil(4000 / max(ic, ih)) = ceil(4000 / 85) = 48
- Upper bound on total servers: ceil(4000 / min(ic, ih)) = ceil(4000 / 70) = 58

Thus 48 ≤ C + H ≤ 58 (helps search). From H ≥ 3 and C ≥ 0 we get H ∈ [3, 58−0] but tighter bounds can be derived per enumeration below.

5) Solving approach (recommended)
Because there are only two integer variables, a tiny enumeration is simplest and fastest:
- Loop H from H_min upward (H_min = 3) while H ≤ UB_total − 0 (UB_total = 58).
- For each H compute the smallest integer C that satisfies both:
  a) 70 C + 85 H ≥ 4000  →  C ≥ ceil((4000 − 85 H) / 70)
  b) 3 C − 7 H ≥ 0 → C ≥ ceil(7 H / 3)
  Take C = max(0, ceil((4000 − 85 H)/70), ceil(7 H / 3)).
- Compute Z = 5 C + 20 H. Track minimum Z (optionally track ties).
This is O(UB_total) steps (≈ 58 iterations) and trivial to implement.

6) Continuous relaxation (useful bound)
Relaxing integrality gives lower bound. For H fixed at its lower bound 3, continuous C = 3745/70 = 53.5 yields continuous objective 5*53.5 + 20*3 = 327.5. The integer solution will be ≥ this.

7) Optimal integer solution (enumeration / analytic reasoning)
- With H forced minimal (H = 3) you need C ≥ ceil((4000 − 85·3)/70) = ceil(3745/70) = 54 and also C ≥ ceil(7·3/3) = 7. So the interaction constraint dominates: C = 54.
- Refills Z = 5·54 + 20·3 = 270 + 60 = 330.

Check small variations:
- H = 4 → minimal C = max(ceil((4000 − 340)/70)=53, ceil(28/3)=10) = 53 → Z = 5·53 + 20·4 = 345 (>330)
- Any further increases in H raise Z because exchanging one cart for one hand increases refills by 15 (−5 + 20), so H = 3 is optimal.

Therefore global optimum (among integers satisfying all constraints):
- C* = 54 cart servers
- H* = 3 hand servers
- Minimum total refills per hour Z* = 330

8) Feasibility and sensitivity notes
- Feasible: Yes (C=54, H=3 satisfies interactions and 70% cart-share).
- Continuous relaxation lower bound = 327.5; integer optimum = 330 (small integrality gap).
- Sensitivity:

  - Relaxing the 70% cart-share to a lower fraction can only (weakly) increase refills because it allows more hand servers (worse per-interaction refill rate). Raising the cart-share requirement (if feasible with H_min) tends to reduce refills (since carts have much lower refills per interaction), but the effect depends on H_min and interaction requirement.

#!/usr/bin/env python3
import math

# Parameters
ic = 70          # interactions per cart server
rc = 5           # refills per cart server
ih = 85          # interactions per hand server
rh = 20          # refills per hand server
I_req = 4000     # required interactions
alpha = 0.70     # required cart-share (not used directly; linearized constraint used)
H_min = 3        # minimum hand servers

# Linearized cart-share: 3C - 7H >= 0  -> C >= ceil(7H/3)
# Practical bounds on total servers
LB_total = math.ceil(I_req / max(ic, ih))   # 48
UB_total = math.ceil(I_req / min(ic, ih))   # 58

best = None  # will hold tuple (Z, C, H)

for H in range(H_min, UB_total + 1):
    # Interaction constraint: C >= ceil((I_req - ih*H) / ic)
    num = I_req - ih * H
    if num <= 0:
        C_inter = 0
    else:
        C_inter = (num + ic - 1) // ic  # ceil division

    # Cart-share linearized constraint: C >= ceil(7H/3)
    C_share = (7 * H + 3 - 1) // 3  # (7H + 2)//3

    C = max(0, C_inter, C_share)

    # Enforce practical upper bound on total servers
    if C + H > UB_total:
        continue

    # Verify constraints (safety)
    if 70 * C + 85 * H < I_req:
        continue
    if 3 * C - 7 * H < 0:
        continue
    if H < H_min:
        continue

    Z = rc * C + rh * H  # 5*C + 20*H

    if best is None or Z < best[0] or (Z == best[0] and (C + H) < (best[2] + best[1])):
        # store as (Z, C, H)
        best = (Z, C, H)

# If found, print details and final objective
if best is None:
    raise RuntimeError("No feasible solution found within the search bounds.")

bestZ, bestC, bestH = best

# Print summary (optional)
print(f"Optimal integer solution found: C = {bestC}, H = {bestH}, Z = {bestZ}")
# Final required print (must be last line)
print(f"FINAL_OBJ: {bestZ}")
