4. Integrality and nonnegativity: S, L ∈ Z_+, (integers ≥ 0).

Notes / reformulation
- Because increasing S only increases the objective and S must be at least 2L, for any fixed L the optimal S is the smallest integer satisfying both S ≥ 2L and 20S ≥ 2000 − 50L. Hence
  S(L) = max( 2L, ceil((2000 − 50L)/20, 0) ).
- Therefore the problem reduces to a single integer search over L ≥ 10: compute S(L) and pick L minimizing Z(L)=S(L)+L.

Practical search bounds
- Upper bound for L: using only large wagons would need L ≥ ceil(2000/50)=40, so search L ∈ {10,...,40}. In practice the optimal L is much smaller (see below).
- LP relaxation gives a lower bound: substitute S=2L (tight because increasing S is bad) ⇒ 90L ≥ 2000 ⇒ L ≥ 2000/90 ≈ 22.222, so continuous optimum yields objective ≈ 66.667 (L ≈ 22.222, S ≈ 44.444). Thus an integer optimum must be ≥ 67.

Solution by enumeration (small loop)
- For L from 10 to 40:
  - required_small_by_capacity = max(0, ceil((2000 − 50L)/20))
  - S = max(2L, required_small_by_capacity)
  - record Z = S + L and pick minimal Z.

```python
#!/usr/bin/env python3
import math

# Problem data
capacity_need = 2000
cap_small = 20
cap_large = 50
min_large = 10

# Search bounds for L
L_min = min_large
L_max = math.ceil(capacity_need / cap_large)  # 40

best = None  # tuple (Z, S, L)

for L in range(L_min, L_max + 1):
    # small wagons required by capacity (non-negative)
    remaining = capacity_need - cap_large * L
    req_small_by_capacity = max(0, math.ceil(remaining / cap_small))
    # enforce ramp ratio S >= 2L
    S = max(2 * L, req_small_by_capacity)
    Z = S + L
    if best is None or Z < best[0]:
        best = (Z, S, L)

# Unpack best solution
best_Z, best_S, best_L = best

# Sanity checks
assert best_L >= min_large
assert best_S >= 2 * best_L
assert cap_small * best_S + cap_large * best_L >= capacity_need
assert isinstance(best_Z, int)

print(f"FINAL_OBJ: {best_Z}")
```
