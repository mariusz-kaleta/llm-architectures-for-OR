Model (variables, objective, constraints)

- Decision variables
  - A ∈ Z_{\ge 0}: number of type II ambulance shifts.
  - V ∈ Z_{\ge 0}: number of hospital van shifts.

- Objective
  - Minimise total daily cost: minimize 820·A + 550·V.

- Constraints
  1. Capacity: 20·A + 15·V ≥ 320.
  2. Union fraction (at most 60% of shifts are vans):
     V ≤ 0.60·(A + V).
     This is equivalent (after rearranging) to V ≤ 1.5·A (or A ≥ (2/3)·V).
  3. Practical integer bounds (optional but useful):
     0 ≤ A ≤ 16  (since ceil(320/20) = 16)
     0 ≤ V ≤ 22  (since ceil(320/15) = 22)
  4. Integrality: A and V integers.

Remarks on the union constraint
- V ≤ 0.60(A + V) ⇒ V − 0.6V ≤ 0.6A ⇒ 0.4V ≤ 0.6A ⇒ V ≤ 1.5A.
  Either form is fine in a solver; the V ≤ 1.5A integer-friendly linear form is convenient.

Solving approach (small discrete search)
- Because A ∈ {0,…,16} and V ∈ {0,…,22}, a direct enumeration (or loop over A computing the minimal feasible V = max(0, ceil((320−20A)/15)) and checking V ≤ floor(1.5A)) is trivial and robust.
- Alternatively formulate and solve with an ILP solver (pulp, ortools, mip, pyomo, gurobi, CPLEX).

Recommended Python libraries
- pulp (simple and pure-Python ILP modeling + CBC solver)
- OR-Tools (Google’s CP-SAT for integer problems)
- mip (python-mip) or pyomo (if you have commercial solvers)
- For brute-force enumeration plain Python / numpy is sufficient.

Optimal solution (found by enumeration / ILP)

- V = 8 van shifts
- Total capacity = 20·10 + 15·8 = 200 + 120 = 320 (meets requirement exactly)

Notes

- Implementation tip: either solve the ILP with pulp/or-tools, or enumerate A = 0..16 and set V = ceil(max(0, (320−20A)/15)). Reject if V > floor(1.5A) or V > 22. Choose the pair with minimum cost.

```python
#!/usr/bin/env python3
# Self-contained script to solve the ILP by enumeration
import math

# Problem data
cost_A = 820
cost_V = 550
cap_A = 20
cap_V = 15
demand = 320

# Practical bounds
A_max = 16
V_max = 22

best_obj = None
best_solution = None

for A in range(0, A_max + 1):
    # Minimum V to meet capacity
    remaining = demand - cap_A * A
    if remaining <= 0:
        V_min = 0
    else:
        V_min = math.ceil(remaining / cap_V)
    # Union constraint: V <= 1.5 * A  => V <= floor(1.5*A)
    V_ub_by_union = math.floor(1.5 * A)
    # Also respect practical V bound
    V_ub = min(V_ub_by_union, V_max)
    if V_min > V_ub:
        continue  # no feasible V for this A
    # For cost minimization and positive cost for V, choose smallest feasible V (V_min)
    V = V_min
    obj = cost_A * A + cost_V * V
    if (best_obj is None) or (obj < best_obj):
        best_obj = obj
        best_solution = (A, V)

# If no feasible solution found (shouldn't happen for this data), raise error
if best_obj is None:
    raise RuntimeError("No feasible solution found")

value = best_obj
# Final required print line
print(f"FINAL_OBJ: {value}")
```
