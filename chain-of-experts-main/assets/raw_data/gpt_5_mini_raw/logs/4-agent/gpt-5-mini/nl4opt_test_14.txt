Formal model

Decision variables
- S ∈ Z≥0 : number of scooters.
- R ∈ Z≥0 : number of rickshaws.

Objective
- Minimize S (the number of scooters used).

Constraints
1) Capacity: 2S + 3R ≥ 300.  (each scooter carries 2, each rickshaw 3)
2) Rickshaw proportion cap: R / (S + R) ≤ 0.4.  This is non-linear; an equivalent linear, integer-compatible form is:
   3R ≤ 2S  (derived from R ≤ 0.4(S+R) ⇒ 0.6R ≤ 0.4S ⇒ R ≤ (2/3)S, multiply both sides by 3)
3) Integrality and non-negativity: S, R ∈ Z≥0. (Capacity ensures S+R>0)

Mathematical ILP
minimize S
subject to
  2S + 3R ≥ 300
  3R ≤ 2S
  S, R ∈ Z≥0

Solution (analytical)
- From 3R ≤ 2S, the maximum allowed rickshaws for a given S is R_max = floor(2S/3).
- Substitute the maximal R to check capacity: require 2S + 3·floor(2S/3) ≥ 300.
- The continuous relaxation (using R = 2S/3) gives 2S + 3·(2S/3) = 4S ≥ 300 ⇒ S ≥ 75.
- Check S = 75: R_max = floor(2·75/3) = 50. Capacity = 2·75 + 3·50 = 150 + 150 = 300, and rickshaw share = 50/(75+50) = 0.4 ⇒ feasible.
- Check S = 74: R_max = floor(148/3) = 49, capacity = 2·74 + 3·49 = 148 + 147 = 295 < 300 ⇒ infeasible.

Therefore the minimum number of scooters is S* = 75, with R* = 50 rickshaws.

Verification
- Capacity: 2·75 + 3·50 = 300 ≥ 300 ✓
- Rickshaw proportion: 50 / (75 + 50) = 0.4 ≤ 0.4 ✓
- Integrality: S and R are integers ✓

Algorithmic approaches
- Direct ILP solver: model as above and solve with an ILP solver (CBC, GLPK, Gurobi, CPLEX).
- Simple enumeration (one-dimensional search): for S from 0 upward compute R_max = floor(2S/3) and test 2S + 3R_max ≥ 300; stop at first feasible S. Because there are only two integer variables, this is trivial and very fast.

Recommended Python libraries
- pulp (uses CBC by default) — easy ILP modelling.
- OR-Tools CP-SAT (ortools.sat.python.cp_model) — very fast for integer problems.
- pyomo (with CBC/Gurobi/CPLEX) — more feature-rich modelling.
(Do not use scipy.optimize for pure-integer problems.)

# Self-contained script to solve the ILP:
#   minimize S
#   s.t. 2S + 3R >= 300
#        3R <= 2S
#        S,R nonnegative integers

# Uses simple enumeration over S and computes R_max = floor(2S/3)

def find_optimal():
    S_opt = None
    R_opt = None
    for S in range(0, 1000):  # upper bound well above what is needed
        R_max = (2 * S) // 3
        if 2 * S + 3 * R_max >= 300:
            S_opt = S
            R_opt = R_max
            break
    if S_opt is None:
        raise ValueError("No feasible solution found within search bounds.")
    return S_opt, R_opt

if __name__ == "__main__":
    S_opt, R_opt = find_optimal()
