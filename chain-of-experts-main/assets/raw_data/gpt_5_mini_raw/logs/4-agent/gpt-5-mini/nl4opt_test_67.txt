Decision variables

Constraints
1. Total-unit limit: F + S ≤ 300.
2. Minimum fertilizer: F ≥ 50.
3. Relative limit (fertilizer at most twice seeds): F ≤ 2 S.
4. Nonnegativity: F ≥ 0, S ≥ 0.

Objective
Minimise total effective preparation time
T(F,S) = 0.5·F + 1.5·S.

Summary of the LP
minimise T = 0.5 F + 1.5 S
subject to
- F + S ≤ 300
- F ≥ 50
- F − 2 S ≤ 0
- F, S ≥ 0

Analytical solution (two-variable LP)
Because seeds cost more time per unit (1.5) than fertilizer (0.5), we want as much fertilizer and as little seeds as allowed by the constraints. From F ≤ 2S we get S ≥ F/2. For any feasible F the best (time-minimising) S is the smallest feasible S, i.e. S = F/2 (provided the total-capacity constraint allows it). Substituting S = F/2 into the objective gives T = 0.5F + 1.5(F/2) = 1.25 F, which is increasing in F. Thus we minimise T by taking F as small as allowed by constraints, i.e. F = 50, and then S = F/2 = 25.

Feasible-checks:
- Total units = F + S = 50 + 25 = 75 ≤ 300 (total-capacity not binding).
- All constraints satisfied.

Optimal solution
- F* = 50 units of fertilizer
- S* = 25 units of seeds
- Minimum total time T* = 0.5·50 + 1.5·25 = 25 + 37.5 = 62.5 minutes

Active (binding) constraints at optimum
- F ≥ 50 (minimum fertilizer) binds.
- F ≤ 2S binds (F = 2S).
- Total-capacity F + S ≤ 300 is not binding (only 75 used).

Feasibility and sensitivity observations
- The minimum total units implied by constraints is 1.5·F_min (because S ≥ F/2). With F_min = 50 this is 75, so the total-capacity must be at least 75 for feasibility. If the cap < 75 the problem is infeasible.
- If the minimum fertilizer requirement changed to f0, and f0 ≤ 200, the same reasoning gives the optimal solution F* = f0, S* = f0/2, with objective T* = 1.25·f0. If f0 > 200 the problem is infeasible (because F ≤ 2S and S ≤ 300−F together imply F ≤ 200).
- Changing the total-capacity has no effect on the solution as long as the cap ≥ 1.5·F_min (here ≥ 75). If the cap is reduced below that threshold the problem becomes infeasible.
- If the per-unit times changed (e.g. seeds become faster than fertilizer), the optimal corner will change; you would then re-evaluate the polygon corners.

# Self-contained script to solve the given LP analytically

# Problem data (directly initialised)
total_capacity = 300       # upper bound: F + S <= total_capacity
min_fertilizer = 50        # F >= min_fertilizer

# From constraint F <= 2 S we get S >= F/2.
# To minimise T = 0.5 F + 1.5 S we choose the smallest feasible S for given F, i.e. S = F/2.
# Substitute: T(F) = 0.5 F + 1.5*(F/2) = 1.25 * F, which is increasing in F.
# So choose the smallest allowed F, i.e. F = min_fertilizer, and S = F/2.

F_opt = float(min_fertilizer)
S_opt = F_opt / 2.0

# Feasibility check: total capacity must allow F + S
if F_opt + S_opt > total_capacity + 1e-9:
    raise ValueError("Problem infeasible with given total_capacity and min_fertilizer.")

# Objective value
