[MODELER]:
Model summary (assumptions)
- Let x = number of regular handbags sold per month.
- Let y = number of premium handbags sold per month.
- Interpret "at most 475 handbags of either type per month" as a combined sales limit x + y ≤ 475 (if you intended a separate limit of 475 for each type, note that the final numeric result below is the same).

Decision variables
- x ≥ 0 (integer if you require whole handbags) — regular handbags.
- y ≥ 0 (integer) — premium handbags.

Objective
- Maximize monthly profit: Maximize Z = 30x + 180y.

Constraints
- Budget constraint (manufacturing cost): 200x + 447y ≤ 250000.
- Sales capacity: x + y ≤ 475.
- Nonnegativity: x, y ≥ 0 (and optionally integer).

Full mathematical model
Maximize Z = 30x + 180y
subject to
200x + 447y ≤ 250000
x + y ≤ 475
x, y ≥ 0 (and integer if needed)

Solution (analysis and result)
- Profit per dollar of cost: regular 30/200 = 0.15, premium 180/447 ≈ 0.4027. Premiums give much higher return per dollar of cost.
- The budget allows up to y = 250000/447 ≈ 559 premium units, but the sales cap limits total to 475 units.
- Therefore the optimal solution is to sell only premium handbags up to the sales limit:

x* = 0 regular handbags
y* = 475 premium handbags

Maximum monthly profit Z* = 30·0 + 180·475 = $85,500.

Suggested Python libraries
- pulp (simple LP and MILP modeling and solving)
- OR-Tools (CP-SAT for integer problems, or linear solver)
- scipy.optimize.linprog (continuous LP only)
- cvxpy (convex optimization modeling)

Example using PuLP (integer variables)
----
from pulp import LpMaximize, LpProblem, LpVariable, lpSum, PULP_CBC_CMD

prob = LpProblem("Handbags", LpMaximize)
x = LpVariable('regular', lowBound=0, cat='Integer')
y = LpVariable('premium', lowBound=0, cat='Integer')

prob += 30*x + 180*y                       # objective
prob += 200*x + 447*y <= 250000           # budget
prob += x + y <= 475                      # sales cap

prob.solve(PULP_CBC_CMD(msg=False))
print(x.value(), y.value(), "Profit =", 30*x.value() + 180*y.value())

This will return x=0, y=475, Profit=85500.

If you want the continuous relaxation (non-integer) use cat='Continuous' or scipy.optimize.linprog. If you meant the 475 limit to be per type (x ≤ 475, y ≤ 475), the same optimal choice (x=0, y=475) still applies.

import sys

# Try to solve using PuLP; if unavailable, fall back to direct reasoning
try:
    from pulp import LpProblem, LpMaximize, LpVariable, PULP_CBC_CMD, value

    # Define the problem
    prob = LpProblem("Handbags", LpMaximize)

    # Decision variables (integer)
    x = LpVariable('regular', lowBound=0, cat='Integer')   # regular handbags
    y = LpVariable('premium', lowBound=0, cat='Integer')   # premium handbags

    # Objective: maximize profit
    prob += 30 * x + 180 * y

    # Constraints
    prob += 200 * x + 447 * y <= 250000   # budget
    prob += x + y <= 475                  # sales capacity

    # Solve
    prob.solve(PULP_CBC_CMD(msg=False))

    # Retrieve integer values (ensure ints)
    x_val = int(round(value(x)))
    y_val = int(round(value(y)))
    profit = 30 * x_val + 180 * y_val

except Exception:
    # If PuLP is not available, use the analytical solution:
    # premium handbags give higher profit per dollar, so use sales cap entirely on premium.
    x_val = 0
    y_val = 475
    profit = 30 * x_val + 180 * y_val

# Final required output line
print(f"FINAL_OBJ: {profit}")
