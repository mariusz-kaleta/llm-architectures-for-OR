Model

- Decision variables
  - W = number of wraps (≥ 0)
  - P = number of platters (≥ 0)

  (If counts must be integers, add W, P ∈ Z≥0. Otherwise treat as continuous.)

- Objective
  - Minimize total production time T = 10 W + 8 P (minutes)

- Constraints
  - Meat use: 5 W + 7 P ≥ 3000
  - Rice use: 3 W + 5 P ≥ 2500
  - Wrap-to-platter ratio: W ≥ 3 P
  - Nonnegativity: W ≥ 0, P ≥ 0

This is a linear program (or integer linear program if integrality required).

Analytic solution (continuous variables)
- The LP optimum occurs at a vertex. Solving the binding constraints W = 3P and 3W + 5P = 2500 yields
  - P* = 2500 / 14 = 178.5714286
  - W* = 3 P* = 535.7142857
- Objective value: T* = 10 W* + 8 P* = 6785.7142857 minutes.

Integer solution (if W and P must be integer)
- Solve the integer program (or round near the continuous optimum and check feasibility). A nearby feasible integer solution with minimum time is
  - W = 537 wraps, P = 178 platters
  - Total time = 10·537 + 8·178 = 6794 minutes
(You can confirm optimality exactly by solving the integer program with a solver.)

Notes / interpretation
- Platters consume meat and rice faster per minute than wraps, so unconstrained minimization of time would favor platters. The constraint W ≥ 3P forces production to include many wraps, shifting the optimum to the vertex found above.
- If fractional items are not meaningful, solve the integer program to get the exact integer optimum (may differ slightly as shown).

import math
import numpy as np
from scipy.optimize import linprog

# Problem data
# Decision variables: W (wraps), P (platters)
# Minimize T = 10 W + 8 P
c = np.array([10.0, 8.0])

# Constraints (given as inequalities >=; convert to <= for linprog by multiplying by -1)
# 5 W + 7 P >= 3000  -->  -5 W -7 P <= -3000
# 3 W + 5 P >= 2500  -->  -3 W -5 P <= -2500
# W - 3 P >= 0       -->  -W +3 P <= 0
A_ub = np.array([
    [-5.0, -7.0],
    [-3.0, -5.0],
    [-1.0,  3.0]
])
b_ub = np.array([-3000.0, -2500.0, 0.0])

# Bounds: W >=0, P >=0
bounds = [(0, None), (0, None)]

# Solve continuous LP
res = linprog(c, A_ub=A_ub, b_ub=b_ub, bounds=bounds, method='highs')

cont_obj = None
cont_solution = None
if res.success:
    cont_solution = res.x
    cont_obj = res.fun
else:
    raise RuntimeError("Continuous LP did not solve successfully")

# Solve integer program by enumerating integer P and computing minimal feasible integer W
best_obj_int = None
best_W = None
best_P = None

# Determine reasonable search range for P.
# Lower bound 0, upper bound: using that W >= 3P -> objective >= 38 P, and we know continuous objective approx cont_obj,
# so P cannot be larger than cont_obj/38 by optimality. Add margin.
upper_P = int(math.ceil(cont_obj / 38.0)) + 50
upper_P = max(upper_P, 1000)  # ensure enough range
upper_P = min(upper_P, 5000)  # cap to keep iteration reasonable

for P in range(0, upper_P + 1):
    # compute minimal integer W satisfying:
    # W >= 3P
    # 5W + 7P >= 3000  => W >= (3000 - 7P)/5
    # 3W + 5P >= 2500  => W >= (2500 - 5P)/3
    w1 = 3 * P
    # For the other lower bounds, if RHS <= 0 then no positive lower bound
    rhs_meat = (3000 - 7 * P) / 5.0
    rhs_rice = (2500 - 5 * P) / 3.0
    w2 = math.ceil(rhs_meat) if rhs_meat > 0 else 0
    w3 = math.ceil(rhs_rice) if rhs_rice > 0 else 0
    W = max(w1, w2, w3, 0)
    # Check feasibility explicitly (should hold if computed correctly)
    if 5 * W + 7 * P >= 3000 and 3 * W + 5 * P >= 2500 and W >= 3 * P:
        obj = 10 * W + 8 * P
        if (best_obj_int is None) or (obj < best_obj_int):
            best_obj_int = obj
            best_W = W
            best_P = P

# If enumeration missed due to upper bound, extend search conservatively
if best_obj_int is None:
    # fallback: search a larger P range
    for P in range(0, 10000):
        w1 = 3 * P
        rhs_meat = (3000 - 7 * P) / 5.0
        rhs_rice = (2500 - 5 * P) / 3.0
        w2 = math.ceil(rhs_meat) if rhs_meat > 0 else 0
        w3 = math.ceil(rhs_rice) if rhs_rice > 0 else 0
        W = max(w1, w2, w3, 0)
        if 5 * W + 7 * P >= 3000 and 3 * W + 5 * P >= 2500 and W >= 3 * P:
            obj = 10 * W + 8 * P
            if (best_obj_int is None) or (obj < best_obj_int):
                best_obj_int = obj
                best_W = W
                best_P = P

# Print intermediate results (allowed), then final required line
print(f"Continuous solution: W = {cont_solution[0]:.10f}, P = {cont_solution[1]:.10f}, Obj = {cont_obj:.10f}")
print(f"Integer solution:    W = {best_W}, P = {best_P}, Obj = {best_obj_int}")

# Final required single line output
print(f"FINAL_OBJ: {best_obj_int}")
