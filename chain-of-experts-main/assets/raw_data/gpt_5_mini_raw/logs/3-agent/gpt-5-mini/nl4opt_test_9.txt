Formal model

Decision variables
- x = number of small containers used (nonnegative integer)
- y = number of large containers used (nonnegative integer)

Objective
- Maximize total paste produced:
  maximize Z = 20x + 30y

Constraints
- Water availability: 10x + 20y ≤ 500
- Powder availability: 15x + 20y ≤ 700
- Integrality and nonnegativity: x, y ∈ Z≥0 (if containers must be whole); if fractional containers allowed, x,y ≥ 0 (continuous LP)

Remarks and solution
- Solve the two resource equalities simultaneously:
  10x + 20y = 500  → x = 50 − 2y
  15x + 20y = 700  → x = 700/15 − (20/15)y = 46.6667 − 1.3333y
  Equating gives y = 5 and x = 40.
- This (x,y) = (40,5) satisfies both constraints exactly, is integer, and yields
  Z = 20·40 + 30·5 = 950 units of paste.
- Because the LP intersection is an integer point, the integer-constrained optimum is the same.

Suggested Python libraries for solving
- For integer (MIP) formulation: PuLP (with CBC/GLPK/Gurobi), OR-Tools CP-SAT (fast MIP), or Pyomo (with a solver)
- For continuous LP: scipy.optimize.linprog, PuLP, cvxpy
- If you want to enumerate or use exact integer programming: PuLP or OR-Tools are recommended.

Minimal example (PuLP)
- Define variables x,y as integers, add constraints and maximize 20*x+30*y. PuLP will find x=40, y=5, Z=950.

# Self-contained brute-force integer solver for the container optimization problem

# Decision variables:
# x = number of small containers (integer >= 0)
# y = number of large containers (integer >= 0)
#
# Objective:
#   Maximize Z = 20*x + 30*y
#
# Constraints:
#   10*x + 20*y <= 500  (water)
#   15*x + 20*y <= 700  (powder)

# No external libraries required for this small integer enumeration.

# Determine reasonable upper bounds for enumeration from resource limits
max_x = min(500 // 10, 700 // 15)  # floor bounds from water and powder
max_y = min(500 // 20, 700 // 20)

best_obj = None
best_x = None
best_y = None

for x in range(max_x + 1):
    for y in range(max_y + 1):
        if 10 * x + 20 * y <= 500 and 15 * x + 20 * y <= 700:
            obj = 20 * x + 30 * y
            if (best_obj is None) or (obj > best_obj):
                best_obj = obj
                best_x = x
                best_y = y

# At this point best_obj should be the optimal objective value (integer)
# Print final objective value in the required format
print(f"FINAL_OBJ: {best_obj}")
