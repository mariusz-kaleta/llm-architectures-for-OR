Formal model

- Decision variables
  - x = number of images assigned to the specialized company (nonnegative, real or integer).
  - y = number of images assigned to the common company (nonnegative, real or integer).

- Parameters (given)
  - N = required images = 10,000.
  - rate_S = 60 images/hour, cost_S = $100/hour.
  - rate_C = 40 images/hour, cost_C = $72/hour.

- Derived per-image costs
  - c_S = cost_S / rate_S = 100 / 60 = 5/3 ≈ $1.6666667 per image.
  - c_C = cost_C / rate_C = 72 / 40 = 1.8 per image.

- Constraints
  1. Meet required dataset size: x + y = N (since cost increases with extra images, the optimal solution will use exactly N images).
  2. At least one third of the work must be done by the specialized company:
     x ≥ N / 3.
  3. Nonnegativity: x ≥ 0, y ≥ 0.

- Objective
  Minimize total cost:
    minimize Z = c_S * x + c_C * y
  or equivalently using hours:
    Z = 100*(x/60) + 72*(y/40).

Solution (analytical)
- With x + y = N and x ≥ N/3, substitute y = N − x and minimize Z(x) = c_S x + c_C (N − x) = c_C N + (c_S − c_C) x.
- Since c_S − c_C = 1.6667 − 1.8 = −0.13333 < 0, Z(x) is decreasing in x. So to minimize cost you should maximize x subject to x ≤ N.
- Therefore the optimum is x* = N, y* = 0 (assign all images to the specialized company).
- Check constraint x* ≥ N/3: 10000 ≥ 3333.33 satisfied.

Numeric result
- Assign x = 10,000 images to the specialized company, y = 0 to the common company.
- Hours at specialized: 10000 / 60 = 166.6667 hours.
- Total cost = (5/3)*10000 = $16,666.67 (or 166.6667 hours × $100/hr).

Remarks and extensions
- If additional constraints exist (e.g., maximum available capacity for a vendor, or a hard deadline D that requires completion within D hours when companies work in parallel), add:
  - capacity constraints: x ≤ cap_S, y ≤ cap_C, or
  - deadline constraint linearized with T: x/60 ≤ T, y/40 ≤ T and T ≤ D.
  Those extensions remain a linear program (add variable T to linearize the max operator).
- If you require integer image assignments, this becomes an integer linear program (but fractional solutions are practically fine because hours can be fractional).

#!/usr/bin/env python3
# Self-contained script to solve the described LP analytically.

# Parameters
N = 10000  # required images
rate_S = 60.0
cost_S = 100.0
rate_C = 40.0
cost_C = 72.0

# Derived per-image costs
c_S = cost_S / rate_S
c_C = cost_C / rate_C

# Decide optimal allocation given constraints:
# x = images to specialized, must satisfy x >= N/3 and x <= N (implicitly)
# Objective: minimize Z = c_S*x + c_C*(N-x) = c_C*N + (c_S-c_C)*x
# If c_S < c_C, objective decreases with x so set x = N
# If c_S > c_C, objective increases with x so set x = N/3 (minimum allowed)
# If equal, any feasible x; choose x = N for concreteness.

if c_S < c_C:
    x_opt = N
elif c_S > c_C:
    x_opt = N / 3.0
else:
    x_opt = N

y_opt = N - x_opt

# Compute hours and total cost
hours_S = x_opt / rate_S
hours_C = y_opt / rate_C
total_cost = cost_S * hours_S + cost_C * hours_C

value = total_cost

print(f"FINAL_OBJ: {value}")
