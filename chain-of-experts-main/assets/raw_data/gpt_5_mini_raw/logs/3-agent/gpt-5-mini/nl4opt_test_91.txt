[MODELER]:
Model (linear programming)

Decision variables
- x ≥ 0 : number of turkey dinners
- y ≥ 0 : number of tuna salad sandwiches

Objective
- Minimize total fat F = 12 x + 8 y

Constraints
- Protein requirement: 20 x + 18 y ≥ 150
- Carbohydrate requirement: 30 x + 25 y ≥ 200
- At most 40% turkey: x ≤ 0.4 (x + y). Equivalently 3 x − 2 y ≤ 0 (or y ≥ 1.5 x)
- Nonnegativity: x ≥ 0, y ≥ 0

(If meals must be integer, add integrality constraints x, y ∈ Z≥0; otherwise treat as continuous.)

Comment on structure / solution
- Tuna has lower fat per unit protein and per unit carbohydrate than turkey, so tuna dominates; the LP solution uses only tuna (x = 0).
- Continuous optimal solution: x* = 0, y* = 150 / 18 = 25/3 ≈ 8.333… sandwiches. Fat = 8 · (25/3) = 200/3 ≈ 66.67 grams.
- If integer (whole meals) is required: protein requires y ≥ 9 (since 18·8 = 144 < 150), so the best integer solution is x = 0, y = 9 with fat = 8·9 = 72 grams. (You can check that any combination with x > 0 produces equal or higher fat.)

Recommended solvers / Python libraries
- Continuous LP: scipy.optimize.linprog or cvxpy
- Modeling and integer/mixed-integer LP: PuLP, OR-Tools (ortools.linear_solver), Pyomo (with CBC/Gurobi/CPLEX backends)
- If you want a short example, PuLP or scipy.optimize.linprog are convenient; for integer programs use PuLP or OR-Tools.

import math

# Dietary optimization problem data (directly initialized)
# Decision variables:
# x >= 0 : number of turkey dinners
# y >= 0 : number of tuna salad sandwiches
#
# Objective: minimize fat F = 12 x + 8 y
#
# Constraints:
# (1) 20 x + 18 y >= 150   (protein)
# (2) 30 x + 25 y >= 200   (carbohydrates)
# (3) 3 x - 2 y <= 0       (at most 40% turkey)  i.e. y >= 1.5 x
# x >= 0, y >= 0

def solve_2x2(a1, b1, c1, a2, b2, c2):
    """Solve 2x2 linear system:
       a1*x + b1*y = c1
       a2*x + b2*y = c2
       Returns (x,y) or None if singular.
    """
    det = a1 * b2 - a2 * b1
    if abs(det) < 1e-12:
        return None
    x = (c1 * b2 - c2 * b1) / det
    y = (a1 * c2 - a2 * c1) / det
    return (x, y)

def is_feasible(x, y, tol=1e-9):
    if x < -tol or y < -tol:
        return False
    # Protein
    if 20*x + 18*y < 150 - tol:
        return False
    # Carbohydrates
    if 30*x + 25*y < 200 - tol:
        return False
    # Ratio: 3x - 2y <= 0  -> y >= 1.5 x
    if 3*x - 2*y > tol:
        return False
    return True

def objective(x, y):
    return 12*x + 8*y

candidates = []

# Intersection of protein (1) and carbs (2)
p = solve_2x2(20, 18, 150, 30, 25, 200)
if p:
    candidates.append(("protein&carbs", p))

# Intersection of protein (1) and ratio (3): y = 1.5 x -> substitute
# Solve 20 x + 18*(1.5 x) = 150  => 20x + 27x = 47x = 150
x13 = 150.0 / 47.0
y13 = 1.5 * x13
candidates.append(("protein&ratio", (x13, y13)))

# Intersection of carbs (2) and ratio (3)
x23 = 200.0 / (30.0 + 25.0 * 1.5/1.0)  # not used; compute properly below
# Better compute by substituting y=1.5x into carbs
x23 = 200.0 / (30.0 + 25.0 * 1.5)
y23 = 1.5 * x23
candidates.append(("carbs&ratio", (x23, y23)))

# Intersections with axes:
# x = 0 with protein and carbs
y_x0_protein = 150.0 / 18.0
y_x0_carbs = 200.0 / 25.0
candidates.append(("x=0_protein", (0.0, y_x0_protein)))
candidates.append(("x=0_carbs", (0.0, y_x0_carbs)))

# y = 0 with protein and carbs (likely infeasible due to ratio constraint, but include)
x_y0_protein = 150.0 / 20.0
x_y0_carbs = 200.0 / 30.0
candidates.append(("y=0_protein", (x_y0_protein, 0.0)))
candidates.append(("y=0_carbs", (x_y0_carbs, 0.0)))

# Also include the origin
candidates.append(("origin", (0.0, 0.0)))

# Evaluate candidates, keep feasible ones
feasible_points = []
for name, (x, y) in candidates:
    if is_feasible(x, y):
        feasible_points.append((name, x, y, objective(x, y)))

# Additionally, consider intersection of protein & carbs computed via solve_2x2 if it exists and wasn't added
# (We already added it above if p found.)
# For robustness, also include intersections of each pair solved via solve_2x2 explicitly:
# protein (20,18,150), carbs (30,25,200), ratio (3,-2,0)
pairs = [
    ("protein&carbs", (20,18,150, 30,25,200)),
    ("protein&ratio", (20,18,150, 3,-2,0)),
    ("carbs&ratio", (30,25,200, 3,-2,0)),
]
for name, (a1,b1,c1,a2,b2,c2) in pairs:
    sol = solve_2x2(a1,b1,c1,a2,b2,c2)
    if sol:
        x, y = sol
        # avoid duplicates (approx)
        already = False
        for _, xx, yy, _ in feasible_points:
            if abs(xx - x) < 1e-9 and abs(yy - y) < 1e-9:
                already = True
                break
        if not already and is_feasible(x,y):
            feasible_points.append((name+"_solve", x, y, objective(x, y)))

# If no feasible candidate found (shouldn't happen), fallback to scanning boundary
if not feasible_points:
    # brute force small continuous grid search to approximate continuous solution
    best = None
    for x in [i * 0.1 for i in range(0, 501)]:  # 0..50
        for y in [j * 0.1 for j in range(0, 501)]:
            if is_feasible(x,y):
                val = objective(x,y)
                if best is None or val < best[0]:
                    best = (val, x, y)
    if best:
        feasible_points.append(("grid_approx", best[1], best[2], best[0]))

# Choose best continuous solution
feasible_points.sort(key=lambda t: t[3])
cont_best = feasible_points[0]  # (name, x, y, obj)
cont_name, cont_x, cont_y, cont_obj = cont_best

# Now compute integer (whole meals) optimal solution by brute force enumeration
# Choose reasonable bounds for enumeration. Upper bound: if all fat from tuna, need at most protein constraint -> y <= ceil(150/18)+20 margin
# But to be safe, use upper bound 100.
int_best = None
max_search = 100
for xi in range(0, max_search+1):
    for yi in range(0, max_search+1):
        if is_feasible(xi, yi):
            val = objective(xi, yi)
            if int_best is None or val < int_best[0]:
                int_best = (val, xi, yi)

# Print summary
print("Continuous relaxation best candidate:")
print(f"  constraints intersect: {cont_name}")
print(f"  x = {cont_x:.12f}, y = {cont_y:.12f}")
print(f"  objective (fat) = {cont_obj:.12f}")

if int_best:
    int_val, int_x, int_y = int_best
    print("Integer (whole meals) best candidate (by enumeration):")
    print(f"  x = {int_x}, y = {int_y}")
    print(f"  objective (fat) = {int_val:.12f}")
else:
    print("No feasible integer solution found within search bounds.")

# Final output line required by instructions (only this text on the last line)
print(f"FINAL_OBJ: {cont_obj}")
