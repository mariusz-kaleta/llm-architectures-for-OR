Model summary
- Decision variables
  - b ∈ Z≥0 : number of electric bikes used
  - s ∈ Z≥0 : number of scooters used

- Objective
  - Maximize total meals: maximize Z = 8b + 5s

- Constraints
  1. Charge availability: 3b + 2s ≤ 200
  2. At most 30% of electric vehicles can be bikes:
     b ≤ 0.3(b + s)  which is algebraically equivalent to 7b ≤ 3s
  3. At least 20 scooters: s ≥ 20
  4. Integrality and nonnegativity: b, s ∈ Z≥0

Analysis and optimal solution (integer)
- Rearranging the 30% constraint: b ≤ 0.3(b+s) → 0.7b ≤ 0.3s → 7b ≤ 3s.
- For a fixed b, the objective increases with s, so for each feasible b we take the largest s permitted by charge: s ≤ (200 − 3b)/2 (and s must also satisfy s ≥ max(20, ⌈7b/3⌉)).
- The fraction constraint together with the charge constraint give: 3b + 2(7b/3) ≤ 200 → 23b ≤ 600 → b ≤ 600/23 ≈ 26.086. So the largest integer b possible is b = 26.
- For b = 26 we need s ≥ ⌈7·26/3⌉ = 61, and the charge constraint gives s ≤ (200 − 3·26)/2 = 61. So s = 61 exactly, using all charge.
- Objective value: Z = 8·26 + 5·61 = 208 + 305 = 513 meals.

Therefore the optimal integer solution is:
- Bikes: 26
- Scooters: 61
- Maximum meals delivered: 513
