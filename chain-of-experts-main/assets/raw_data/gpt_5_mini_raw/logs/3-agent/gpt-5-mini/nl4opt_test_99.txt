Decision variables
- s ∈ Z≥0 : number of small teams
- l ∈ Z≥0 : number of large teams

Objective
- Maximize total area mowed: Z = 50 s + 80 l

Constraints
1. Employee limit: 3 s + 5 l ≤ 150
2. Small teams at least 3× large teams: s ≥ 3 l
3. Minimum teams: s ≥ 10, l ≥ 6
4. Integrality: s, l integers

Suggested solvers / Python libraries
- PuLP (simple MIP modeling + CBC / GLPK)
- OR-Tools CP-SAT (fast, robust integer solver)
- Pyomo + a MIP solver (CBC, GLPK, Gurobi if available)

Solution (analytic)
- Small-team efficiency = 50/3 ≈ 16.667 sq ft per employee
- Large-team efficiency = 80/5 = 16 sq ft per employee
Because small teams are more efficient, we should use as many small teams as possible subject to constraints. For each feasible l (≥6), maximum s is floor((150 − 5l)/3) but must also satisfy s ≥ 3l and s ≥ 10. Checking feasible l values shows the optimum is:
- l = 6, s = 40 (uses all 150 employees: 3·40 + 5·6 = 150)
- Total area = 50·40 + 80·6 = 2,480 sq ft

Hence use 40 small teams and 6 large teams.

Root cause
- The container ran something like:
  python -c 'unknown
  #!/usr/bin/env python3
  ...'
- Python interprets the first token "unknown" as code, so you get NameError: name 'unknown' is not defined.

What to fix
- Remove the stray "unknown" prefix from the string passed to python -c.
- The text passed to -c should be just the Python source (the shebang is unnecessary when running with -c and can be dropped too).

- Wrong (what produced the error):
