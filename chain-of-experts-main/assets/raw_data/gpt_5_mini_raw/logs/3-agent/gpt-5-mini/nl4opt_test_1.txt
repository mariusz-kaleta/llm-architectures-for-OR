Model

- Decision variables:
  - L = number of large pills (integer, L ≥ 0)
  - S = number of small pills (integer, S ≥ 0)

- Objective: minimize total filler used
  - Minimize F = 2L + 1·S

- Constraints:
  1. Medicinal ingredient supply: 3L + 2S ≤ 1000
  2. At least 100 large pills: L ≥ 100
  3. At least 60% of total are small: S ≥ 0.6(L + S) ⇔ S ≥ 1.5 L
  4. Integrality: L, S ∈ Z≥0  (if non-integer pills are not allowed)

Feasibility analysis and solution

From constraint 3 we have S ≥ 1.5L. To minimize F = 2L + S for a given L, choose S as small as allowed, i.e. S = ceil(1.5L) (or S = 1.5L if continuous). The medicinal-ingredient constraint gives an upper bound on S:
S ≤ (1000 − 3L)/2.

Feasible L values must satisfy 1.5L ≤ (1000 − 3L)/2 ⇒ 6L ≤ 1000 ⇒ L ≤ 166.66…, so integer L ∈ {100, …, 166}.

The objective with S = 1.5L becomes F = 2L + 1.5L = 3.5L (ignoring integer rounding). This is increasing in L, so the minimum occurs at the smallest feasible L, which is L = 100. Then S = 1.5·100 = 150 (this satisfies the ingredient constraint: 3·100 + 2·150 = 600 ≤ 1000).

Thus the optimal production plan is:
- Large pills L = 100
- Small pills S = 150
- Total filler used = 2·100 + 1·150 = 350 units

Suggested solvers/libraries
- For exact integer solution: OR-Tools CP-SAT (ortools.sat.python), PuLP (with CBC/GLPK/GUROBI), or Pyomo + a MIP solver.
- For continuous LP relaxation: scipy.optimize.linprog or PuLP.

Example (PuLP) sketch

- Use PuLP to define integer variables L, S, minimize 2L + S, and add the constraints above; solve with the default CBC solver. This will return L=100, S=150.

import math

# Brute-force integer search (self-contained, no external solver needed)

best = None  # tuple (F, L, S)

# From constraints analysis: L integer, 100 <= L <= 166
for L in range(100, 167):
    S_min = math.ceil(1.5 * L)  # from S >= 1.5 L
    S_max = math.floor((1000 - 3 * L) / 2)  # from 3L + 2S <= 1000
    if S_min <= S_max and S_min >= 0:
        S = S_min  # to minimize F = 2L + S choose smallest feasible S
        F = 2 * L + S
        if best is None or F < best[0]:
            best = (F, L, S)

if best is None:
    # No feasible solution found (should not happen for given data)
    final_obj = None
else:
    final_obj, L_opt, S_opt = best
    # Ensure integer representation if essentially integer
    if abs(final_obj - round(final_obj)) < 1e-9:
        final_obj = int(round(final_obj))

print(f"FINAL_OBJ: {final_obj}")
