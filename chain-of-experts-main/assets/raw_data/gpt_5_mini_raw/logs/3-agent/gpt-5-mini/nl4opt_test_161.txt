- g ∈ Z≥0 : number of gas grills.
- e ∈ Z≥0 : number of electric grills.

Objective
- Minimize total number of grills: minimize Z = g + e.

Constraints
1. Production capacity (at least 150 patties/min):
   20 g + 30 e ≥ 150
   (equivalently 2 g + 3 e ≥ 15)

2. Cooking oil capacity (at most 140 units/min):
   20 g + 25 e ≤ 140
   (equivalently 4 g + 5 e ≤ 28)

3. Electric grills must be fewer than gas grills (strict):
   e < g
   For integer variables this is represented exactly as
   e ≤ g − 1.

4. Integrality and nonnegativity:
   g, e ∈ Z≥0.

Summary (integer linear program)
Minimize g + e
subject to
  2 g + 3 e ≥ 15
  4 g + 5 e ≤ 28
  e ≤ g − 1
  g, e ∈ Z≥0

Feasibility analysis and solution
- From the oil constraint 4 g + 5 e ≤ 28 we have g ≤ 7 when e = 0, and g is small for larger e. Because grills must be integer, you can enumerate feasible integer pairs (g = 0..7 with e = 0..g−1) and check the production constraint.
- Doing this shows there is no integer pair (g, e) satisfying all three constraints simultaneously. Therefore the ILP as stated (with e ≤ g − 1) is infeasible: no integer solution exists.

Practical relaxations and their solutions
- If the intent was "electric grills must be no more than gas grills" (i.e., e ≤ g instead of strict e < g), the integer problem is feasible. The optimal integer solution minimizing g+e is
  g = 3, e = 3 (total grills = 6),
  which satisfies 20·3 + 30·3 = 150 patties/min and 20·3 + 25·3 = 135 ≤ 140 oil units/min.

- If you relax integrality and allow fractional grills (continuous LP) with e ≤ g, the LP optimum is also g = 3, e = 3 (total 6). # Self-contained script to analyze the ILP described in the prompt.
# It enumerates integer feasible solutions for the exact model (with e <= g - 1),
# reports infeasibility if none exist, and also finds the optimal solution for
# the relaxed-but-still-integer alternative model (e <= g).

def main():
    # Decision variables ranges (derived from oil constraint 4g+5e <= 28).
    # When e = 0, max g = floor(28/4) = 7. So g in 0..7 is sufficient.
    max_g = 7

    feasible_original = []
    for g in range(0, max_g + 1):
        for e in range(0, max(0, g)):  # e <= g - 1  => e in 0..g-1
            if 2 * g + 3 * e >= 15 and 4 * g + 5 * e <= 28:
                feasible_original.append((g, e))

    # Report results for the original (strict) model
    if feasible_original:
        # Find best (min g+e)
        best = min(feasible_original, key=lambda x: x[0] + x[1])
        best_obj = best[0] + best[1]
        print("Original model (with e <= g - 1) is FEASIBLE.")
        print(f"Best integer solution found: g = {best[0]}, e = {best[1]}, objective = {best_obj}")
        final_obj = best_obj
    else:
        print("Original model (with e <= g - 1) is INFEASIBLE: no integer (g,e) satisfy all constraints.")
        final_obj = "infeasible"

    # For completeness: analyze the practical relaxation e <= g (non-strict) with integer variables
    feasible_relaxed = []
    for g in range(0, max_g + 1):
        for e in range(0, g + 1):  # e <= g
            if 2 * g + 3 * e >= 15 and 4 * g + 5 * e <= 28:
                feasible_relaxed.append((g, e))
    if feasible_relaxed:
        best_relaxed = min(feasible_relaxed, key=lambda x: x[0] + x[1])
        best_relaxed_obj = best_relaxed[0] + best_relaxed[1]
        print("Relaxed integer model (with e <= g) is FEASIBLE.")

    else:
        print("Relaxed integer model (with e <= g) is INFEASIBLE as well (unexpected).")

    # The script must end by printing the final objective line exactly in the required format.
    # According to the formal analysis, the original strict model is infeasible; we return that status.
    print(f"FINAL_OBJ: {final_obj}")

if __name__ == "__main__":
    main()
