[MODELER]:
Model

Decision variables
- x1 = number of z-tube ads (integer, x1 >= 0)
- x2 = number of soorchle ads (integer, x2 >= 0)
- x3 = number of wassa ads (integer, x3 >= 0)

Objective
- Maximize total audience:
  maximize Z = 400000 x1 + 5000 x2 + 3000 x3

Constraints
- Budget: 1000 x1 + 200 x2 + 100 x3 <= 10000
- Soorchle limit: x2 <= 15
- At most one third of total ads on wassa:
  x3 <= (1/3) (x1 + x2 + x3)  ⇔  2 x3 <= x1 + x2
- At least 5% of ads on z-tube:
  x1 >= 0.05 (x1 + x2 + x3)  ⇔  19 x1 >= x2 + x3
- Integrality and nonnegativity: x1, x2, x3 ∈ Z_{\ge 0}

Notes on formulation
- All constraints are linear, so this is an integer linear program (ILP).
- The alternative algebraic forms (2 x3 <= x1 + x2 and 19 x1 >= x2 + x3) avoid fractional coefficients.

Solution insight
- Viewers per dollar: z-tube = 400000/1000 = 400 viewers per dollar; soorchle = 25 v/$; wassa = 30 v/$.
- z-tube gives by far the best return per dollar, so the optimal solution uses as many z-tube ads as budget allows under integer constraints.
- Budget allows at most floor(10000/1000) = 10 z-tube ads. (x1 = 10 uses the full budget.)
- With x1 = 10, x2 = 0, x3 = 0 all constraints are satisfied:
  - budget: 1000*10 = 10000
  - soorchle limit: 0 <= 15
  - wassa fraction: 0 <= (1/3)*10
  - z-tube share: 10 >= 0.05*10
- Total audience = 10 * 400000 = 4,000,000 viewers. This is optimal.

Suggested solvers/libraries
- pulp (Python) — easy ILP modelling, uses CBC by default.
- OR-Tools CP-SAT (Google) — powerful integer solver, good for larger / harder combinations.
- pyomo + a solver (CBC, GLPK, Gurobi) — more structured modelling if you need extensions.

Example (pulp) code
- This will solve the ILP and confirm the solution:

from pulp import LpProblem, LpMaximize, LpVariable, LpInteger, lpSum, PULP_CBC_CMD

# Problem
prob = LpProblem("ads", LpMaximize)

# Variables
x1 = LpVariable("z_tube", lowBound=0, cat=LpInteger)
x2 = LpVariable("soorchle", lowBound=0, cat=LpInteger)
x3 = LpVariable("wassa", lowBound=0, cat=LpInteger)

# Objective
prob += 400000*x1 + 5000*x2 + 3000*x3

# Constraints
prob += 1000*x1 + 200*x2 + 100*x3 <= 10000
prob += x2 <= 15
prob += 2*x3 <= x1 + x2            # at most one third on wassa
prob += 19*x1 >= x2 + x3           # at least 5% on z-tube

# Solve
prob.solve(PULP_CBC_CMD(msg=False))

print(x1.value(), x2.value(), x3.value(), "Total viewers:", 400000*x1.value() + 5000*x2.value() + 3000*x3.value())

Result from the model: x1 = 10, x2 = 0, x3 = 0, total viewers = 4,000,000.

```python
#!/usr/bin/env python3
# Self-contained script to solve the ILP by complete enumeration (no external solver required).

# Data / parameters
audience = {"x1": 400000, "x2": 5000, "x3": 3000}
cost = {"x1": 1000, "x2": 200, "x3": 100}
budget = 10000
soorchle_limit = 15  # x2 <= 15

# Search bounds derived from budget and explicit limits
max_x1 = budget // cost["x1"]          # at most 10
max_x2 = soorchle_limit                # at most 15
max_x3 = budget // cost["x3"]          # at most 100, but tighter bounds apply in loop

best_obj = None
best_solution = None

for x1 in range(0, max_x1 + 1):
    # x2 limited by its explicit cap
    for x2 in range(0, max_x2 + 1):
        # Budget remaining after x1 and x2
        remaining_budget = budget - (cost["x1"] * x1 + cost["x2"] * x2)
        if remaining_budget < 0:
            continue
        # upper bound for x3 from budget
        ub_x3_budget = remaining_budget // cost["x3"]
        # upper bound from "at most one third on wassa": 2*x3 <= x1 + x2 -> x3 <= (x1+x2)//2
        ub_x3_fraction = (x1 + x2) // 2
        # upper bound from "at least 5% z-tube": 19*x1 >= x2 + x3 -> x3 <= 19*x1 - x2
        ub_x3_zshare = 19 * x1 - x2
        ub_x3 = min(ub_x3_budget, ub_x3_fraction, ub_x3_zshare)
        if ub_x3 < 0:
            continue
        for x3 in range(0, ub_x3 + 1):
            # All constraints already enforced by bounds, but double-check for safety
            if cost["x1"] * x1 + cost["x2"] * x2 + cost["x3"] * x3 > budget:
                continue
            if x2 > soorchle_limit:
                continue
            if 2 * x3 > x1 + x2:
                continue
            if 19 * x1 < x2 + x3:
                continue
            # feasible, compute objective
            obj = audience["x1"] * x1 + audience["x2"] * x2 + audience["x3"] * x3
            if (best_obj is None) or (obj > best_obj):
                best_obj = obj
                best_solution = (x1, x2, x3)

# If no feasible solution found (shouldn't happen), set best_obj to 0
if best_obj is None:
    best_obj = 0

# Final required print
print(f"FINAL_OBJ: {best_obj}")
```
