Formal model

Decision variables
- g ∈ Z≥0 : number of gas grills
- e ∈ Z≥0 : number of electric grills

Objective
- Minimize total grills: minimize Z = g + e

Constraints
1. Production requirement (patties per minute):
   20 g + 30 e ≥ 150

2. Cooking oil capacity (units per minute):
   20 g + 25 e ≤ 140

3. Electric grills strictly fewer than gas grills:
   e < g. (With integer variables this is equivalent to e ≤ g − 1.)

4. Integrality and nonnegativity:
   g, e ∈ Z, g ≥ 0, e ≥ 0

Remarks on feasibility and solution

- The two continuous (real-valued) linear constraints intersect at
  20g + 30e = 150 and 20g + 25e = 140 ⇒ e = 2, g = 4.5.
  That continuous intersection is feasible for the e < g requirement (2 < 4.5) and gives objective value 6.5.

- However, with the integer restriction and the strict requirement e < g (i.e. e ≤ g − 1), there is no integer pair (g,e) that satisfies all constraints simultaneously. A short enumeration over feasible g (note oil constraint implies g ≤ 7) shows no integer e exists such that
  (150 − 20g)/30 ≤ e ≤ (140 − 20g)/25 and e ≤ g − 1.

- If the wording is intended to allow equality (i.e. “the number of electric grills must be no more than the number of gas grills”, e ≤ g), then the best integer solution is
  g = 3, e = 3,
  which satisfies 20·3 + 30·3 = 150 (production), 20·3 + 25·3 = 135 ≤ 140 (oil), and gives minimal total grills Z = 6.

Actionable options
- If the strict inequality e < g is required: the integer problem is infeasible — the chain cannot meet the patties and oil limits while keeping strictly fewer electric than gas grills.
- If the intention was e ≤ g (no more electric than gas), then the optimal integer solution is g = 3, e = 3 (total grills = 6).
- If strict inequality must be kept but buying fractional grills is allowed (not realistic), the LP optimum is g = 4.5, e = 2 (objective 6.5).

#!/usr/bin/env python3
# Self-contained script to analyze the grill-sizing integer program described.
# It enumerates integer (g,e) solutions for both strict (e < g) and non-strict (e <= g)
# electric-vs-gas requirements, reports feasibility and optimal integer solutions,
# and falls back to the continuous LP intersection if both integer formulations are infeasible.
#
# At the very end this script prints exactly one final line in the required format:
# print(f"FINAL_OBJ: {value}")
#
# No external solver libraries are required; the integer domain is small so simple enumeration is used.

def main():
    # Problem coefficients (initialized directly)
    # Constraints:
    #  20*g + 30*e >= 150   (production)
    #  20*g + 25*e <= 140   (oil capacity)
    # Variables g,e are nonnegative integers.
    coef_prod_g, coef_prod_e, rhs_prod = 20, 30, 150
    coef_oil_g, coef_oil_e, rhs_oil = 20, 25, 140

    # Reasonable enumeration bounds: g and e won't be large due to constraints.
    # Oil constraint alone implies 20*g <= 140 => g <= 7, and 25*e <= 140 => e <= 5 (if g=0).
    max_g = 20  # safe upper bound
    max_e = 20

    strict_feasible_solutions = []
    nonstrict_feasible_solutions = []

    for g in range(0, max_g + 1):
        for e in range(0, max_e + 1):
            prod_ok = (coef_prod_g * g + coef_prod_e * e) >= rhs_prod
            oil_ok = (coef_oil_g * g + coef_oil_e * e) <= rhs_oil
            if not (prod_ok and oil_ok):
                continue
            # strict: e < g  --> e <= g-1
            if e <= g - 1:
                strict_feasible_solutions.append((g, e, g + e))
            # non-strict: e <= g
            if e <= g:
                nonstrict_feasible_solutions.append((g, e, g + e))

    # Find minima if any
    strict_min = None
    nonstrict_min = None

    if strict_feasible_solutions:
        strict_min = min(strict_feasible_solutions, key=lambda x: x[2])
    if nonstrict_feasible_solutions:
        nonstrict_min = min(nonstrict_feasible_solutions, key=lambda x: x[2])

    # Continuous intersection (LP relaxation) computed analytically:
    # Solve:
    # 20g + 30e = 150
    # 20g + 25e = 140
    # Subtracting gives 5e = 10 -> e = 2, then g = (150 - 30*2)/20 = 4.5
    cont_g = 4.5
    cont_e = 2.0
    cont_obj = cont_g + cont_e  # 6.5

    # Reporting results (these prints are allowed; final printed line is the required FINAL_OBJ line)
    if strict_min:
        g_star, e_star, obj_star = strict_min
        print("Strict (e < g) integer model: feasible")
        print(f"  Best integer solution (strict): g = {g_star}, e = {e_star}, total = {obj_star}")
    else:
        print("Strict (e < g) integer model: infeasible")

    if nonstrict_min:
        g2, e2, obj2 = nonstrict_min
        print("Non-strict (e <= g) integer model: feasible")
        print(f"  Best integer solution (non-strict): g = {g2}, e = {e2}, total = {obj2}")
    else:
        print("Non-strict (e <= g) integer model: infeasible")

    print("Continuous LP intersection (relax integrality):")
    print(f"  g = {cont_g}, e = {cont_e}, total = {cont_obj}")

    # Decide which objective to present as the final value:
    # Preference order:
    # 1) Strict integer solution if feasible
    # 2) Else non-strict integer solution if feasible
    # 3) Else continuous LP objective (fractional grills)
    if strict_min:
        final_obj_value = strict_min[2]
    elif nonstrict_min:
        final_obj_value = nonstrict_min[2]
    else:
        final_obj_value = cont_obj

    # Final required single-line output (must be the very last printed line)
    print(f"FINAL_OBJ: {final_obj_value}")

if __name__ == "__main__":
    main()
