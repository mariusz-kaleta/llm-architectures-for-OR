[MODELER]:
Formal model (linear program)

Decision variables
- x = units of coconut oil added (>= 0)
- y = units of lavender added (>= 0)
You may treat x,y as continuous (units large) or integer if you require integer units.

Constraints
1. Minimum coconut oil: x >= 300
2. Maximum of each ingredient: x <= 550, y <= 550
3. At most three times as much coconut as lavender: x <= 3 y
4. Nonnegativity: x >= 0, y >= 0

Objective
Minimize the total time until the mixture is effective. If each unit contributes additively to processing time (units are processed sequentially or their required times sum), the total time is
Minimize T = 0.7 x + 0.9 y

Remarks on "one must be added before the other"
- If the interpretation is that you cannot add both simultaneously but each unit's required time simply adds, the total time is 0.7x + 0.9y and the order does not change the sum.
- If you intended a different makespan model (e.g., only the last finishing unit matters), state that explicitly and the objective would change (it would become a minimax/makespan problem).

Solve the LP
Because both coefficients in the objective are positive, the objective is minimized by choosing the smallest feasible x and y. From constraints:
- x must be at least 300, so take x = 300.
- From x <= 3 y, we get y >= x/3 = 100, and there is no lower bound on y other than this, so take y = 100.

Optimal solution: x = 300 units coconut, y = 100 units lavender.
Total time = 0.7*300 + 0.9*100 = 210 + 90 = 300 minutes.

Suggested Python libraries to solve
- pulp (simple LP modeling + solvers)
- scipy.optimize.linprog (for continuous LPs)
- OR-Tools linear_solver (if you prefer Google OR-Tools)
- CVXPY (if you want a higher-level convex optimization interface)

Example using PuLP
(you can run this in Python)
from pulp import LpProblem, LpMinimize, LpVariable, LpStatus, value

prob = LpProblem("Mix_Time_Min", LpMinimize)
x = LpVariable('coconut', lowBound=0, upBound=550)
y = LpVariable('lavender', lowBound=0, upBound=550)
prob += 0.7*x + 0.9*y                      # objective
prob += x >= 300                           # min coconut
prob += x <= 3*y                           # coconut at most 3*lavender

prob.solve()
print("Status:", LpStatus[prob.status])
print("coconut =", value(x))
print("lavender =", value(y))
print("Total time =", value(prob.objective))

This will give the same result: coconut = 300, lavender = 100, total time = 300 minutes.

[PROGRAMMER-VALIDATOR]:
import itertools

# Problem data
min_x = 300.0
max_x = 550.0
max_y = 550.0

# Objective coefficients
c_x = 0.7
c_y = 0.9

# Tolerance for feasibility checks
tol = 1e-9

# Define the set of equality constraints to consider for vertices:
# Represented as tuples ('type', value) where type in {'x_const','y_const','x_eq_3y'}
eq_constraints = [
    ('x_const', min_x),
    ('x_const', max_x),
    ('y_const', max_y),
    ('x_eq_3y', None)
]

def solve_pair(a, b):
    # Solve intersection of two equalities from eq_constraints
    # Returns (x,y) or None if inconsistent
    (ta, va) = a
    (tb, vb) = b

    # If both are same exact constraints, skip
    if a == b:
        return None

    # Handle various combinations
    if ta == 'x_const' and tb == 'y_const':
        return (va, vb)
    if ta == 'y_const' and tb == 'x_const':
        return (vb, va)
    if ta == 'x_const' and tb == 'x_eq_3y':
        return (va, va / 3.0)
    if ta == 'x_eq_3y' and tb == 'x_const':
        return (vb, vb / 3.0)
    if ta == 'y_const' and tb == 'x_eq_3y':
        return (3.0 * vb, vb)
    if ta == 'x_eq_3y' and tb == 'y_const':
        return (3.0 * va if va is not None else None, va if va is not None else vb)

    # Both x_const but different values -> inconsistent
    if ta == 'x_const' and tb == 'x_const':
        return None
    if ta == 'y_const' and tb == 'y_const':
        return None

    return None

def is_feasible(x, y):
    if x is None or y is None:
        return False
    if x + tol < min_x:     # x >= min_x
        return False
    if x - tol > max_x:     # x <= max_x
        return False
    if y - tol > max_y:     # y <= max_y
        return False
    if x - 3.0*y > tol:     # x <= 3y  -> x - 3y <= 0
        return False
    if x < -tol or y < -tol:
        return False
    return True

candidates = []

# include intersections of every pair of equality constraints
for a, b in itertools.combinations(eq_constraints, 2):
    pt = solve_pair(a, b)
    if pt is not None:
        x, y = pt
        if is_feasible(x, y):
            candidates.append((x, y))

# Also consider some boundary corners explicitly to be safe
explicit_points = [
    (min_x, min_x / 3.0),       # x = min_x, x = 3y -> y = x/3
    (min_x, max_y),             # x = min_x, y = max_y
    (max_x, max_x / 3.0),       # x = max_x, x = 3y
    (max_x, max_y),             # x = max_x, y = max_y
]

for x, y in explicit_points:
    if is_feasible(x, y):
        candidates.append((x, y))

# Remove duplicates (within tolerance)
unique = []
for x, y in candidates:
    found = False
    for ux, uy in unique:
        if abs(ux - x) <= 1e-8 and abs(uy - y) <= 1e-8:
            found = True
            break
    if not found:
        unique.append((x, y))

# If no candidates found (shouldn't happen), fallback to analytical feasible point
if not unique:
    # feasible minimal choices by logic
    x = min_x
    y = x / 3.0
    unique = [(x, y)]

# Evaluate objective at candidates and pick best (minimum)
best_obj = None
best_pt = None
for x, y in unique:
    obj = c_x * x + c_y * y
    if best_obj is None or obj < best_obj - tol:
        best_obj = obj
        best_pt = (x, y)

# As a sanity check, include the analytically obvious feasible point (300,100)
analytical_x = min_x
analytical_y = analytical_x / 3.0
if is_feasible(analytical_x, analytical_y):
    analytical_obj = c_x * analytical_x + c_y * analytical_y
    if best_obj is None or analytical_obj < best_obj - tol:
        best_obj = analytical_obj
        best_pt = (analytical_x, analytical_y)

# Final objective value
value = best_obj if best_obj is not None else 0.0

print(f"FINAL_OBJ: {value}")

import itertools

# Problem data
min_x = 300.0
max_x = 550.0
max_y = 550.0

# Objective coefficients
c_x = 0.7
c_y = 0.9

tol = 1e-9

# Represent equality constraints as equations a*x + b*y = c
# x = 300 -> 1*x + 0*y = 300
# x = 550 -> 1*x + 0*y = 550
# y = 550 -> 0*x + 1*y = 550
# x = 3y  -> 1*x - 3*y = 0
eqs = [
    (1.0, 0.0, min_x),
    (1.0, 0.0, max_x),
    (0.0, 1.0, max_y),
    (1.0, -3.0, 0.0)
]

def intersect(e1, e2):
    a1, b1, c1 = e1
    a2, b2, c2 = e2
    det = a1 * b2 - a2 * b1
    if abs(det) <= tol:
        return None
    x = (c1 * b2 - c2 * b1) / det
    y = (a1 * c2 - a2 * c1) / det
    return (x, y)

def is_feasible(x, y):
    if x is None or y is None:
        return False
    if x + tol < min_x:   # x >= min_x
        return False
    if x - tol > max_x:   # x <= max_x
        return False
    if y - tol > max_y:   # y <= max_y
        return False
    if x - 3.0 * y > tol: # x <= 3y
        return False
    if x < -tol or y < -tol:
        return False
    return True

candidates = []

# Intersections of every pair of equality constraints
for e1, e2 in itertools.combinations(eqs, 2):
    pt = intersect(e1, e2)
    if pt is not None:
        x, y = pt
        if is_feasible(x, y):
            candidates.append((x, y))

# Also add some obvious boundary points (corners)
explicit_points = [
    (min_x, min_x / 3.0),  # x=min_x with x=3y
    (min_x, max_y),        # x=min_x, y=max_y
    (max_x, max_x / 3.0),  # x=max_x with x=3y
    (max_x, max_y),        # x=max_x, y=max_y
    # Also the analytically expected optimum
    (min_x, min_x / 3.0)
]

for x, y in explicit_points:
    if is_feasible(x, y):
        candidates.append((x, y))

# Remove duplicates within tolerance
unique = []
for x, y in candidates:
    found = False
    for ux, uy in unique:
        if abs(ux - x) <= 1e-8 and abs(uy - y) <= 1e-8:
            found = True
            break
    if not found:
        unique.append((x, y))

# If no candidates found (shouldn't happen), fallback to analytical point
if not unique:
    x = min_x
    y = x / 3.0
    unique = [(x, y)]

# Evaluate objective
best_obj = None
best_pt = None
for x, y in unique:
    obj = c_x * x + c_y * y
    if best_obj is None or obj < best_obj - tol:
        best_obj = obj
        best_pt = (x, y)

# As an extra check, directly evaluate the analytically obvious feasible point
anal_x = min_x
anal_y = anal_x / 3.0
if is_feasible(anal_x, anal_y):
    anal_obj = c_x * anal_x + c_y * anal_y
    if best_obj is None or anal_obj < best_obj - tol:
        best_obj = anal_obj
        best_pt = (anal_x, anal_y)

value = best_obj if best_obj is not None else 0.0

print(f"FINAL_OBJ: {value}")
