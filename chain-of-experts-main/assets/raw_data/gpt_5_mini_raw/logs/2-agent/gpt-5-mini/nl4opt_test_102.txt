Below is a clear linear-programming formulation and a short note on solving it.

Decision variables
- R = number of servings of Rice (servings/day), R ≥ 0
- K = number of servings of Kebab (servings/day), K ≥ 0

Objective
- Minimize total daily cost:  minimize Z = 3R + 2K

Constraints
- Calories constraint: 300R + 200K ≥ 2200
- Protein constraint: 4.5R + 4K ≥ 30
- Nonnegativity: R ≥ 0, K ≥ 0

(If servings must be integral, add integer constraints R, K integer; otherwise treat as continuous.)

Remarks / analytic solution
- The LP is feasible. The minimum cost is $22. One optimal solution is R = 22/3 ≈ 7.333 servings (Rice) and K = 0, cost = 3*(22/3) = 22. Another is R = 0, K = 11, cost = 2*11 = 22. In fact any nonnegative (R,K) satisfying 300R + 200K = 2200 also meets the protein requirement and yields cost 22, so there is an entire segment of optimal solutions.

Python libraries for solving
- pulp (easy-to-use LP modeling with CBC solver)
- ortools (Google OR-Tools linear solver)
- scipy.optimize.linprog (simple LP solver)
- pyomo (more advanced modeling; works with many solvers)
- cvxopt (convex optimization solver)

Optional example (Pulp)
- Define variables R,K >=0
- Objective: lpMinimize(3*R + 2*K)
- Constraints: 300*R + 200*K >= 2200; 4.5*R + 4*K >= 30
- Solve with default CBC solver — will return cost 22 and one optimal (R,K).

# Problem data
cost_R = 3.0    # cost per serving of Rice
cost_K = 2.0    # cost per serving of Kebab
cal_R = 300.0   # calories per serving of Rice
cal_K = 200.0   # calories per serving of Kebab
req_cal = 2200.0
# Protein data (kept for completeness; will be shown to be redundant)
prot_R = 4.5
prot_K = 4.0

# Compute cost per calorie for each food
cpc_R = cost_R / cal_R
cpc_K = cost_K / cal_K

# Because cpc_R == cpc_K, any combination meeting calorie requirement on equality
# gives the same cost. Compute minimal cost as cost_per_calorie * required calories.
# Use average of the two computed (they are equal within floating tolerance).
cpc = 0.5 * (cpc_R + cpc_K)
final_obj = cpc * req_cal

# If effectively an integer, make it int for cleaner output
if abs(final_obj - round(final_obj)) < 1e-9:
