1) Decision variables
- x = number of LED fixtures (integer, x ≥ 0)
- y = number of fluorescence fixtures (integer, y ≥ 0)

2) Objective
Minimize the total number of light changes per decade:
Minimize Z = 3x + 4y
(LEDs are changed 3 times/decade, fluorescents 4 times/decade.)

3) Constraints
- Minimum fixtures: x + y ≥ 300
- Electricity limit (units per hour): 5x + 8y ≤ 2000
- At least 30% fluorescence: y ≥ 0.30 (x + y)
  (equivalently 7y ≥ 3x, or y ≥ (3/7)x, or implement as integer constraint y ≥ ceil(0.3*(x+y)))
- x, y integer and nonnegative

4) Analytical solution (reasoning)
- For fixed total N = x + y, objective = 3x + 4y = 3N + y. So for fixed N you minimize Z by minimizing y.
- The 30% requirement forces y ≥ 0.3 N. To minimize changes you should pick the smallest feasible N (because Z grows roughly as 3.3N if y is at the 30% minimum).
- The smallest allowable N is 300, so set N = 300 and y = 0.3·300 = 90 (integer), x = 300 − 90 = 210.
- Check electricity: 5·210 + 8·90 = 1050 + 720 = 1770 ≤ 2000, so feasible.

Thus the optimal integer solution is:
- x = 210 LEDs
- y = 90 fluorescent lamps

Total changes per decade: Z = 3·210 + 4·90 = 990
Electricity used: 1770 units/hour (within the 2000 limit).

# Bounds from electricity constraint:
max_x = 2000 // 5  # 400
max_y = 2000 // 8  # 250

best_val = None
best_sol = None

for x in range(0, max_x + 1):
    # Given 7*y >= 3*x -> y >= ceil(3*x/7)
    y_min_from_ratio = (3 * x + 6) // 7  # ceil(3*x/7)
    # Also x+y >= 300 -> y >= 300 - x

    y_start = max(y_min_from_ratio, y_min_from_total)
    if y_start > max_y:
        continue
    for y in range(y_start, max_y + 1):
        if 5 * x + 8 * y > 2000:
            # since y increases, further y will only increase electricity use; break inner loop
            break

        # x + y >= 300 already enforced by starting y
        Z = 3 * x + 4 * y
        if (best_val is None) or (Z < best_val):
            best_val = Z
            best_sol = (x, y)

# Print results (informational)
