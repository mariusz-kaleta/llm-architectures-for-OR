[MODELER]:
Formal model

Variables
- x = number of crab cakes (meals), x >= 0
- y = number of lobster rolls (meals), y >= 0

Objective
- Minimize total unsaturated fat: minimize F = 4x + 6y

Constraints
- Vitamin A: 5x + 8y >= 80
- Vitamin C: 7x + 4y >= 100
- Lobster fraction (at most 40%): y <= 0.4(x + y). Equivalently 0.6y <= 0.4x → 3y <= 2x (or y <= (2/3)x)
- x, y >= 0
(If meals must be whole, add integer constraints x, y ∈ Z≥0; otherwise treat as continuous.)

Continuous linear program (allowing fractional meals)
- Solve the LP min 4x+6y subject to the constraints above.
- The feasible corner points to check include:
  1) intersection of both vitamin equalities:
     solve 5x + 8y = 80 and 7x + 4y = 100 → (x,y) = (40/3, 5/3) ≈ (13.333, 1.667).
     F = 4*(40/3) + 6*(5/3) = 190/3 ≈ 63.333.
  2) (x,y) = (16,0) (meets A and C, y=0): F = 64.
  3) intersection of vitamin C equality and lobster-fraction boundary:
     (x,y) = (300/29, 200/29) ≈ (10.345, 6.897): F ≈ 82.76.
- The minimum for the continuous LP is at (x,y) = (40/3, 5/3) with F = 190/3 ≈ 63.333 unsaturated-fat units.

Integer (whole-meal) version
- Meals must be integer. Check nearby integer feasible solutions; the best integer solution is
  x = 16 crab cakes, y = 0 lobster rolls (this meets both vitamin requirements and the 40% rule).
  F = 4*16 + 6*0 = 64.
- No integer feasible pair gives total fat below 64.

Recommended solvers / libraries
- Continuous LP: scipy.optimize.linprog, PuLP (with CBC), OR-Tools linear solver.
- Mixed-integer (integer meals): PuLP (CBC, GLPK), OR-Tools CP-SAT or MIP solver, or commercial solvers (Gurobi, CPLEX) if available.
- For a quick Python MILP: use PuLP or OR-Tools CP-SAT.

Short PuLP sketch (MILP)
- Define LpVariables x,y with cat='Integer' (or 'Continuous').
- Add constraints 5*x + 8*y >= 80; 7*x + 4*y >= 100; 3*y <= 2*x; x>=0; y>=0.
- Minimize 4*x + 6*y and solve.

Answer
- If fractional meals are allowed: eat 40/3 ≈ 13.333 crab cakes and 5/3 ≈ 1.667 lobster rolls (min unsat fat ≈ 63.333 units).
- If meals must be whole: eat 16 crab cakes and 0 lobster rolls (min unsat fat = 64 units).

import math

# Solve the continuous LP (allow fractional meals) and the integer (whole meals) version by brute force.
# Variables:
# x = crab cakes, y = lobster rolls
# Objective: minimize F = 4x + 6y
# Constraints:
# 5x + 8y >= 80
# 7x + 4y >= 100
# 3y <= 2x  (equivalent to y <= (2/3)x)
# x, y >= 0

# First try to solve using scipy.optimize.linprog for the continuous relaxation.
use_linprog = True
continuous_solution = None

try:
    from scipy.optimize import linprog

    # Minimize c^T * [x,y]
    c = [4.0, 6.0]

    # Convert >= constraints to <= by multiplying by -1:
    # -5x -8y <= -80
    # -7x -4y <= -100
    A_ub = [
        [-5.0, -8.0],
        [-7.0, -4.0],
        [-2.0, 3.0],  # -2x + 3y <= 0  corresponds to 3y <= 2x
    ]
    b_ub = [-80.0, -100.0, 0.0]

    bounds = [(0, None), (0, None)]

    res = linprog(c=c, A_ub=A_ub, b_ub=b_ub, bounds=bounds, method="highs")
    if res.success:
        x_cont, y_cont = res.x
        obj_cont = res.fun
        continuous_solution = (x_cont, y_cont, obj_cont)
    else:
        use_linprog = False
except Exception:
    use_linprog = False

# If linprog unavailable or failed, solve analytically by checking corner points.
if not continuous_solution:
    # The relevant corner is intersection of both vitamin equalities:
    # 5x + 8y = 80
    # 7x + 4y = 100
    # Solve 2x2 system
    # Use linear algebra
    a = [[5.0, 8.0], [7.0, 4.0]]
    b = [80.0, 100.0]
    det = a[0][0]*a[1][1] - a[0][1]*a[1][0]
    if abs(det) < 1e-12:
        raise RuntimeError("Singular system unexpectedly.")
    x_cont = (b[0]*a[1][1] - a[0][1]*b[1]) / det
    y_cont = (a[0][0]*b[1] - b[0]*a[1][0]) / det
    obj_cont = 4.0*x_cont + 6.0*y_cont
    continuous_solution = (x_cont, y_cont, obj_cont)

# Now solve integer version by brute force search over reasonable bounds.
# From constraints, we can bound x and y to a reasonable max.
# For example, if y = 0 then x must be at least 16 (from 5x>=80). If x = 0 then y must be at least 10 (from 7x+4y>=100 -> y>=25).
# We'll search x in [0, 100], y in [0, 100], which is small enough.
best_int = None  # tuple (obj, x, y)
max_search = 100
for x in range(0, max_search + 1):
    for y in range(0, max_search + 1):
        if 5*x + 8*y < 80:
            continue
        if 7*x + 4*y < 100:
            continue
        if 3*y > 2*x:
            continue
        obj = 4*x + 6*y
        if (best_int is None) or (obj < best_int[0]) or (obj == best_int[0] and (x + y) < (best_int[1] + best_int[2])):
            best_int = (obj, x, y)

# Print summary (these prints are allowed; final required line is printed last)
print("Continuous (fractional) solution:")
print(f" x = {continuous_solution[0]:.12f}, y = {continuous_solution[1]:.12f}, objective = {continuous_solution[2]:.12f}")
if best_int:
    print("Best integer (whole meals) solution found by brute force:")
    print(f" x = {best_int[1]}, y = {best_int[2]}, objective = {best_int[0]}")
else:
    print("No integer feasible solution found within search bounds.")

# Final required single-line output exactly in the specified format.
# We'll report the continuous LP optimal objective as the final objective value (the LP was the primary task).
final_value = continuous_solution[2]
print(f"FINAL_OBJ: {final_value}")
