[MODELER]:
Model

- Decision variables
  - x = number of basketballs produced (integer, x ≥ 0)
  - y = number of footballs produced (integer, y ≥ 0)

- Objective
  - Maximize total items: maximize Z = x + y

- Constraints
  1. Material limit: 5x + 3y ≤ 1500
  2. Time limit: x + 2y ≤ 750
  3. Basketballs at least three times footballs: x ≥ 3y
  4. At least 50 footballs: y ≥ 50
  5. Integrality and nonnegativity: x, y ∈ Z, x ≥ 0, y ≥ 0

Remarks on solving
- The LP relaxation (allowing real x,y) shows the time constraint is never binding given the ratio and material limits; the intersection of x = 3y and 5x + 3y = 1500 gives the continuous optimum at y = 1500/18 = 83.333..., x = 3y = 250, total = 333 1/3.
- Because x,y must be integers, y must be an integer ≤ 83 (y = 84 is infeasible). For y = 83 the material constraint allows x up to 250, and x must be ≥ 3·83 = 249; choosing x = 250 yields the integer optimum.

Solution
- Produce 250 basketballs and 83 footballs.
- Total equipment = 250 + 83 = 333 items.

Suggested solvers / Python libraries
- For integer (MILP) solve: PuLP (with CBC), OR-Tools CP-SAT, or commercial solvers (Gurobi, CPLEX).
- For LP relaxation only: scipy.optimize.linprog (not integer).
- Example libraries: pulp, ortools (recommended: OR-Tools CP-SAT for guaranteed integer optimality; PuLP for simple MILP formulation with CBC).

[PROGRAMMER-VALIDATOR]:
import math

# Problem data (initialized directly)
# Decision variables: x (basketballs), y (footballs) integers >= 0
# Constraints:
# 1) 5x + 3y <= 1500
# 2) x + 2y <= 750
# 3) x >= 3y
# 4) y >= 50

best_obj = -1
best_x = None
best_y = None

# Reasonable bounds for search
y_min = 50
y_max = 500  # upper bound: material alone would allow up to 500 footballs (3*y <= 1500 -> y<=500)

for y in range(y_min, y_max + 1):
    # Compute max x allowed by material: 5x <= 1500 - 3y -> x <= (1500 - 3y)/5
    rem_material = 1500 - 3 * y
    if rem_material < 0:
        continue
    x_mat_max = rem_material // 5  # integer floor

    # Compute max x allowed by time: x <= 750 - 2y
    rem_time = 750 - 2 * y
    if rem_time < 0:
        continue
    x_time_max = rem_time  # already integer

    x_max = min(x_mat_max, x_time_max)

    # x must be at least 3y
    x_min_req = 3 * y

    if x_max < x_min_req:
        continue

    # Choose x to maximize objective x + y => choose x = x_max
    x = x_max
    obj = x + y

    if obj > best_obj:
        best_obj = obj
        best_x = x
        best_y = y

# Print solution details (optional)
print(f"Optimal solution found: x = {best_x}, y = {best_y}, total = {best_obj}")

# Final required line (must be the last line exactly in this format)
print(f"FINAL_OBJ: {best_obj}")
