[MODELER]:
Model summary

Decision variables
- x = number of scooters produced per month (x ≥ 0)
- y = number of bikes produced per month (y ≥ 0)

Objective
- Maximize profit: maximize Z = 200x + 300y

Constraints
- Design team time: 2x + 4y ≤ 5000
- Engineering team time: 3x + 5y ≤ 6000
- Nonnegativity: x ≥ 0, y ≥ 0
(Optionally add integer constraints x, y ∈ Z+ if discrete units required.)

Solution (analytical)
Evaluate corner points of the feasible region:
- x = 0 ⇒ y ≤ min(5000/4, 6000/5) = min(1250, 1200) ⇒ y = 1200, profit = 300·1200 = $360,000
- y = 0 ⇒ x ≤ min(5000/2, 6000/3) = min(2500, 2000) ⇒ x = 2000, profit = 200·2000 = $400,000
- intersection of the two resource constraints (2x+4y=5000 and 3x+5y=6000) gives x = −500 (infeasible)

Therefore the optimal feasible choice is x = 2000 scooters and y = 0 bikes, with maximum profit $400,000 per month.

Intuition
- Engineering is the binding resource (3·2000 = 6000 hours used). Profit per engineering hour: scooters 200/3 ≈ 66.67, bikes 300/5 = 60, so scooters give more profit per engineering hour.

Recommended solvers/libraries
- For linear programming: pulp (MILP/LP), scipy.optimize.linprog, OR-Tools (CP-SAT or linear solver)
- For integer/MILP: pulp, OR-Tools, or commercial solvers (Gurobi, CPLEX) if available

Example (Python with pulp)

import math

# Problem data
profit_x = 200.0  # profit per scooter
profit_y = 300.0  # profit per bike

# Constraints:
# 2x + 4y <= 5000
# 3x + 5y <= 6000
# x >= 0, y >= 0

# Compute axis intercepts (maximums when the other variable is zero)
x_max = min(5000.0 / 2.0, 6000.0 / 3.0)
y_max = min(5000.0 / 4.0, 6000.0 / 5.0)

# Solve intersection of the two constraint lines:
# 2x + 4y = 5000
# 3x + 5y = 6000
A = [[2.0, 4.0],
     [3.0, 5.0]]
B = [5000.0, 6000.0]

det = A[0][0]*A[1][1] - A[0][1]*A[1][0]
intersection = None
if abs(det) > 1e-12:
    xi = (B[0]*A[1][1] - A[0][1]*B[1]) / det
    yi = (A[0][0]*B[1] - B[0]*A[1][0]) / det
    intersection = (xi, yi)

# Candidate corner points
candidates = [
    (0.0, 0.0),
    (x_max, 0.0),
    (0.0, y_max)
]
if intersection is not None:
    xi, yi = intersection
    # Include intersection only if it is feasible (nonnegative)
    if xi >= -1e-9 and yi >= -1e-9:
        # clip tiny negatives to zero
        xi = max(0.0, xi)
        yi = max(0.0, yi)
        candidates.append((xi, yi))

# Evaluate objective at each candidate
best_point = None
best_value = -math.inf
for (x, y) in candidates:
    # Check feasibility explicitly (to be safe with numerical issues)
    if 2.0*x + 4.0*y <= 5000.0 + 1e-8 and 3.0*x + 5.0*y <= 6000.0 + 1e-8 and x >= -1e-9 and y >= -1e-9:
        value = profit_x * x + profit_y * y
        if value > best_value:
            best_value = value
            best_point = (x, y)

# Prepare final objective value (as integer if very close)
if abs(best_value - round(best_value)) < 1e-6:
    value = int(round(best_value))
else:
    value = float(best_value)

# Final output line (must be exactly this format)
print(f"FINAL_OBJ: {value}")
